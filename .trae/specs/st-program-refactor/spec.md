# ST程序架构重构 Spec

## Why

当前ST程序架构存在以下核心问题：

1. **过度碎片化**：17+个功能块，输送机拆成8个FB，功能散落三层
2. **IO映射混乱**：主程序和Device层都有IO映射，点位变更时修改分散
3. **HMI交互缺失**：工站未对接HMI数据（模式选择、参数设置、状态显示），数据类型不全(只有bool缺int/real)
4. **报警机制不完整**：无公共报警汇总区(D区)，无法统一管理，无MES扩展能力
5. **工站职责不清**：工站不应直接操作硬件地址，应接收主控下发的逻辑信号，输出结果给主控判断
6. **工艺与程序不匹配**：实际"4层输送→取放料→打胶机送料"三段式未体现

## What Changes

### 核心架构：主控(含IO映射) + 3工站(纯逻辑) + 公共服务

```
┌──────────────────────────────────────────────────────────────┐
│                        主控程序                               │
│  职责：                                                       │
│  ① IO映射(X→逻辑输入, 逻辑输出→Y)                            │
│  ② HMI数据读取(M/D区) → 分发给工站(bool/int/real)            │
│  ③ 外部信号接收(打胶机等) → 分发给工站                        │
│  ④ 工站实例化 + 工站间信号连接                                │
│  ⑤ 工站输出汇总 → 驱动Y地址 + 回写M/D区给HMI                 │
│  ⑥ 报警汇总到D区                                              │
└──┬─────────┬─────────┬─────────┬──────────┬──────────────────┘
   │         │         │         │          │
   ▼         ▼         ▼         ▼          ▼
 X地址输入  HMI(M/D)  打胶机信号  工站输出   Y地址输出
   │         │         │         ↑          │
   └────┬────┴────┬────┴─────────┤──────────┘
        │         │              │
   ┌────▼────┬────▼────┐   ┌────▼────┐
   │四层输送机│取放料机构│   │打胶机送料│
   │ (纯逻辑) │ (纯逻辑) │   │ (纯逻辑) │
   └─────────┴─────────┘   └─────────┘
```

**关键设计原则**：
- **FB不碰X/Y/M/D地址** — 所有硬件交互由主控负责
- **FB输入全部来自主控** — 包括传感器信号(HMI/外部)、控制命令(HMI)、参数(HMI)、上游工站信号
- **FB输出全部给主控** — 包括执行器请求、状态信息、报警代码；由主控判断后驱动Y或回写M/D
- **数据类型完整** — 不只是BOOL，还包括INT(位置、计数)、REAL(速度、位置设定)

### 各组件详细定义

#### 一、主控程序 — IO映射 + 协调中心（最核心组件）

```structuredtext
PROGRAM 主控程序
VAR
    // === 工站实例 ===
    fb四层输送机      : FB_四层输送机;
    fb取放料机构      : FB_取放料机构;
    fb打胶机送料机构   : FB_打胶机送料机构;
    fb公共报警        : FB_公共报警;
    
    // === IO映射中间变量 ===
    // 四层输送机 - 传感器输入(X→逻辑)
    bLx_分料前感应器    : ARRAY[1..4] OF BOOL;
    bLx_到位感应器1     : ARRAY[1..4] OF BOOL;
    bLx_到位感应器2     : ARRAY[1..4] OF BOOL;
    bLx_阻挡气缸上位    : ARRAY[1..4] OF BOOL;
    bLx_阻挡气缸下位    : ARRAY[1..4] OF BOOL;
    bLx_分料气缸上位    : ARRAY[1..4] OF BOOL;
    bLx_分料气缸下位    : ARRAY[1..4] OF BOOL;
    
    // 四层输送机 - 执行器输出(逻辑→Y)
    bLx_阻挡电磁阀      : ARRAY[1..4] OF BOOL;
    bLx_分料电磁阀      : ARRAY[1..4] OF BOOL;
    bLx_输送带正转      : ARRAY[1..4] OF BOOL;
    bLx_输送带慢速      : ARRAY[1..4] OF BOOL;
    
    // 取放料机构 - 传感器输入
    b升降_动点          : BOOL;   // := Remote_X0
    b升降_原点          : BOOL;   // := Remote_X1
    // ... 更多传感器
    
    // 取放料机构 - 执行器输出
    b升降_上升          : BOOL;   // → Remote_Y10
    b升降_下降          : BOOL;   // → Remote_Y11
    // ... 更多执行器
    
    // HMI数据读取(M/D区)
    bHMI_自动模式       : BOOL;   // := M?
    bHMI_手动模式       : BOOL;   // := M?
    bHMI_启动           : BOOL;   // := M?
    bHMI_停止           : BOOL;   // := M?
    rHMI_输送速度设定    : REAL;   // := D?
    rHMI_伺服速度设定    : REAL;   // := D?
    iHMI_分料时间设定    : INT;    // := D?
    
    // 打胶机外部信号
    b打胶机_允许送料    : BOOL;   // := %IX76
    b打胶机_取料完成    : BOOL;   // := %IX102
    
    // 状态回写给HMI
    bHMI_系统运行中      : BOOL;   // → M?
    bHMI_系统故障        : BOOL;   // → M?
    iHMI_全局报警代码    : INT;    // → D?
END_VAR

// ========== 主程序主体 ==========
// 第一步：IO映射（X地址 → 逻辑变量）
bLx_分料前感应器[1] := %IX51;
bLx_分料前感应器[2] := %IX61;
bLx_分料前感应器[3] := %IX71;
bLx_分料前感应器[4] := %IX81;
// ... 所有X地址映射

// 第二步：HMI数据读取
bHMI_自动模式 := %MX100;
rHMI_输送速度设定 := %DW200;

// 第三步：调用各工站（传入逻辑变量，非X/Y地址）
fb四层输送机(
    i_b使能 := TRUE,
    i_b自动模式 := bHMI_自动模式,
    i_b手动模式 := bHMI_手动模式,
    i_b启动 := bHMI_启动,
    i_b停止 := bHMI_停止,
    i_r运行速度 := rHMI_输送速度设定,
    i_i定时参数 := iHMI_分料时间设定,
    // 传感器输入（已映射的逻辑变量）
    i_Lx_分料前感应器 := bLx_分料前感应器,
    i_Lx_到位感应器1 := bLx_到位感应器1,
    // ...
    // 执行器输出（逻辑变量，后续映射到Y）
    o_Lx_阻挡电磁阀 => bLx_阻挡电磁阀,
    o_Lx_分料电磁阀 => bLx_分料电磁阀,
    o_Lx_输送带正转 => bLx_输送带正转,
    // 状态输出
    o_b运行中 => bHMI_系统运行中,
    o_i本站报警代码 => ,
);

fb取放料机构(
    i_b使能 := TRUE,
    i_b自动模式 := bHMI_自动模式,
    // ... 控制信号来自HMI
    // 传感器输入（已映射）
    i_升降_动点 := b升降_动点,
    // 上游信号来自工站输出
    i_输送机_L1_放料完成 := fb四层输送机.o_L1_放料完成,
    // ...
    // 输出给主控判断
    o_升降_上升 => b升降_上升,
    o_放料完成_给送料机构 => ,
    o_i本站报警代码 => ,
);

// 第四步：输出映射（逻辑变量 → Y地址）
%RY20 := bLx_阻挡电磁阀[1];
%RY21 := bLx_阻挡电磁阀[2];
Remote_Y10 := b升降_上升;
// ...

// 第五步：HMI状态回写
%MX110 := bHMI_系统运行中;
%D300 := iHMI_全局报警代码;

// 第六步：报警汇总
fb公共报警(
    i_i输送机报警 := fb四层输送机.o_i本站报警代码,
    i_i取放料报警 := fb取放料机构.o_i本站报警代码,
    i_i送料机构报警 := fb打胶机送料机构.o_i本站报警代码,
);
%D400 := fb公共报警.o_w全局报警字;
```

#### 二、每个工站的统一结构标准（纯逻辑，不碰任何地址）

```structuredtext
FUNCTION_BLOCK FB_xxx工站
VAR_INPUT
    // === 使能与模式（来自主控/HMI）===
    i_b使能          : BOOL;
    i_b自动模式      : BOOL;     // 来自主控(读自HMI M区)
    i_b手动模式      : BOOL;
    
    // === 控制按钮（来自主控/HMI）===
    i_b启动          : BOOL;
    i_b停止          : BOOL;
    i_b复位          : BOOL;
    
    // === 手动操作（来自主控/HMI）===
    i_b点动正转      : BOOL;
    i_b点动反转      : BOOL;
    
    // === 参数设置（来自主控/HMI，支持INT/REAL）===
    i_r运行速度       : REAL;     // 速度参数
    i_r位置设定       : REAL;     // 位置参数
    i_i定时参数       : INT;      // 时间参数
    i_i计数设定       : INT;      // 计数参数
    
    // === 传感器输入（来自主控，已做IO映射）===
    i_xxx传感器1     : BOOL;
    i_xxx传感器2     : BOOL;
    i_xxx编码器值    : INT;      // 编码器反馈值
    i_xxx模拟量输入   : WORD;     // 模拟量输入
    
    // === 上游工站信号（来自主控，连接其他工站输出）===
    i_b上游完成       : BOOL;
    i_i上游状态码     : INT;      // 上游状态详情
END_VAR

VAR_OUTPUT
    // === 执行器请求（给主控，由主控映射到Y地址）===
    o_yxx执行器1      : BOOL;     // → 主控 → Y地址
    o_yxx执行器2      : BOOL;
    o_r伺服脉冲频率   : REAL;     // → 主控 → 伺服驱动
    o_i伺服目标位置   : INT;      // → 主控 → 位置指令
    
    // === 下游工站信号（给主控，由主控转发）===
    o_b本站完成       : BOOL;     // → 主控 → 下游工站输入
    o_i本站状态码     : INT;      // → 主控 → HMI显示
    
    // === 状态输出（给主控，回写给HMI）===
    o_b运行中         : BOOL;     // → 主控 → M区
    o_b故障           : BOOL;
    o_i当前状态       : INT;
    o_r当前位置       : REAL;     // 当前位置反馈
    o_i当前计数值     : INT;      // 计数反馈
    
    // === 报警输出（给主控，汇总到D区）===
    o_i本站报警代码   : INT;      // → 主控 → FB_公共报警
END_VAR

// 内部变量：纯逻辑，无任何地址引用
VAR
    s_e自动状态       : INT := 0;
    s_e手动状态       : INT := 0;
    s_t动作定时器      : TON;
    s_i内部计数        : INT := 0;
    s_r内部位置        : REAL := 0.0;
END_VAR
```

#### 四、FB_公共报警 — 报警汇总与MES扩展

```structuredtext
FUNCTION_BLOCK FB_公共报警
VAR_INPUT
    i_i输送机报警     : INT;      // 来自主控(←四层输送机输出)
    i_i取放料报警     : INT;      // 来自主控(←取放料机构输出)
    i_i送料机构报警   : INT;      // 来自主控(←送料机构输出)
    i_b复位           : BOOL;     // 来自主控(←HMI复位按钮)
END_VAR

VAR_OUTPUT
    // → 主控 → D区 → HMI读取
    o_w全局报警字     : WORD;     // 所有报警OR组合
    o_i当前报警代码   : INT;      // 最高优先级
    o_b任何报警激活   : BOOL;     // 全局互锁用
    
    // MES扩展预留
    o_iMES报警队列    : ARRAY[0..9] OF INT;
    o_iMES报警计数    : INT;
    o_b新报警标志     : BOOL;     // 新报警脉冲
END_VAR
```

**D区规划**：
| 地址 | 内容 | 方向 | 用途 |
|------|------|------|------|
| D400 | 全局报警字(WORD) | 输出 | HMI报警灯 |
| D402 | 当前报警代码(INT) | 输出 | HMI文本 |
| D404 | 报警计数(INT) | 输出 | 统计 |
| D406~D425 | 报警队列(×10) | 输出 | MES上传 |
| M200 | 任何报警激活 | 输出 | 全局互锁 |

**报警码段分配**：
| 范围 | 归属 | 示例 |
|------|------|------|
| 1~99 | 四层输送机 | 01=阻挡超时, 02=分料超时, 03=变频器故障... |
| 101~199 | 取放料机构 | 101=伺服故障, 102=夹紧超时... |
| 201~299 | 打胶机送料机构 | 201=通讯超时, 202=安全区异常... |

### FBD使用说明格式


## ADDED Requirements

### Requirement: 主控负责所有IO映射

主控程序 SHALL 负责所有IO映射工作：X地址→逻辑输入变量、逻辑输出变量→Y地址。工站FB SHALL 不包含任何X/Y/M/D地址引用。

### Requirement: FB为纯逻辑块

工站功能块 SHALL 为纯逻辑块，所有输入来自主控（包括传感器信号、HMI控制、参数、上游信号），所有输出给主控（包括执行器请求、状态、报警）。

### Requirement: 数据类型完整性

工站接口 SHALL 支持完整的数据类型：BOOL(状态/控制)、INT(位置/计数/报警码)、REAL(速度/位置设定)，不限于布尔类型。

### Requirement: HMI数据经主控分发

HMI数据(M/D区的模式、按钮、参数) SHALL 由主控统一读取并分发给各工站，不由工站直接访问M/D区。

### Requirement: 公共报警汇总到D区

所有工站报警 SHALL 经主控汇总到公共D区(D400起始)，支持HMI读取和MES上传扩展。

### Requirement: 中文命名

所有功能块名、文件夹名、文档名 SHALL 使用中文。

### Requirement: FBD使用说明

每个工站 SHALL 附带FBD格式使用说明文档，展示逻辑针脚（非物理地址）。

## REMOVED

- **REMOVED**: Device/Logic/Main三层碎片化架构
- **REMOVED**: 工站内直接操作X/Y地址的设计
- **REMOVED**: 独立的FB_Device_IOMapping（合并入主控程序）
