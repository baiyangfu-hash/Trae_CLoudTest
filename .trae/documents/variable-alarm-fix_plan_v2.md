# 变量命名与报警系统修复计划 (V2)

## 1. 问题分析

### 1.1 变量命名问题
- **功能块命名**：当前FB块命名使用中文，如`FB_四层输送机`、`FB_取放料机构`等，不符合规范
- **变量命名**：部分变量缺少类型前缀，如`bLx_分料前感应器`应改为`i_bLx_分料前感应器`
- **输入输出变量**：部分变量未遵循`i_变量类型_变量描述`和`q_变量类型_变量描述`的规范

### 1.2 报警系统问题
- **多报警处理**：当前系统只显示最高优先级报警，无法同时显示所有报警
- **HMI触发条件**：HMI显示报警事件的触发条件是bool量，需要为每个报警添加对应的bool量
- **报警查找**：当多个报警一起出现时，缺少查找机制

### 1.3 文档同步问题
- 每次迭代结束需要同步文档，确保代码与文档一致

## 2. 修复方案

### 2.1 变量命名修复
- **功能块命名**：保持中文命名，确保清晰易懂
- **变量命名**：修复变量命名，确保遵循`i_变量类型_变量描述`和`q_变量类型_变量描述`规范
- **内部变量**：保持现有命名风格，确保清晰易懂

### 2.2 报警系统完善
- **多报警显示**：修改公共报警功能块，增加报警状态数组，记录每个工站的当前报警状态
- **HMI触发条件**：为每个报警代码添加对应的bool量输出，用于HMI显示触发
- **报警查找机制**：在公共报警功能块中增加报警状态查询功能

### 2.3 文档同步
- 更新程序架构文档和详细设计说明书，确保与代码一致
- 确保报警码定义文档与实际报警代码对应

## 4. 技术要点

### 4.1 变量命名规范
- **输入变量**：`i_变量类型_变量描述`
- **输出变量**：`q_变量类型_变量描述`
- **内部变量**：`变量类型_变量描述`
- **变量类型前缀**：b(BOOL)、i(INT)、r(REAL)、s(STRING)等

### 4.2 报警系统设计
- **多报警处理**：使用数组记录每个工站的报警状态
- **HMI触发**：为每个报警代码添加对应的bool量输出
- **报警优先级**：保持现有的优先级机制，同时支持显示所有报警
- **报警队列**：保持现有的MES报警队列功能
