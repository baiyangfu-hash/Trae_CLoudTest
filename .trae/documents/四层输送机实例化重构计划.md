# 四层输送机重构计划：从巨型FB → 单层FB×4实例化

## 问题分析

### 当前问题
- `FB_四层输送机_V4.0.0.st` 是一个巨型功能块，内部使用 `ARRAY[1..4]` 处理4层数据
- 输入接口有大量数组（如 `i_Lx_分料前感应器: ARRAY[1..4] OF BOOL`），共61个输入、32个输出
- 不符合图片中展示的"每层独立FB实例化"模式
- FBD使用说明文档格式与图片中的样式不一致

### 目标架构（参考用户提供的图片）

```
主控程序 (或 FB_四层输送机管理)
├── fbL1_输送机 : FB_单层输送机  (第1层)  ← 实例1, 层号=1
├── fbL2_输送机 : FB_单层输送机  (第2层)  ← 实例2, 层号=2
├── fbL3_输送机 : FB_单层输送机  (第3层)  ← 实例3, 层号=3
└── fbL4_输送机 : FB_单层输送机  (第4层)  ← 实例4, 层号=4
```

## 重构方案

### 方案选择：保留 FB_四层输送机 作为容器 + 新建 FB_单层输送机

**理由**：
- 主控程序已经够复杂（IO映射+协调），不宜再增加4个直接实例化
- 四层之间可能有协调逻辑（如按顺序放料），需要一个管理层
- 符合"高内聚低耦合"原则：单层FB只管一层，四层FB只管协调

### 文件变更清单

| 操作 | 文件 | 说明 |
|------|------|------|
| **新建** | `四层输送机/FB_单层输送机_V4.0.0.st` | 单层纯逻辑功能块（核心） |
| **重写** | `四层输送机/FB_四层输送机_V4.0.0.st` | 改为容器FB，内部实例化4个单层FB |
| **重写** | `四层输送机/FBD使用说明.md` | 采用图片格式的FBD针脚图 |
| **修改** | `主控/主控程序_V4.0.0.st` | 适配新的四层输送机接口变化 |
| **修改** | `PLC变量定义文档_VAR-DJ-2026-005-V4.0.0.md` | 同步更新 |

## 详细实施步骤

### Step 1: 创建 FB_单层输送机 — 单层纯逻辑功能块

**文件**: `02_PLC程序/通用ST程序及变量表/四层输送机/FB_单层输送机_V4.0.0.st`

**接口设计**（单层，无ARRAY）：

```structuredtext
FUNCTION_BLOCK FB_单层输送机
VAR_INPUT
    (* === 系统控制 === *)
    i_b使能           : BOOL;
    i_b自动模式         : BOOL;
    i_b手动模式         : BOOL;
    i_b启动             : BOOL;
    i_b停止             : BOOL;
    i_b复位             : BOOL;
    
    (* === 手动操作(本层6个) === *)
    i_b阻挡下降         : BOOL;    // 手动-阻挡下降
    i_b阻挡上升         : BOOL;    // 手动-阻挡上升
    i_b分料推出         : BOOL;    // 手动-分料推出
    i_b分料复位         : BOOL;    // 手动-分料复位
    i_b输送正转         : BOOL;    // 手动-正转
    i_b输送反转         : BOOL;    // 手动-反转
    
    (* === 工艺参数 === *)
    i_r输送速度         : REAL;
    i_i分料时间         : INT;
    i_i阻挡等待时间      : INT;
    
    (* === 传感器输入(本层7个) === *)
    i_分料前感应器       : BOOL;    // 材料到位入口检测
    i_到位感应器1        : BOOL;    // 到位检测A
    i_到位感应器2        : BOOL;    // 到位检测B(冗余)
    i_阻挡气缸上位       : BOOL;    // 阻挡释放位置
    i_阻挡气缸下位       : BOOL;    // 阻挡阻挡位置
    i_分料气缸上位       : BOOL;    // 分料收回位置
    i_分料气缸下位       : BOOL;    // 分料推出位置
    
    (* === 上游/下游信号 === *)
    i_b上游允许放料      : BOOL;    // 取放料机构已取走，允许继续放料
END_VAR

VAR_OUTPUT
    (* === 执行器请求(本层5个) === *)
    o_阻挡电磁阀         : BOOL;    // → 主控 → Y地址
    o_分料电磁阀         : BOOL;    // → 主控 → Y地址
    o_输送带正转         : BOOL;    // → 主控 → Y地址
    o_输送带慢速         : BOOL;    // → 主控 → Y地址
    o_输送带反转         : BOOL;    // → 主控 → Y地址
    
    (* === 下游信号 === *)
    o_b放料完成          : BOOL;    // → 主控 → 取放料机构
    
    (* === 状态/HMI === *)
    o_b运行中             : BOOL;
    o_b故障               : BOOL;
    o_i当前状态           : INT;     // 0~8 + 99故障
    
    (* === 报警 === *)
    o_i报警代码           : INT;     // 0~7
END_VAR
```

**内部逻辑**：从现有 `FB_四层输送机` 中提取单层的状态机逻辑（9步自动模式 + 手动控制 + 初始化 + 7种报警），去掉所有 `[iLayerIdx]` 数组索引。

### Step 2: 重写 FB_四层输送机 — 改为容器FB

**文件**: `02_PLC程序/通用ST程序及变量表/四层输送机/FB_四层输送机_V4.0.0.st`

**新职责**：
- 内部声明4个 `FB_单层输送机` 实例
- 将输入的数组/分散信号分发到各实例
- 收集各实例输出，汇合成数组/汇总信号
- 可选的四层协调逻辑（如顺序放料控制）

```structuredtext
FUNCTION_BLOCK FB_四层输送机
VAR_INPUT
    (* 系统控制 - 直接透传给各层 *)
    i_b使能               : BOOL;
    i_b自动模式           : BOOL;
    i_b手动模式           : BOOL;
    i_b启动               : BOOL;
    i_b停止               : BOOL;
    i_b复位               : BOOL;
    
    (* 参数 - 直接透传 *)
    i_r输送速度           : REAL;
    i_i分料时间           : INT;
    i_i阻挡等待时间        : INT;
    
    (* 手动操作 - 4层 × 6 = 24个 *)
    i_b阻挡下降           : ARRAY[1..4] OF BOOL;
    i_b阻挡上升           : ARRAY[1..4] OF BOOL;
    i_b分料推出           : ARRAY[1..4] OF BOOL;
    i_b分料复位           : ARRAY[1..4] OF BOOL;
    i_b输送正转           : ARRAY[1..4] OF BOOL;
    i_b输送反转           : ARRAY[1..4] OF BOOL;
    
    (* 传感器 - 4层 × 7 = 28个 *)
    i_分料前感应器         : ARRAY[1..4] OF BOOL;
    i_到位感应器1          : ARRAY[1..4] OF BOOL;
    i_到位感应器2          : ARRAY[1..4] OF BOOL;
    i_阻挡气缸上位         : ARRAY[1..4] OF BOOL;
    i_阻挡气缸下位         : ARRAY[1..4] OF BOOL;
    i_分料气缸上位         : ARRAY[1..4] OF BOOL;
    i_分料气缸下位         : ARRAY[1..4] OF BOOL;
    
    (* 下游反馈 - 4层取放料完成信号 *)
    i_b取放料完成          : ARRAY[1..4] OF BOOL;  // 取放料机构已取走该层物料
END_VAR

VAR_OUTPUT
    (* 执行器 - 4层 × 5 = 20个 *)
    o_阻挡电磁阀           : ARRAY[1..4] OF BOOL;
    o_分料电磁阀           : ARRAY[1..4] OF BOOL;
    o_输送带正转           : ARRAY[1..4] OF BOOL;
    o_输送带慢速           : ARRAY[1..4] OF BOOL;
    o_输送带反转           : ARRAY[1..4] OF BOOL;
    
    (* 下游信号 *)
    o_b放料完成            : ARRAY[1..4] OF BOOL;  // 给取放料机构
    
    (* 汇总状态 *)
    o_b运行中              : BOOL;    // 任一层运行中
    o_b故障                : BOOL;    // 任一层故障
    o_i当前状态            : INT;     // 主要状态码
    o_Lx当前步序           : ARRAY[1..4] OF INT;
    
    (* 汇总报警 *)
    o_i本站报警代码        : INT;     // 最高优先级报警
END_VAR

VAR
    (* 4个单层实例 *)
    fbLayer1 : FB_单层输送机;
    fbLayer2 : FB_单层输送机;
    fbLayer3 : FB_单层输送机;
    fbLayer4 : FB_单层输送机;
END_VAR
```

**主体逻辑**：调用4个实例，做输入分发和输出收集。

| 风险 | 影响 | 应对措施 |
