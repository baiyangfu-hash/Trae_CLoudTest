# 变量命名与报警系统修复计划

## 1. 问题分析

### 1.1 变量命名问题
- **功能块命名**：当前FB块命名不符合规范，如`FB_四层输送机`、`FB_取放料机构`等，应使用英文功能描述
- **变量命名**：部分变量命名不符合规范，如`bLx_分料前感应器`、`bLx_到位感应器1`等，缺少类型前缀
- **输入输出变量**：部分变量未遵循`i_变量类型_变量描述`和`q_变量类型_变量描述`的规范

### 1.2 报警系统问题
- **多报警处理**：当前系统在多个报警一起出现时，只显示最高优先级的报警，无法同时显示所有报警
- **HMI触发条件**：HMI显示报警事件的触发条件是bool量，需要确保每个报警都有对应的bool量触发
- **报警查找**：当多个报警一起出现时，需要提供一种机制来查看所有当前活跃的报警

## 2. 修复方案

### 2.1 变量命名修复
- **功能块命名**：将FB块命名改为符合规范的格式，如`FB_Conveyor4Layer`、`FB_PickPlace`等
- **变量命名**：修复变量命名，确保遵循`变量类型_变量描述`的规范
- **输入输出变量**：确保输入变量使用`i_变量类型_变量描述`格式，输出变量使用`q_变量类型_变量描述`格式

### 2.2 报警系统完善
- **多报警显示**：修改公共报警功能块，增加一个报警状态数组，记录每个工站的当前报警状态
- **HMI触发条件**：为每个报警代码添加对应的bool量输出，用于HMI显示触发
- **报警查找机制**：在公共报警功能块中增加报警状态查询功能，允许HMI查询所有当前活跃的报警

## 3. 实施步骤

### 3.1 变量命名修复

   - 修复输入变量命名，如将`bLx_分料前感应器`改为`i_bLx_分料前感应器`
   - 修复输出变量命名，如将`bLx_阻挡电磁阀`改为`q_bLx_阻挡电磁阀`
   - 确保所有变量命名符合规范

### 3.2 报警系统完善
1. **修改公共报警功能块**：
   - 增加报警状态数组，记录每个工站的当前报警状态

2. **修改主控程序**：
   - 更新功能块实例化名称
   - 更新变量引用
   - 确保报警信号正确传递

3. **HMI集成**：
   - 确保HMI能够接收和显示所有报警
   - 实现报警列表显示功能
   - 确保报警触发条件正确

## 4. 技术要点

### 4.1 变量命名规范
- **功能块命名**：`FB_英文功能描述`
- **输入变量**：`i_变量类型_变量描述`
- **输出变量**：`q_变量类型_变量描述`
- **内部变量**：`变量类型_变量描述`
- **变量类型前缀**：b(BOOL)、i(INT)、r(REAL)、s(STRING)等

### 4.2 报警系统设计
- **多报警处理**：使用数组记录每个工站的报警状态
- **HMI触发**：为每个报警代码添加对应的bool量输出
- **报警优先级**：保持现有的优先级机制，同时支持显示所有报警
- **报警队列**：保持现有的MES报警队列功能
