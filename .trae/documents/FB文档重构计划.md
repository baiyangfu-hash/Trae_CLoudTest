# FB文档重构计划：拆分FBD文档为规范化文档体系

## 📋 任务概述

**目标**: 将3个FB的FBD使用说明文档拆分为符合规范的独立文档（详细设计说明书、变更记录、接口文档、使用说明），删除原有FBD文档，并更新所有项目相关文档确保无重复且一致。

**涉及FB**:
1. ✅ FB_1004_GlueMachineFeeder_BufferFraming (打胶机送料机构)
2. ✅ FB_1002_SingleLayerConveyor_BufferFraming / FB_1001_Conveyor4Layer_BufferFraming (四层输送机)
3. ✅ FB_1003_PickPlace_BufferFraming (取放料机构)

---

## 🎯 核心原则

### ⚠️ 重要：变更记录即变更台帐（强制性规范）

**用户明确要求**: 变更记录不是简单的版本历史，而是**项目的正式变更台帐**，必须严格遵循项目管理规范。

**核心要求**:
1. ✅ **每次FB版本更新必须记录**：
   - 🔧 Bug修复（DEF - 缺陷修复）
   - ✨ 功能增加（REQ - 需求变更 / OPT - 优化改进）
   - ⚙️ 配置调整（CFG - 配置调整）
   - 🚨 紧急变更（EMRG - 紧急变更）

2. ✅ **变更记录格式**：遵循 `00_项目管理/04_变更管理` 规范
   - 参考模板：`CHG-DOCU-2026-001.md`
   - 包含：变更编号、技术领域、业务性质、影响范围、变更原因、变更内容、实施记录、验证结果

3. ✅ **变更记录与版本号联动**：
   - FB源代码(.st)修改 → 变更记录新增条目 → 文档版本号升级
   - 4个文档(DSN/IFC/UM/CHG)版本号必须同步

### 文档拆分策略

| 原FBD文档章节 | 拆分目标文档 | 说明 |
|--------------|-------------|------|
| 功能块概述、架构定位、设计原则 | **详细设计说明书 (DSN)** | 设计层面的技术描述 |
| 版本变更记录、版本历史 | **变更记录 (CHG)** | 版本管理和变更追踪 |
| 接口针脚详细说明(输入/输出) | **接口文档 (IFC)** | 完整的IO接口定义 |
| FBD连接示例、ST调用示例 | **使用说明 (UM)** | 调用方法和使用指南 |
| 自动模式状态机详解 | **详细设计说明书 (DSN)** | 核心逻辑实现细节 |
| 使用注意事项、调优建议、调试步骤 | **使用说明 (UM)** | 实操指导 |
| FBD图形化表示 | ❌ 删除 | ASCII art图示过于冗长，改用文字描述 |

### 文档命名规范

按照项目规范，采用以下命名格式：
- `详细设计说明书_DSN-{FB简称}-V{版本}.md`
- `接口文档_IFC-{FB简称}-V{版本}.md`
- `使用说明_UM-{FB简称}-V{版本}.md`
- `变更记录_CHG-{FB简称}-V{版本}.md`

#### 1. 变更触发条件与分类

| 变更类型 | 触发场景 | 业务性质 | 示例 |
|---------|---------|----------|------|

| ✨ **功能增加** | 客户新需求、工艺调整 | REQ (需求变更) / OPT (优化) | 新增传感器接口、增加步骤 |

| 🚨 **紧急变更** | 安全相关、生产中断 | EMRG (紧急变更) | 急停逻辑修改、安全联锁 |

```

         │
         ▼

│  3. 同步更新文档  │ ← 同时更新 DSN/IFC/UM 文档
```
