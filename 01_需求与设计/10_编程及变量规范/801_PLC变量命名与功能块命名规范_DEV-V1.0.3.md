## 1. 总则

### 1.1 目的

为了统一PLC变量命名规范，提高代码可读性和可维护性，特制定本规则。

### 1.2 适用范围

本规范适用于所有PLC编程系统，包括Autoshop、Work3等，确保跨系统的命名一致性。

### 1.3 核心原则

1. **可读性**：变量名应清晰表达其用途和含义
2. **一致性**：在整个项目中使用一致的命名风格
3. **简洁性**：在保证可读性的前提下，变量名应尽可能简洁
4. **避免歧义**：变量名应避免使用容易引起歧义的词汇

## 2. 变量命名规范

### 2.1 变量命名规则

#### 2.1.1 FB，FC等功能块输入变量

- **命名格式**：`i_变量类型_变量描述`
- **示例**：`i_bStartButton`（启动按钮信号）

#### 2.1.2 FB，FC输出变量

- **命名格式**：`q_变量类型_变量描述`
- **示例**：`q_bMotorRun`（电机运行输出）

#### 2.1.3 FB，FC等功能块输入输出变量

- **命名格式**：`iq_变量类型_变量描述`
- **示例**：`iq_bMotorRun`（电机运行双向信号）

#### 2.1.4 内部变量

- **命名格式**：`变量类型_变量描述`
- **示例**：`bRunning`（运行状态内部变量）

#### 2.1.5 全局变量

- **命名格式**：`变量类型_变量描述`
- **示例**：`bSysAutoMode`（系统自动模式全局变量）

#### 2.1.6 上升沿/下降沿检测变量

- **命名格式**：`LDP_变量类型_变量描述`（上升沿）或 `LDF_变量类型_变量描述`（下降沿）
- **示例**：`LDP_bStart`（启动信号上升沿）或 `LDF_bStop`（停止信号下降沿）

#### 2.1.7 硬件IO变量

- **命名格式**：`In_变量类型_变量描述`（输入）或 `Out_变量类型_变量描述`（输出）
- **示例**：`In_bStartButton`（启动按钮硬件输入）或 `Out_bMotorRun`（电机运行硬件输出）

### 2.2 变量类型前缀

| 变量类型 | 前缀 | 示例                                    |
| ---- | -- | ------------------------------------- |
| 布尔型  | b  | `bStart`, `bStop`, `bAlarm`           |
| 整数型  | i  | `iSpeed`, `iCount`, `iTemperature`    |
| 长整数型 | li | `liSpeed`, `liCount`, `liTemperature` |
| 实数型  | r  | `rPressure`, `rFlow`, `rLevel`        |
| 字符串型 | s  | `sProductName`, `sRecipeID`           |
| 浮点数型 | f  | `fPressure`, `fFlow`, `fLevel`        |
| 数组   | a  | `aMotorSpeed[3]`, `aSensorData[10]`   |

### 2.3 常量命名

- 常量名应使用全大写字母，单词之间用下划线分隔
- 示例：`MAX_SPEED`, `MIN_TEMPERATURE`, `DEFAULT_PRESSURE`

## 3. 功能块命名规范

### 3.1 功能块命名格式

- **命名格式**：`FB_编号_英文功能描述_设备对象`
- **示例**：`FB_1001_ConveyorControl_BeltConveyor`（皮带输送机启停控制）

### 3.2 输入输出变量前缀规范

#### 3.2.1 输入变量前缀 (VAR_INPUT)

| 前缀格式 | 适用范围 | 示例 | 说明 |
|----------|----------|------|------|
| `i_` + 变量类型 + `_` + 功能描述 | 标准输入变量 | `i_bStartButton`, `i_iSpeedSetpoint` | FB/FC的标准输入参数 |
| `i_b` + `Lx_` + 功能描述 | HMI手动操作缓存变量（从M区读取后传给FB） | `i_bLx_阻挡下降`, `i_bLx_输送正转` | Lx = Logic auxiliary（逻辑辅助），用于区分直接DI映射的输入变量与经主程序缓存后的HMI操作变量 |

> **HMI辅助缓存变量特殊说明**：当主程序从M区（HMI辅助控制区）读取手动操作信号后，需缓存为中间变量再传给功能块时，使用 `i_bLx_` 前缀。其中 `Lx` 表示"联锁/逻辑辅助"(Logic auxiliary)，用于与直接DI映射的输入变量做视觉区分。

#### 3.2.2 输出变量前缀 (VAR_OUTPUT)

| 前缀格式 | 适用范围 | 示例 | 说明 |
|----------|----------|------|------|
| `q_` + 变量类型 + `_` + 功能描述 | 标准输出变量 | `q_bMotorRun`, `q_rCurrentPosition` | FB/FC的标准输出参数 |

### 3.3 功能块编号规则

#### 3.3.1 基础控制模块

- **编号区间**：0100-1999
- **说明**：按工作流内功能优先级编排（如先启停、再调节、后报警）
- **唯一性**：同一工作流内编号唯一，跨工作流可重复（通过工作流编号区分）

#### 3.3.2 主控模块

- **编号区间**：2000-2999
- **说明**：主控层 FB（总调度 / 全局监控 / 模式切换）
- **唯一性**：同一项目内编号唯一，跨项目可重复（通过项目号区分）

### 3.4 功能块命名示例

#### 工作流功能块示例：

- `FB_1001_ConveyorControl_BeltConveyor`（皮带输送机启停控制）
- `FB_1002_SpeedRegulation_VFDMainConveyor`（主输送线变频无级调速）
- `FB_1001_GraspControl_EndEffectorGripper`（机械臂末端夹爪开合控制）
- `FB_1001_DataAcquisition_SlaveDevice`（从站设备状态采集上传）
- `FB_1002_CommandIssuance_Actuator`（执行机构运行参数指令下发）

#### 主控层功能块示例：

- `FB_2000_MainDispatch_AllEquipment`（全设备工作流调度+模式切换）
- `FB_2001_MainStatusMonitor_AllStation`（全局状态监控+故障汇总）
- `FB_2002_MainModeSwitch_OperationMode`（手动/自动/调试模式切换）

## 4. 注释规范

### 4.1 变量注释

- 所有变量都应添加注释，说明其用途和含义
- 注释应简洁明了，避免冗余
- 对于复杂变量，应详细说明其使用方法和注意事项

### 4.2 功能块注释

- 功能块应包含头部注释，说明功能、版本、输入输出变量等信息
- 注释应包含变更记录，便于追踪历史修改

## 5. 实施要求

1. 所有开发人员必须严格遵守本命名规范
2. 项目负责人应在项目启动时向团队成员传达本规范
3. 在代码审查过程中，应检查变量命名是否符合本规范
4. 对于不符合规范的变量名，应及时进行修改

## 附录 D：IDE 工具兼容性注意事项

### D.1 中文标识符支持现状

| IDE/工具 | 中文标识符支持 | 备注 |
|----------|---------------|------|
| 博图(TIA Portal) | ✅ 完全支持 | 西门子官方IDE |
| Autoshop | ✅ 完全支持 | 欧姆龙官方IDE |
| Work3 | ✅ 完全支持 | 三菱官方IDE |
| Codesys | ✅ 完全支持 | IEC 61131-3 标准环境 |
| Dynamic Siemens Language Support(VS Code插件) | ⚠️ 部分支持 | FUNCTION_BLOCK 类型中中文标识符可能触发 TC001 误报 |
| 其他第三方LSP/编辑器 | ❓ 视具体实现而定 | |

### D.2 推荐做法

1. **变量名和功能块名**：优先使用英文或拼音标识符（如 `bStartButton`, `FB_1001_Conveyor4Layer_BufferFraming`）
2. **注释和文档**：鼓励使用中文，提高可读性
3. **项目决策**：若团队统一使用博图等官方IDE开发，可放宽此限制；若使用VS Code+第三方LSP开发，建议遵循英文标识符原则
4. **已有项目**：已使用中文标识符的项目无需强制重构，但新项目建议采用英文标识符

### D.3 LSP误报处理指南

当使用 Dynamic Siemens Language Support 等 VS Code 插件遇到 TC001 "undeclared symbol" 误报时：

1. 确认变量已在 VAR 区域正确声明
2. 检查拼写是否完全一致（含全角/半角字符）
3. 若确认为中文标识符导致的误报，可安全忽略该错误
4. 最终编译以实际PLC编程软件（博图/Autoshop/Work3）为准
