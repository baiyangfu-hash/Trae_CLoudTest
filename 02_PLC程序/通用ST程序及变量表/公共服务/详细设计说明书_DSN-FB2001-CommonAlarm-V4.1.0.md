## 3. 功能描述

### 3.1 核心功能

FB_2001_CommonAlarm_AllStation 是一个**纯逻辑功能块**，作为边框缓存机的**全局报警管理中心**：

- **三站报警汇聚**: 接收四层输送机(1~7段)、取放料机构(101~108段)、打胶机送料机构(201~205段)三个工站的报警代码
- **全局报警字计算**: 将所有有效报警码进行OR组合，输出给D400供HMI报警灯使用
- **最高优先级选取**: 从多工站同时报警时选取数值最小的报警码作为当前显示报警
- **MES报警队列管理**: 维护最近10条去重报警记录(D406~D425)，供MES系统读取上传
- **新报警脉冲标志**: 检测到新报警时输出单周期脉冲，触发主控进行MES数据上传
- **多报警状态输出**: 分别输出各工站报警激活BOOL量，支持HMI分站显示
- **完全解耦设计**: 无任何X/Y/M/D物理地址引用，所有数据通过逻辑变量交互

## 5. 核心算法设计

### 5.1 全局报警字计算算法

```
全局报警字计算 (每个扫描周期执行):

o_w全局报警字 := 
    WORD_OF_INT(i_i输送机报警) OR 
    WORD_OF_INT(i_i取放料报警) OR 
    WORD_OF_INT(i_i送料机构报警);

含义:
  - 将三个工站的报警码分别转换为WORD类型
  - 进行按位OR运算, 合并为一个16位报警字
  - 非零值表示至少有一个工站在报警
  - 写入D400供HMI判断是否有报警(用于驱动报警灯)
```

### 5.2 最高优先级选取算法

```
当前报警代码选取 (MIN优先级原则):

IF (i_i输送机报警 > 0 AND i_i输送机报警 < 当前最小值) THEN
    当前最小值 := i_i输送机报警;
END_IF;

IF (i_i取放料报警 > 0 AND i_i取放料报警 < 当前最小值) THEN
    当前最小值 := i_i取放料报警;
END_IF;

IF (i_i送料机构报警 > 0 AND i_i送料机构报警 < 当前最小值) THEN
    当前最小值 := i_i送料机构报警;
END_IF;

o_i当前报警代码 := 当前最小值;  (* 输出到D402 *)

优先级规则:
  - 数值越小 = 优先级越高
  - 0表示无报警(不参与比较)
  - 多报警同时存在时, 只显示最高优先级的一个
  - 示例: 同时有ALM_1(输送机) + ALM_102(取放料) → 显示ALM_1
```

### 5.3 MES报警队列管理算法

```
队列操作流程:

检测到新报警:
  1. 判断是否为新报警:
     ├─ 条件A: 上一周期o_w全局报警字=0, 本周期≠0 (从无到有)
     ├─ 条件B: 本周期o_w全局报警字 ≠ 上一周期 (内容变化)
     └─ 条件C: 本周期o_i当前报警代码 ≠ 上一周期 (优先级切换)
  
  2. 若为新报警:
     a. 检查去重: 新报警码是否已存在于队列中?
        ├─ 已存在 → 不重复入队, 仅更新计数器
        └─ 不存在 → 入队操作:
           - 队列元素后移一位([0]←[1], [1]←[2], ..., [8]←[9])
           - 新报警码写入队列[0]位置(最新)
           - 若队列满(已有10条), 最旧记录[9]被覆盖丢弃
  
  3. 更新统计:
     o_iMES报警计数 += 1
     o_b新报警标志 := TRUE (仅当周期有效, 下一周期自动FALSE)

复位操作(i_b复位=TRUE):
  - o_w全局报警字 := 0
  - o_i当前报警代码 := 0
  - o_b任何报警激活 := FALSE
  - o_b新报警标志 := FALSE
  - o_iMES报警计数 := 0
  - o_iMES报警队列[0..9] 全部清零
  - 内部指针/计数器归零
```

## 6. D区地址分配表

| D地址 | 位宽 | 来源变量 | 说明 |
|-------|------|----------|------|
| **D400** | WORD (16bit) | o_w全局报警字 | 各报警码的OR组合，非零即有报警 |
| **D402** | INT (16bit) | o_i当前报警代码 | 当前最高优先级报警码数值 |
| **D404** | INT (16bit) | o_iMES报警计数 | 自上次复位以来的累计报警次数 |
| **D406~D425** | 10 x INT (200bit) | o_iMES报警队列[0..9] | MES报警队列(索引0最新, 索引9最旧) |

## 附录: 特殊设计决策记录

| 决策ID | 决策内容 | 原因 | 替代方案(未采用) |
|:------:|:---------|:-----|:---------------|
| D001 | 采用独立FB而非分散在各工站中 | 职责单一, 易于维护和扩展 | 分散在各工站(增加耦合度, 逻辑复杂) |
| D002 | 使用MIN优先级原则 | 数值越小越紧急, 符合直觉 | MAX原则或其他自定义规则(不符合行业惯例) |
| D003 | MES队列采用10条长度 | 平衡存储空间和信息完整性 | 过长(浪费D区资源)/过短(丢失历史信息) |
| D004 | 使用去重策略 | 避免同一报警反复入队占满队列 | 不去重(可能导致队列快速填满同一种报警) |
| D005 | 新报警脉冲标志 | 让主控只需检测上升沿即可触发上传 | 持续标志(需要主控自己处理边沿, 增加复杂度) |
