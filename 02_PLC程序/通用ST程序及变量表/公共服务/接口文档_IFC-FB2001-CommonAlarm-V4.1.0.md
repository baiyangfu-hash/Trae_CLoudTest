## 3. 输入接口定义 (VAR_INPUT)

### 3.1 三站报警代码输入 (3个INT)

#### 3.1.1 i_i输送机报警

| 属性 | 值 |
|------|-----|
| **针脚名称** | `i_i输送机报警` |
| **数据类型** | INT |
| **功能描述** | 四层输送机工站的当前报警代码(0=无报警 / 1~7=有报警) |
| **来源** | FB_1001_Conveyor4Layer_BufferFraming.o_i本站报警代码 |
| **有效值域** | 0, 1, 2, 3, 4, 5, 6, 7 |
| **优先级范围** | 高优先级段(全系统最小编号) |

#### 3.1.2 i_i取放料报警

| 属性 | 值 |
|------|-----|
| **针脚名称** | `i_i取放料报警` |
| **数据类型** | INT |
| **功能描述** | 取放料机构工站的当前报警代码(0=无报警 / 101~108=有报警) |
| **来源** | FB_1003_PickPlace_BufferFraming.o_i本站报警代码 |
| **有效值域** | 0, 101, 102, 103, 104, 105, 106, 107, 108 |
| **优先级范围** | 中优先级段 |

#### 3.1.3 i_i送料机构报警

| 属性 | 值 |
|------|-----|
| **针脚名称** | `i_i送料机构报警` |
| **数据类型** | INT |
| **功能描述** | 打胶机送料机构工站的当前报警代码(0=无报警 / 201~205=有报警) |
| **来源** | FB_1004_GlueMachineFeeder_BufferFraming.o_i本站报警代码 |
| **有效值域** | 0, 201, 202, 203, 204, 205 |
| **优先级范围** | 低优先级段(最大编号范围) |

### 3.2 操作按钮 (1个BOOL)

#### 3.2.1 i_b复位

| 属性 | 值 |
|------|-----|
| **针脚名称** | `i_b复位` |
| **数据类型** | BOOL |
| **功能描述** | HMI复位按钮信号，用于清除所有报警状态和MES队列 |
| **触发方式** | **电平有效**（按下执行复位） |
| **行为描述** | 清除全部输出归零、清空MES队列、重置内部计数器和指针 |
| **来源** | HMI M区变量(M4) |

## 4. 输出接口定义 (VAR_OUTPUT)

### 4.1 核心报警输出组 (3个)

#### 4.1.1 o_w全局报警字

| 属性 | 值 |
|------|-----|
| **针脚名称** | `o_w全局报警字` |
| **数据类型** | WORD (16bit) |
| **功能描述** | 三个工站报警码的OR组合结果，非零表示至少有一个工站在报警 |
| **写入目标** | 主控 → **D400** (HMI报警灯指示) |
| **计算公式** | `WORD_OF_INT(i_i输送机报警) OR WORD_OF_INT(i_i取放料报警) OR WORD_OF_INT(i_i送料机构报警)` |
| **典型用途** | HMI判断是否有活跃报警，驱动报警灯闪烁 |

#### 4.1.2 o_i当前报警代码

| 属性 | 值 |
|------|-----|
| **针脚名称** | `o_i当前报警代码` |
| **数据类型** | INT |
| **功能描述** | 当前最高优先级的报警码数值(多报警时取MIN) |
| **写入目标** | 主控 → **D402** (HMI文本显示) |
| **选取规则** | 数值越小优先级越高；0表示无报警 |
| **典型用途** | HMI显示当前主要报警的名称和描述 |

#### 4.1.3 o_b任何报警激活

| 属性 | 值 |
|------|-----|
| **针脚名称** | `o_b任何报警激活` |
| **数据类型** | BOOL |
| **功能描述** | 全局报警激活标志，任一工站有报警时为TRUE |
| **写入目标** | 主控 → **M200** (全局运行互锁) |
| **逻辑关系** | `o_w全局报警字 <> 0` |
| **典型用途** | 禁止自动运行模式，防止在故障状态下继续生产 |

### 4.2 MES数据接口组 (3个)

#### 4.2.1 o_iMES报警计数

| 属性 | 值 |
|------|-----|
| **针脚名称** | `o_iMES报警计数` |
| **数据类型** | INT |
| **功能描述** | 自上次复位操作以来的累计报警次数(每次新报警+1) |
| **写入目标** | 主控 → **D404** |
| **清零条件** | 执行复位操作(`i_b复位=TRUE`)后归零 |

#### 4.2.2 o_iMES报警队列 [ARRAY OF INT, 0..9]

| 属性 | 值 |
|------|-----|
| **针脚名称** | `o_iMES报警队列[0..9]` |
| **数据类型** | ARRAY[0..9] OF INT (每个元素16bit) |
| **功能描述** | 最近10条去重报警记录队列(索引0最新, 索引9最旧) |
| **写入目标** | 主控 → **D406 ~ D425** (10 x INT = 20个D地址) |
| **存储格式** | 三菱PLC的INT为16bit小端存储, 每个INT占2个连续D地址 |
| **入队规则** | 新报警触发时入队到[0], 已有元素后移; 队列满时覆盖[9]; 同一码不重复入队 |
| **清零条件** | 复位操作后全部清零 |

**队列索引与D地址对应表**:

| 索引 | D地址(起始) | D地址(结束) | 含义 |
|:----:|:----------:|:-----------:|:-----|
| [0] | D406 | D407 | 最新一条报警码 |
| [1] | D408 | D409 | 第2新报警码 |
| [2] | D410 | D411 | 第3新报警码 |
| [3] | D412 | D413 | 第4新报警码 |
| [4] | D414 | D415 | 第5新报警码 |
| [5] | D416 | D417 | 第6新报警码 |
| [6] | D418 | D419 | 第7新报警码 |
| [7] | D420 | D421 | 第8新报警码 |
| [8] | D422 | D423 | 第9新报警码 |
| [9] | D424 | D425 | 最旧一条报警码 |

#### 4.2.3 o_b新报警标志

| 属性 | 值 |
|------|-----|
| **针脚名称** | `o_b新报警标志` |
| **数据类型** | BOOL |
| **功能描述** | 检测到新报警时的单周期脉冲标志(仅当周期TRUE, 下一周期自动FALSE) |
| **用途** | 触发主控程序进行MES数据上传(D406~D425内容) |
| **触发条件** | 满足以下任一条件即产生脉冲:<br>① 从无报警变为有报警<br>② 报警内容发生变化(o_w全局报警字改变)<br>③ 最高优先级切换(o_i当前报警代码改变) |

### 4.3 分站状态输出组 (4个)

#### 4.3.1 o_i工站报警状态 [ARRAY OF INT, 1..3]

| 属性 | 值 |
|------|-----|
| **针脚名称** | `o_i工站报警状态[1..3]` |
| **数据类型** | ARRAY[1..3] OF INT |
| **功能描述** | 各工站当前的原始报警代码(未经优先级选取) |
| **数组索引约定** | `[1]=四层输送机`, `[2]=取放料机构`, `[3]=打胶机送料机构` |
| **用途** | 主控读取后可分别显示各工站的独立报警状态 |

#### 4.3.2 o_b输送机报警激活 / o_b取放料报警激活 / o_b送料机构报警激活

| 属性 | 值 |
|------|-----|
| **针脚名称** | `o_b输送机报警激活` / `o_b取放料报警激活` / `o_b送料机构报警激活` |
| **数据类型** | BOOL (x3) |
| **功能描述** | 各工站独立的报警激活BOOL量 |
| **用途** | 支持HMI分站显示各工站是否有报警(如分色指示灯) |

  ✓ 完全解耦设计(无物理地址依赖)

  D400 = 全局报警字 (WORD)
  D402 = 当前报警码 (INT)
  D404 = MES累计计数 (INT)
