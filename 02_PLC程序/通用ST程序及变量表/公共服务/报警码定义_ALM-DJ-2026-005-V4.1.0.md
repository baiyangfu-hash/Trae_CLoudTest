## 1. 文档概述

### 1.1 目的

本文档定义边框缓存机(DJ-2026-005)全部报警码的编码规则、归属工站、触发条件及处理建议。所有报警码由三个工站功能块独立检测后，汇总至 **FB_2001_CommonAlarm_AllStation** 功能块进行统一管理。

### 1.2 报警码段分配原则

```
|------|------------------|------------------|------------------|
|  1~99   | 101~199          | 201~299          |
| 四层输送机  | 取放料机构         | 打胶机送料机构      |
| (高优先级)  | (中优先级)         | (低优先级)         |
|------|------------------|------------------|------------------|
```

**优先级规则**: 数值越小，优先级越高。当多工站同时报警时，FB_2001_CommonAlarm_AllStation选取数值最小的报警码作为当前显示报警。

### 1.3 数据流向

```
FB_1001_Conveyor4Layer    FB_1003_PickPlace        FB_1004_GlueMachineFeeder
  | o_i本站报警代码        | o_i本站报警代码         | o_i本站报警代码
  | (1~7)                | (101~108)              | (201~205)
  v                      v                       v
+-------------------------------------------------------+
|                  FB_2001_CommonAlarm_AllStation (纯逻辑块)                   |
|                                                       |
|  输入: i_i输送机报警 / i_i取放料报警 / i_i送料机构报警    |
|  输入: i_b复位                                        |
|                                                       |
|  输出: o_w全局报警字      --> 主控 --> D400 (HMI报警灯)  |
|  输出: o_i当前报警代码     --> 主控 --> D402 (HMI文本)   |
|  输出: o_b任何报警激活     --> 主控 --> M200 (全局互锁)  |
|  输出: o_iMES报警队列[10] --> 主控 --> D406~D425       |
|  输出: o_iMES报警计数     --> 主控 --> D404             |
|  输出: o_b新报警标志      --> 主控 (脉冲,触发MES上传)    |
+-------------------------------------------------------+
```

## 2. 四层输送机报警码 (1~7 段)

**归属功能块**: `FB_1001_Conveyor4Layer_BufferFraming`
**输出变量**: `o_i本站报警代码`
**优先级**: 高 (1~7 为全系统最小编号范围)

| 报警码 | 名称 | 触发条件 | 可能原因 | 处理建议 |
|--------|------|----------|----------|----------|
| **1** | 阻挡下降超时 | 阻挡气缸发出下降指令后，在设定时间内未收到下位传感器信号 | 1. 气源压力不足<br>2. 阻挡气缸卡死或机械干涉<br>3. 下位磁性开关故障/接线松动<br>4. 电磁阀线圈烧毁 | 1. 检查气源压力(≥0.5MPa)<br>2. 手动测试气缸动作是否顺畅<br>3. 检查下位传感器信号(X地址)<br>4. 检查电磁阀输出(Y地址) |
| **2** | 上升超时 | 阻挡气缸发出上升(释放)指令后，在设定时间内未收到上位传感器信号 | 1. 气源压力不足<br>2. 阻挡气缸复位弹簧失效<br>3. 上位磁性开关故障<br>4. 材料卡在阻挡位置阻碍上升 | 1. 检查气源压力<br>2. 确认阻挡位置无材料堆积<br>3. 检查上位传感器信号<br>4. 手动操作验证气缸复位 |
| **3** | 分料推出超时 | 分料气缸发出推出指令后，在设定时间内未收到下位(推出到位)信号 | 1. 分料气缸卡死<br>2. 推出路径有材料阻挡<br>3. 下位磁性开关故障<br>4. 电磁阀不动作 | 1. 确认分料区域无材料堆积<br>2. 手动测试分料气缸<br>3. 检查下位传感器<br>4. 检查电磁阀供电 |
| **4** | 复位超时 | 故障复位后，设备在规定时间内未能完成初始化回到安全状态 | 1. 存在未消除的硬件故障<br>2. 传感器信号异常导致初始化条件不满足<br>3. 多个气缸同时超时 | 1. 先排除各层具体故障<br>2. 检查所有传感器信号是否正常<br>3. 手动将所有气缸归位后再复位 |
| **5** | 输送超时 | 输送带启动后，在设定时间内材料未到达到位感应器位置 | 1. 输送带打滑或皮带松弛<br>2. 变频器故障或未启动<br>3. 上游未放料<br>4. 到位传感器故障 | 1. 检查变频器运行状态和频率<br>2. 检查皮带张紧度<br>3. 确认上游已放料<br>4. 检查到位感应器信号 |
| **6** | 传感器冲突 | 同一气缸的上位和下位传感器同时为TRUE | 1. 传感器接线短路<br>2. 两个磁性开关同时损坏<br>3. IO模块通道故障<br>4. 气缸在中位时两个开关都感应(安装距离过近) | 1. 检查传感器接线(有无短接)<br>2. 断电后用万用表测量传感器阻值<br>3. 更换可疑传感器<br>4. 调整传感器安装间距 |
| **7** | 变频器故障 | 变频器反馈故障信号(ALM) | 1. 变频器过流/过压/过热<br>2. 电机缺相或接地<br>3. 变频器参数错误<br>4. 通信丢失 | 1. 查看变频器面板错误代码<br>2. 检查电机绝缘和接线<br>3. 确认变频器参数设置<br>4. 重启变频器电源 |

## 4. 打胶机送料机构报警码 (201~205 段)

**归属功能块**: `FB_1004_GlueMachineFeeder_BufferFraming`
**输出变量**: `o_i本站报警代码`
**优先级**: 低 (201~205 为最大编号范围)

|--------|------|----------|----------|----------|

## 6. 报警处理流程

### 6.1 正常报警流程

```
工站检测故障 → 设置o_i本站报警代码(非零)
       ↓
FB_2001_CommonAlarm_AllStation接收三站报警输入
       ↓
计算 o_w全局报警字 (OR组合)
计算 o_i当前报警代码 (最小值)
置位 o_b任何报警激活 = TRUE
       ↓
判断是否为新报警?
  ├─ 是 → 入队MES队列(去重) + o_b新报警标志=TRUE(1周期脉冲)
  └─ 否 → 保持现有状态
       ↓
主控读取输出:
  - D400→HMI报警灯闪烁
  - D402→HMI显示报警文本
  - M200→禁止自动运行
  - o_b新报警标志↑→触发MES上传D406~D425
```

### 6.2 复位流程

```
操作员按下HMI复位按钮
       ↓
i_b复位 = TRUE (电平有效)
       ↓
FB_2001_CommonAlarm_AllStation执行复位:
  - o_w全局报警字 = 0
  - o_i当前报警代码 = 0
  - o_b任何报警激活 = FALSE
  - o_b新报警标志 = FALSE
  - o_iMES报警计数 = 0
  - MES队列[0..9] 全部清零
  - 内部指针/计数器归零
       ↓
主控同步清除:
  - D400 = 0 (HMI报警灯灭)
  - D402 = 0 (HMI文本清空)
  - M200 = FALSE (解除互锁)
  - D404 = 0 (计数归零)
  - D406~D425 = 0 (队列清空)
       ↓
各工站在下一周期检测到i_b复位信号后自行清除内部报警
→ 设备恢复正常, 可重新启动
```

### 6.3 新报警检测机制

新报警判定满足以下任一条件即产生脉冲:

1. **从无到有**: 上一周期三站均无报警(`o_w全局报警字=0`)，本周期至少一站有报警
2. **报警内容变化**: 本周期的 `o_w全局报警字` 与上一周期不同
3. **最高优先级切换**: 本周期的 `o_i当前报警代码` 与上一周期不同

**脉冲特性**: `o_b新报警标志` 仅在检测到新报警的当个扫描周期为TRUE，下一周期自动恢复FALSE。

## 7. MES数据接口规范

### 7.1 报警队列格式

| 索引 | D地址 | 含义 |
|------|-------|------|
| [0] | D406, D407 | 最新一条报警码(INT, 小端存储) |
| [1] | D408, D409 | 第2新报警码 |
| [2] | D410, D411 | 第3新报警码 |
| [3] | D412, D413 | 第4新报警码 |
| [4] | D414, D415 | 第5新报警码 |
| [5] | D416, D417 | 第6新报警码 |
| [6] | D418, D419 | 第7新报警码 |
| [7] | D420, D421 | 第8新报警码 |
| [8] | D422, D423 | 第9新报警码 |
| [9] | D424, D425 | 最旧一条报警码 |

**注意**: 三菱PLC的INT类型为16bit，占用2个连续D寄存器(小端模式：低字节在前)。每个INT报警码实际占2个D地址。

### 7.2 MES上传触发时机

```
主控每个扫描周期检测 o_b新报警标志
  ↓
检测到上升沿( FALSE → TRUE )
  ↓
立即执行:
  1. 读取 o_iMES报警队列[0..9]
  2. 读取 o_iMES报警计数
  3. 读取 o_i当前报警代码
  4. 组装MES报文并通过通信口发送
  ↓
上传完成后等待下一个 o_b新报警标志 脉冲
```

### 7.3 去重策略

- 同一报警码重复出现时不重复入队
- 不同报警码按时间顺序依次入队
- 队列满(10条)时覆盖最旧记录
- 复位操作清空整个队列
