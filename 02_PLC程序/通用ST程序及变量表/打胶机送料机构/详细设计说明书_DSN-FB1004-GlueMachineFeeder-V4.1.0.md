## 3. 功能描述

### 3.1 核心功能

FB_1004_GlueMachineFeeder_BufferFraming 是一个**纯逻辑功能块**，实现边框缓存机打胶机送料机构的完整控制逻辑：

- **6步自动状态机**: 完整的送料循环控制（空闲 → 等待放料完成 → 移动到取料位 → 等待允许 → 发送允许抓料 → 返回待机）
- **自动/手动双模式**: 支持自动循环运行和手动点动调试
- **初始化流程**: 上电或复位后自动执行伺服状态检查和初始化
- **内置报警机制**: 5种报警类型的检测、锁存与输出（201~205段）
- **安全区管理**: 与取放料机构的区域互锁逻辑，防止机械碰撞
- **完全解耦设计**: 无任何X/Y/M/D物理地址引用，所有数据通过逻辑变量交互

### 3.2 工艺流程概述

```
核心工艺: 接收取放料放料完成 → X2轴移动到取料位 → 等待打胶机允许 → 
         发送抓料许可 → 等待取料完成 → X2轴返回待机位 → 循环

自动循环特点:
┌─────────────────────────────────────────────────────────────┐
│  步骤0: 空闲/待机位置                                        │
│  ├── 等待启动触发                                           │
│  └── 收到启动后进入步骤1                                     │
│                                                             │
│  步骤1: 等待取放料放料完成                                   │
│  ├── 等待 i_取放料_放料完成 = TRUE                          │
│  ├── 同时检查安全区信号(必须开放)                            │
│  └── 放料完成后进入步骤2                                     │
│                                                             │
│  步骤2: 移动到打胶机取料位置 (X2轴前移)                      │
│  ├── s_bX2轴请求前移 := TRUE                                │
│  ├── 等待 s_bX2轴在取料位 = TRUE(主控反馈)                  │
│  └── 到达后进入步骤3                                         │
│                                                             │
│  步骤3: 等待打胶机允许送料                                   │
│  ├── 等待 i_打胶机允许送料 = TRUE                           │
│  ├── 启动通讯超时定时器(10s)                                │
│  └── 允许后进入步骤4                                         │
│                                                             │
│  步骤4: 发送允许抓料 + 等待取料完成                           │
│  ├── o_允许抓料 := TRUE (-> Y44)                            │
│  ├── 等待 i_打胶机取料完成 := TRUE                          │
│  └── 取料完成后进入步骤5                                     │
│                                                             │
│  步骤5: 返回待机位置 (X2轴后移)                              │
│  ├── s_bX2轴请求后移 := TRUE                                │
│  ├── 等待 s_bX2轴在待机位 := TRUE(主控反馈)                  │
│  └── 到位后回到步骤1, 继续下一轮循环                         │
│                                                             │
│  循环特点: 步骤5完成后自动回到步骤1, 无需重新启动             │
└─────────────────────────────────────────────────────────────┘
```

## 5. 状态机设计

### 5.1 6步状态转换总图

```
FB_1004 GlueMachineFeeder 6步状态机:

步骤0(空闲) → 步骤1(等待放料完成) → 步骤2(X2前移到取料位)
→ 步骤3(等待打胶机允许) → 步骤4(发送允许抓料+等待取料完成)
→ 步骤5(X2返回待机位) → 回到步骤1

关键特点:
- 安全区管理: 步骤1必须确认取放料机构已退出工作区
- 打胶机交互: 步骤3/4与外部设备通过硬接线握手
- 循环特点: 步骤5完成后自动回到步骤1, 无需重新启动
```

## 6. 报警机制设计

### 6.1 报警码定义 (201~205段)

| 报警码 | 名称 | 触发条件 | 触发步骤 | 清除条件 | 优先级 |
|--------|------|----------|----------|----------|:------:|
| 201 | 通讯超时 | 打胶机未在规定时间(10s)内响应允许请求 | 步骤3 | 复位按钮 | 中 |
| 202 | 安全区冲突 | X2需进入工作区但取放料机构仍在区内 | 步骤1/2 | 冲突消失+复位 | 高 |
| 203 | X2轴伺服故障 | X2轴驱动器ALM信号 | - | 故障消除+复位 | 高 |
| 204 | 移动超时 | X2轴未在规定时间内到达目标位 | 步骤2/5 | 复位按钮 | 中 |
| 205 | 到达超时 | 到达目标位后等待下游响应超时 | 步骤4 | 复位按钮 | 低 |

## 7. 安全区管理

```
安全区互锁逻辑:

取放料机构工作区范围:
  ├─ Z轴下降区域 + X1轴移动区域
  └─ 当取放料机构在此区域工作时, X2轴禁止进入

安全区信号链:
  取放料机构.o_b在工作区 → 主控.i_b取放料在工作区 → 本FB.i_b安全区开放(取反)
  
安全区检查时机:
  ├─ 步骤1→2转换前: 必须确认 i_b安全区开放 = TRUE
  └─ 若不满足: 保持步骤1等待, 或触发 ALM_202(安全区冲突)
```
