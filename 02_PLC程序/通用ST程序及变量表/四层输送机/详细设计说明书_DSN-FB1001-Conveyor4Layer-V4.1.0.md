## 3. 功能描述

### 3.1 核心功能概述

四层输送机系统采用**两级功能块层次结构**设计：

#### 容器功能块 - FB_1001_Conveyor4Layer
- **职责**: 纯容器，负责输入分发和输出收集
- **不包含业务逻辑**: 仅将主控传入的数组信号拆分给4个单层FB实例
- **接口统一**: 对外提供统一的ARRAY[1..4]接口，隐藏内部层次结构

#### 单层功能块 - FB_1002_SingleLayerConveyor
- **职责**: 单层输送带的完整控制逻辑
- **9步自动状态机**: 空闲→等待启动→推料气缸伸出→输送带正转→检测到位→输送带停止→推料气缸缩回→等待循环结束→返回空闲
- **完全解耦**: 无任何X/Y/M/D物理地址引用，所有数据通过逻辑变量交互
- **内置报警机制**: 7种报警类型的检测、锁存与输出（1~7段）

### 4.2 职责分工表

| 维度 | FB_1001 (容器) | FB_1002 (单层) |
|:-----|:---------------|:--------------|
| **主要职责** | 信号路由和聚合 | 业务逻辑控制 |
| **是否包含状态机** | ❌ 不包含 | ✅ 包含9步状态机 |
| **输入处理** | 将ARRAY[1..4]拆分为单个信号 | 接收单个层的信号 |
| **输出处理** | 将单个输出聚合成ARRAY[1..4] | 输出单个层的控制信号 |
| **报警处理** | 收集4层报警并输出最高优先级 | 检测本层报警并输出代码 |
| **实例数量** | 1个 | 4个(Layer1~Layer4) |
| **复杂度** | 低（纯数据映射） | 中（完整状态机） |

### 4.4 设计原则

| 原则 | 说明 | 实现方式 |
|------|------|----------|
| **层次化封装** | 容器屏蔽内部细节 | 主控只需与容器交互，无需知道内部有4个单层FB |
| **单一职责** | 容器只做路由，单层只做逻辑 | 高内聚低耦合 |
| **完全解耦** | 无任何X/Y/M/D引用 | 所有接口为VAR_INPUT/VAR_OUTPUT逻辑变量 |
| **4层独立运行** | 各层状态机互不干扰 | 4个独立FB实例，独立计数器和状态 |
| **默认值回退** | 未连接的参数使用安全默认值 | T_DEF_*系列常量提供回退值 |
| **故障安全** | 使能丢失时全部输出归零 | `i_b使能=FALSE` 强制清零 |

### 5.2 各步骤详细说明表

| 步骤 | 名称 | 输入条件 | 动作输出 | 转出条件 | 转出目标 | 超时报警 | 典型时间 |
|:----:|:-----|:---------|:---------|:---------|:--------:|:-------:|:-------:|

| 3 | 输送带正转 | 阻挡下位 | 正转=TRUE + 保持阻挡 | 产品到位光电=TRUE | 步骤4 | ALM_5 (10s) | ~2~3s |

## 6. 报警机制设计

### 6.1 报警码定义 (1~7段)

| 报警码 | 含义 | 触发条件 | 触发步骤 | 清除条件 | 优先级 |
|--------|------|----------|----------|----------|:------:|
| 1 | 阻挡下降超时 | 阻挡气缸未在规定时间(5s)内到达下位 | 步骤2 | 复位按钮 | 高 |
| 2 | 阻挡上升超时 | 阻挡气缸未在规定时间(5s)内到达上位 | 步骤6 | 复位按钮 | 高 |
| 3 | 分料推出超时 | 分料气缸未在规定时间(5s)内到达下位 | 步骤7 | 复位按钮 | 高 |
| 4 | 复位超时 | 故障复位后设备未能初始化到安全状态 | 初始化 | 复位按钮 | 高 |
| 5 | 输送超时 | 材料未在规定时间内到达/离开到位位置 | 步骤1/步骤3 | 复位按钮 | 高 |
| 6 | 传感器冲突 | 同一气缸上下位传感器同时为TRUE | 持续检测 | 冲突消失自动清除 | 低 |
| 7 | 变频器故障 | 变频器反馈故障信号(ALM) | - | 故障消除+复位 | **最高** |

### 6.2 多层报警汇总机制

```
报警优先级汇总(FB_1001容器层):

4个单层FB各自产生报警代码(0或1~7):
  Layer1: o_i本站报警代码 = 0 (正常)
  Layer2: o_i本站报警代码 = 1 (阻挡下降超时)
  Layer3: o_i本站报警代码 = 0 (正常)
  Layer4: o_i本站报警代码 = 0 (正常)

容器层汇总逻辑:
  o_i本站报警代码 := MIN(
      fbLayer1.o_i本站报警代码,
      fbLayer2.o_i本站报警代码,
      fbLayer3.o_i本站报警代码,
      fbLayer4.o_i本站报警代码
  );
  
  结果: o_i本站报警代码 := 1 (输出最小非零值)

原则: 数值越小优先级越高; 0表示无报警(不参与MIN比较)
```

## 7. 初始化流程

### 7.1 初始化触发条件

以下任一条件满足时触发初始化：
1. **首次使能**: `i_b使能` 从FALSE变为TRUE
2. **复位操作**: 操作员按下`i_b复位`按钮
3. **故障清除**: 报警被复位清除后

## 8. 手动模式设计

### 8.1 手动操作功能

在手动模式下(`i_b手动模式=TRUE`)，每层提供6个手动操作点：

| 手动操作 | 功能描述 | 互锁关系 | 安全限制 |
|---------|---------|----------|----------|
| `i_bLx_手动阻挡下降` | 该层阻挡气缸下降(阻挡材料) | 与手动阻挡上升互斥 | 受过载保护 |
| `i_bLx_手动阻挡上升` | 该层阻挡气缸上升(释放材料) | 与手动阻挡下降互斥 | 受限位保护 |
| `i_bLx_手动分料推出` | 该层分料气缸推出(分离材料) | 与手动分料复位互斥 | 受限位保护 |
| `i_bLx_手动分料复位` | 该层分料气缸复位(收回) | 与手动分料推出互斥 | 受限位保护 |
| `i_bLx_手动正转` | 该层输送带正向点动 | 与手动反转互斥 | 受过载保护 |
| `i_bLx_手动反转` | 该层输送带反向点动(退料用) | 与手动正转互斥 | 受过载保护 |

### 8.2 数组索引约定

```
数组索引说明:

ARRAY[1] = L1 (第1层 / 底层)  ← 最先接收来料
ARRAY[2] = L2 (第2层)
ARRAY[3] = L3 (第3层)
ARRAY[4] = L4 (第4层 / 顶层)  ← 最后接收来料

物理布局示意:
     ┌─────────┐  L4 (顶层)  ARRAY[4]
     │  Layer4 │
     ├─────────┤  L3         ARRAY[3]
     │  Layer3 │
     ├─────────┤  L2         ARRAY[2]
     │  Layer2 │
     ├─────────┤  L1 (底层)  ARRAY[1]
     │  Layer1 │
     └─────────┘
        ↑ 来料方向(从下方进入)
```
