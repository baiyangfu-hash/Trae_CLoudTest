## 3. 功能描述

### 3.1 核心功能

FB_1003_PickPlace_BufferFraming 是本项目**最复杂的功能块**，实现边框缓存机取放料机构的完整控制逻辑：

- **11步自动状态机**: 完整的取放料循环控制（空闲 → 等待放料完成 → 取料下降 → 夹紧 → 检测 → 上升 → 移动 → 放料下降 → 松开 → 上升 → 完成通知）
- **4层循环管理**: 一次取两根边框（前后夹紧同时动作），4层总共循环取2次完成全部取料
- **自动/手动双模式**: 支持自动循环运行和手动点动调试（14个手动操作点）
- **初始化流程**: 上电或复位后自动执行气缸复位和伺服使能初始化
- **内置报警机制**: 8种报警类型的检测、锁存与输出（101~108段）
- **双夹爪同时动作**: 前后夹紧同时夹紧/松开，一次抓取两根边框
- **产品检测逻辑**: 4个光电传感器AND判断，确保取料可靠
- **完全解耦设计**: 无任何X/Y/M/D物理地址引用，所有数据通过逻辑变量交互

### 3.2 核心工艺说明

```
核心工艺: 一次取两根边框, 4层总共循环取2次

取料顺序:
┌─────────────────────────────────────────────────────┐
│  第1次取料 (s_i已完成层数=0)                        │
│  ├── 等待 L1+L2 放料完成                            │
│  ├── Z轴下降 -> 前后夹紧同时夹紧(一次取两根)        │
│  ├── 产品检测(长边x2 + 短边x2, 4传感器AND判断)      │
│  ├── Z轴上升 -> X1轴移动到放料位置                  │
│  ├── Z轴下降 -> 前后夹紧同时松开                    │
│  ├── Z轴上升 -> 通知下游(打胶机送料机构)            │
│  └── s_i已完成层数 += 2 (变为2)                     │
│                                                     │
│  第2次取料 (s_i已完成层数=2)                        │
│  ├── 等待 L3+L4 放料完成                            │
│  ├── 重复上述取放料流程                             │
│  └── s_i已完成层数 += 2 (变为4, 全部完成)           │
│                                                     │
│  循环重置: s_i已完成层数 >= 4 时归零, 等待下一轮启动 │
└─────────────────────────────────────────────────────┘
```

### 4.3 设计原则

| 原则 | 说明 | 实现方式 | 特殊性（相比其他FB） |
|------|------|----------|:-----|
| **纯逻辑** | 不包含X/Y/M/D地址引用 | 所有接口为VAR_INPUT/VAR_OUTPUT逻辑变量 | 接口数量最多(52输入)，映射复杂度最高 |
| **数据解耦** | 通过变量与主控交互 | 主控负责IO映射，本FB只处理业务逻辑 | 需要处理X1轴内部请求标志传递 |
| **故障安全** | 使能丢失时所有输出归零 | `i_b使能=FALSE`时强制清零所有输出 | 涉及6个气缸+3个伺服输出，归零范围广 |
| **自包含** | 内部完成全部控制逻辑 | 无需外部辅助FB，独立完成11步状态机 | 状态机步骤最多，逻辑最复杂 |
| **层循环** | 自动管理4层取放料进度 | 内置`s_i已完成层数`计数器，支持断点续接 | **独有特性**：其他FB无此循环逻辑 |
| **双夹爪** | 前后夹紧同时动作 | 步骤3/8中同时输出前/后夹紧信号 | **独有特性**：一次取两根边框 |
| **产品检测** | 4传感器AND判断 | 步骤4中检测长边x2 + 短边x2全为TRUE | **独有特性**：其他FB无此检测机制 |

### 5.2 各步骤详细说明（11步完整表）

| 步骤 | 名称 | 输入条件 | 动作输出 | 转出条件 | 转出目标 | 超时报警 | 特殊说明 |
|:----:|:-----|:---------|:---------|:---------|:---------|:--------:|:--------|

| 1 | **等待放料完成** | 进入步骤0 | 无 | 对应层放料完成(L1+L2或L3+L4) | 步骤2 | - | **关键步骤**: 根据`s_i已完成层数`决定等待哪些层 |

### 5.3 4层循环管理逻辑详解

#### 5.3.1 循环原理

```
4层循环计数器工作原理:

核心变量:
  s_i已完成层数: INT  (* 已完成的层数累计值 *)
  s_i取料次数:    INT  (* 当前第几次取料 *)
  LAYER_TOTAL:    INT = 4  (* 总层数常量 *)

初始状态:
  s_i已完成层数 = 0
  s_i取料次数 = 0

═════════════════════════════════════════════════════════
  第1轮取料 (s_i已完成层数 = 0)
═════════════════════════════════════════════════════════
  ├─ 启动触发 -> s_i取料次数 := 1
  ├─ 步骤1: 等待条件判断
  │   └─ s_i已完成层数(0) < 2 -> 等待 L1 AND L2 放料完成
  │      (即: i_输送机_L1_放料完成 AND i_输送机_L2_放料完成)
  ├─ 步骤2~9: 执行完整取放料循环(取L1+L2两层)
  ├─ 步骤10完成:
  │   ├─ s_i已完成层数 += 2  =>  变为 2
  │   └─ 分支判断: s_i已完成层数(2) < LAYER_TOTAL(4)
  │      └─ -> 回到步骤1 (继续循环, 无需重新启动!)

═════════════════════════════════════════════════════════
  第2轮取料 (s_i已完成层数 = 2)
═════════════════════════════════════════════════════════
  ├─ s_i取料次数 := 2 (自动递增)
  ├─ 步骤1: 等待条件判断
  │   └─ s_i已完成层数(2) >= 2 -> 等待 L3 AND L4 放料完成
  │      (即: i_输送机_L3_放料完成 AND i_输送机_L4_放料完成)
  ├─ 步骤2~9: 执行完整取放料循环(取L3+L4两层)
  ├─ 步骤10完成:
  │   ├─ s_i已完成层数 += 2  =>  变为 4
  │   └─ 分支判断: s_i已完成层数(4) >= LAYER_TOTAL(4)
  │      └─ -> 回到步骤0 (全部完成, 归零等待下一轮)

═════════════════════════════════════════════════════════
  全部完成 & 循环重置
═════════════════════════════════════════════════════════
  ├─ s_i已完成层数(4) >= LAYER_TOTAL(4)
  ├─ 归零操作:
  │   ├─ s_i已完成层数 := 0
  │   ├─ s_i取料次数 := 0
  │   └─ 状态机回到步骤0(空闲待机)
  └─ 等待下一次启动触发(新的i_b启动上升沿)
```

#### 5.3.2 步骤1的条件判断伪代码

```st
(* 步骤1: 等待放料完成的条件判断逻辑 *)
CASE s_i已完成层数 OF
    0, 1:
        (* 第1次取料: 等待L1+L2 *)
        b放料完成条件 := i_输送机_L1_放料完成 AND i_输送机_L2_放料完成;
        
    2, 3:
        (* 第2次取料: 等待L3+L4 *)
        b放料完成条件 := i_输送机_L3_放料完成 AND i_输送机_L4_放料完成;
        
    ELSE
        (* 异常状态: 已完成层数超出范围 *)
        b放料完成条件 := FALSE;
        (* 触发内部错误处理 *)
END_CASE;

IF b放料完成条件 THEN
    (* 条件满足, 转到步骤2 *)
    s_i步序 := STP_取料下降;  (* = 2 *)
END_IF;
```

#### 5.3.3 步骤10的分支判断伪代码

```st
(* 步骤10完成后: 决定回到步骤1还是步骤0 *)
s_i已完成层数 := s_i已完成层数 + 2;  (* 每次取2层 *)

IF s_i已完成层数 >= LAYER_TOTAL THEN
    (* 4层全部取完, 回到空闲 *)
    s_i步序 := STP_空闲;       (* = 0 *)
    s_i已完成层数 := 0;        (* 归零 *)
ELSE
    (* 还有未取的层, 继续循环 *)
    s_i步序 := STP_等待放料完成; (* = 1 *)
    (* 注意: 不需要重新按启动按钮! 自动继续 *)
END_IF;
```

### 5.4 双夹爪同时动作机制

#### 5.4.1 设计背景

取放料机构采用**双夹爪设计**（前夹爪 + 后夹爪），目的是在一次取料动作中同时抓取两根边框，提升效率。

```
机械结构示意:

    ┌──────────────────────────────────────┐
    │           边框缓存机工作台            │
    │                                      │
    │   ┌─────┐  ┌─────┐                   │
    │   │ L1  │  │ L2  │  ← 第1、2层边框    │
    │   └──┬──┘  └──┬──┘                   │
    │      │  前夹爪  │                      │
    │      │ (抓L1)  │                      │
    │   ┌──┴──┐  ┌──┴──┐                   │
    │   │ L3  │  │ L4  │  ← 第3、4层边框    │
    │   └──┬──┘  └──┬──┘                   │
    │      │  后夹爪  │                      │
    │      │ (抓L2)  │                      │
    │      ▼         ▼                      │
    │   [前夹紧气缸] [后夹紧气缸]            │
    │                                      │
    └──────────────────────────────────────┘

取料时: 前夹爪抓L1(或L3), 后夹爪抓L2(或L4), 同时动作!
```

#### 5.4.2 步骤3（夹紧）的双夹爪同步逻辑

```st
(* 步骤3: 双夹爪同时夹紧 *)
(* 进入步骤3时的动作 *)
o_前夹紧_夹紧  := TRUE;   (* 前夹紧电磁阀通电 *)
o_后夹紧_夹紧  := TRUE;   (* 后夹紧电磁阀通电 *)
o_前夹紧_松开  := FALSE;
o_后夹紧_松开  := FALSE;

(* 启动夹紧确认定时器 *)
s_t夹紧定时器(IN := TRUE, PT := T#i_i夹紧确认时间ms);

(* 等待夹紧完成 - 两种判断方式并行 *)
IF s_t夹紧定时器.Q THEN
    (* 方式A: 时间到(默认方式) *)
    b夹紧完成 := TRUE;
ELSIF i_前夹紧_动点 AND i_后夹紧_动点 THEN
    (* 方式B: 两个磁开关都检测到夹紧到位 *)
    b夹紧完成 := TRUE;
    (* 注意: 如果使用传感器确认方式, 可以缩短等待时间 *)
END_IF;

IF b夹紧完成 THEN
    s_t夹紧定时器(IN := FALSE);  (* 停止定时器 *)
    s_i步序 := STP_产品检测;      (* -> 步骤4 *)
ELSIF s_t夹紧超时.Q THEN
    (* 超时未夹紧到位 -> 触发ALM_105 *)
    o_i本站报警代码 := 105;
    s_i步序 := STP_故障;          (* -> 步骤99 *)
END_IF;
```

#### 5.4.3 步骤8（松开）的双夹爪同步逻辑

```st
(* 步骤8: 双夹爪同时松开 *)
(* 与步骤3对称, 但松开方向相反 *)
o_前夹紧_夹紧  := FALSE;
o_后夹紧_夹紧  := FALSE;
o_前夹紧_松开  := TRUE;   (* 前夹紧松开电磁阀通电 *)
o_后夹紧_松开  := TRUE;   (* 后夹紧松开电磁阀通电 *);

(* 启动松开确认定时器 *)
s_t松开定时器(IN := TRUE, PT := T#i_i夹紧确认时间ms);

(* 等待松开完成 *)
IF s_t松开定时器.Q OR (i_前夹紧_原点 AND i_后夹紧_原点) THEN
    b松开完成 := TRUE;
END_IF;

IF b松开完成 THEN
    s_i步序 := STP_放料上升;      (* -> 步骤9 *)
ELSIF s_t松开超时.Q THEN
    o_i本站报警代码 := 106;        (* ALM_106: 松开超时 *)
    s_i步序 := STP_故障;
END_IF;
```

### 5.5 产品检测逻辑（4传感器AND判断）

#### 5.5.1 检测原理

```
产品检测传感器布局:

    ┌──────────────────────────────────────────┐
    │              边框(已夹紧)                 │
    │                                          │
    │    ┌──────────────────────────┐          │
    │    │                          │          │
    │  [S1]                      [S2] │  ← 长边检测传感器(x2)
    │  (长边1)                    (长边2)        │
    │    │                          │          │
    │    │                          │          │
    │    │                          │          │
    │  [S3]                      [S4] │  ← 短边检测传感器(x2)
    │  (短边1)                    (短边2)        │
    │    └──────────────────────────┘          │
    │                                          │
    └──────────────────────────────────────────┘

传感器对应关系:
  S1 = i_长边1检测  (远程IO_1: XC)
  S2 = i_长边2检测  (远程IO_1: XD)
  S3 = i_短边1检测  (远程IO_1: XE)
  S4 = i_短边2检测  (远程IO_1: XF)
```

#### 5.5.2 AND判断逻辑

```st
(* 步骤4: 产品检测 - 4传感器AND判断 *)
b产品检测通过 := 
    i_长边1检测 AND    (* S1: 长边1存在 *)
    i_长边2检测 AND    (* S2: 长边2存在 *)
    i_短边1检测 AND    (* S3: 短边1存在 *)
    i_短边2检测;      (* S4: 短边2存在 *)

(* 检测结果处理 *)
IF b产品检测通过 THEN
    (* 4个传感器全部为TRUE -> 边框确实被夹住了 *)
    s_i步序 := STP_取料上升;  (* -> 步骤5: 开始上升 *)
ELSE
    (* 任一传感器为FALSE -> 可能原因:                    *)
    (*   1. 边框没有正确放置在取料位置                    *)
    (*   2. 夹爪没有真正夹紧(虽然气缸动作了)              *)
    (*   3. 传感器脏污或偏移                             *)
    (*   4. 边框尺寸不符合规格                          *)
    
    (* 启动检测超时定时器 - 允许短暂的不稳定期 *)
    IF NOT s_t检测等待.IN THEN
        s_t检测等待(IN := TRUE, PT := T#500ms);  (* 等待500ms稳定 *)
    ELSIF s_t检测等待.Q THEN
        (* 500ms后仍未全部为TRUE -> 触发ALM_107 *)
        o_i本站报警代码 := 107;  (* 产品检测失败 *)
        s_i步序 := STP_故障;     (* -> 步骤99 *)
    END_IF;
END_IF;
```

> 完整真值表请参阅: [接口文档_IFC-FB1003-PickPlace-V4.0.0.md](./接口文档_IFC-FB1003-PickPlace-V4.0.0.md) 第6章

### 5.6 X1轴运动请求机制（内部请求标志）

#### 5.6.1 设计原因

本功能块的Z轴可以直接输出脉冲(`o_Z轴_脉冲输出`)和方向(`o_Z轴_方向输出`)，但**X1轴的运动不由本FB直接输出脉冲**，而是通过内部请求标志传递给主控程序统一处理。

```
设计理由:
┌─────────────────┐    s_bX1轴请求_*    ┌─────────────────┐
│  FB_1003        │ ──────────────────> │  主控程序(Main)  │
│  (纯逻辑)       │                     │                 │
│  决定"何时移动"  │                     │  处理"如何移动"   │
│  和"移动方向"    │                     │  (脉冲输出/定位)  │
└─────────────────┘                     └─────────────────┘

原因:
1. X1轴可能被多个FB共享(如未来扩展)
2. 主控需要统一协调多轴运动(防干涉)
3. 保持FB的纯逻辑特性(不直接操作硬件)
```

#### 5.6.2 内部请求标志定义

| 变量名 | 类型 | 含义 | 有效时机 | 触发步骤 |
|--------|:----:|:-----|:--------:|:--------:|
| `s_bX1轴请求前移` | BOOL | 请求X1轴向取料方向移动(正向) | 手动模式下手动点动前有效 | 手动操作 |
| `s_bX1轴请求后移` | BOOL | 请求X1轴向放料方向移动(反向) | 步骤6期间为TRUE | 步骤6 |

**访问方式**: 这些是内部VAR变量，需要在主控中通过实例名访问：
```st
bX1ForwardReq := fb取放料机构.s_bX1轴请求前移;
bX1BackwardReq := fb取放料机构.s_bX1轴请求后移;
```

#### 5.6.3 步骤6的X1轴运动时序

```
步骤6: 移动到放料位置的完整时序

本FB(Z轴/X1轴)           主控程序              X1轴伺服
     │                      │                     │
     │── s_bX1轴请求后移=ON ─>│                     │
     │                      │── 脉冲输出(Y1/Y2) ──>│
     │                      │   方向(Y5)=后移       │
     │                      │                     │
     │   (等待...)           │── 位置反馈检查 ──────│
     │                      │                     │
     │<── s_bX1轴在放料位=ON─│<── 到达目标位置 ────│
     │   (由主控设置)        │                     │
     │                      │                     │
     │── 转到步骤7           │                     │
```

## 6. 报警机制设计

### 6.1 报警码定义 (101~108段)

> 完整报警码定义请参阅: [接口文档_IFC-FB1003-PickPlace-V4.0.0.md](./接口文档_IFC-FB1003-PickPlace-V4.0.0.md) 第4.3节

| 报警码 | 名称 | 触发条件 | 触发步骤 | 清除条件 | 优先级 |
|:------:|:-----|:---------|:--------:|:---------|:------:|
| 0 | 无报警 | - | - | - | - |
| 101 | Z轴伺服故障 | Z轴伺服驱动器ALM有效 | 所有步骤(持续) | 故障消除+复位 | **最高** |
| 102 | X1轴伺服故障 | X1轴伺服驱动器ALM有效 | 所有步骤(持续) | 故障消除+复位 | **最高** |
| 103 | 升降下降超时 | 升降未在规定时间内到达下位 | 步骤2 / 步骤7 | 复位按钮 | 高 |
| 104 | 升降上升超时 | 升降未在规定时间内到达上位 | 步骤5 / 步骤9 | 复位按钮 | 高 |
| 105 | 夹紧超时 | 夹爪未在规定时间内夹紧到位 | 步骤3 | 复位按钮 | 高 |
| 106 | 松开超时 | 夹爪未在规定时间内松开到位 | 步骤8 | 复位按钮 | 高 |
| 107 | 产品检测失败 | 4个产品检测传感器未全为TRUE | 步骤4 | 复位按钮 | 中 |
| 108 | 移动超时(X1轴) | X1轴未在规定时间内到达放料位置 | 步骤6 | 复位按钮 | 高 |

### 6.2 报警优先级处理规则

**优先级规则**: 数值越小优先级越高。多个报警同时存在时输出最小值。

```
优先级层次:
  Level 1 (最高): 伺服故障 101/102
    → 立即停止对应轴的所有运动请求
    → 需要先排除物理故障
    
  Level 2 (高):   超时类 103/104/105/106/108
    → 停止当前步骤的动作输出
    → 保持当前位置(不归零, 方便排查)
    → 需要按复位清除
    
  Level 3 (中):   检测类 107
    → 停止当前步骤, 不继续后续动作
    → 夹爪保持当前状态(可能仍夹紧!)
    → 需要人工检查后复位
```

### 6.3 报警处理流程

```
报警检测与处理流程:

状态机运行中 (步骤0~10)
     │
     ├─ 持续检测: 伺服故障(101/102)
     │   ├─ i_Z轴_伺服故障=TRUE  → ALM_101, 停止Z轴所有输出
     │   └─ i_X1轴_伺服故障=TRUE → ALM_102, 停止X1轴请求
     │
     ├─ 步骤2/7中: 检查升降是否到达下位
     │   └─ 超时(i_i升降动作时间)未到达 → ALM_103
     │
     ├─ 步骤5/9中: 检查升降是否到达上位
     │   └─ 超时(i_i升降动作时间)未到达 → ALM_104
     │
     ├─ 步骤3中: 检查夹紧是否完成
     │   └─ 超时(i_i夹紧确认时间)未完成 → ALM_105
     │
     ├─ 步骤4中: 检查4传感器AND结果
     │   └─ 500ms后仍未全TRUE → ALM_107
     │
     ├─ 步骤6中: 检查X1轴是否到达放料位
     │   └─ 超时未到达 → ALM_108
     │
     └─ 步骤8中: 检查松开是否完成
         └─ 超时(i_i夹紧确认时间)未完成 → ALM_106

报警激活后:
     │
     ├─ o_i本站报警代码 := 最高优先级活跃报警码
     ├─ o_b故障 := TRUE
     ├─ s_i步序 := 99 (故障态)
     ├─ 状态机暂停(不再执行步骤转换)
     └─ 运动输出保持最后状态(安全考虑: 夹爪保持夹紧!)

报警清除流程:
     │
     └─ 操作员按下 i_b复位 按钮
         ├─ 清除所有报警锁存标志
         ├─ o_i本站报警代码 := 0
         ├─ o_b故障 := FALSE
         ├─ 强制所有输出归零(包括夹爪松开!)
         ├─ s_i步序 := 0 (回到空闲)
         ├─ 执行初始化流程
         └─ 初始化完成后可重新启动
```

### 8.2 手动模式约束

1. **生效条件**: 仅在`i_b手动模式=TRUE`且`i_b使能=TRUE`且本站无故障(`o_b故障=FALSE`)时生效
2. **互锁关系**: 同一组内的上升/下降、夹紧/松开互斥（不能同时为TRUE）
3. **安全限制**: 
   - 伺服点动受限位开关保护
   - 气缸操作受磁性开关反馈约束
4. **输出方式**: 
   - Z轴点动: 直接输出`o_Z轴_脉冲输出`和`o_Z轴_方向输出`
   - X1轴点动: 通过`s_bX1轴请求_*`传递给主控
   - 气缸操作: 直接输出对应的`o_*`信号
5. **模式切换**: 从手动切到自动时，必须先停止所有手动操作（建议先回到步骤0）
