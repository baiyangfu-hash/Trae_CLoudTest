### 4.2 变更原因

#### 4.2.1 背景
原有FBD使用说明文档(`FBD使用说明_FBD-DJ-2026-005-V4.0.0.md`)存在以下问题：

1. **内容臃肿**: 单个文档约650行，混合了设计说明、接口定义、使用指南、变更历史等多种不同职责的内容
2. **查找效率低**: 需要在650行文档中搜索特定信息（如接口定义或调用示例）
3. **维护困难**: 修改任何一部分都需要编辑整个大文档
4. **不符合规范**: 未遵循项目的文档分层标准（DSN/IFC/UM/CHG分离原则）
5. **ASCII art冗余**: 包含大量过于详细的图形化表示，占用大量篇幅但信息密度低

#### 4.2.2 必要性
用户明确要求：
- "每个fb添加详细设计说明书，变更记录，接口文档以及使用说明"
- "把原有的fbd说明相应的内容拆分到其他文档里"
- "删除fbd文档，规范里也不要有这个fbd文档，内容臃肿"

#### 4.2.3 参考
- 用户需求: 2026-04-24 直接指令
- 项目规范: 文档分层标准（DSN设计、IFC接口、UM使用、CHG变更）

### 6.2 典型变更场景示例

#### 场景A: Bug修复 (DEF)

```markdown
### V4.0.1 (2026-04-25): 修复X2轴移动超时误报

**变更编号**: CHG-PLC-DEF-2026-005
**变更类型**: CHG-PLC (程序缺陷修复)
**业务性质**: DEF (缺陷修复)
**影响范围**: LOCAL (局部 - 仅影响打胶机送料FB的超时判断逻辑)

#### 变更原因:
- **背景**: 现场调试发现X2轴在大负载情况下偶尔触发ALM_204超时报警
- **必要性**: 当前T_DEF_移动超时时间=5s过短，无法满足大负载工况
- **参考**: 现场调试报告 DR-2026-001

#### 变更内容:
- **变更前**: T_DEF_移动超时时间 = 5000ms (5秒)
- **变更后**: T_DEF_移动超时时间 = 8000ms (8秒)
- **具体改动**: 
  1. 修改常量定义 T_DEF_移动超时时间 从5000改为8000
  2. 更新DSN文档中的参数推荐值
  3. 更新UM文档中的调优建议

#### 影响分析:
- Scope: 🟢 低 - 仅修改一个常量值
- Schedule: 🟢 低 - 修改简单，无需大量测试
- Cost: 🟢 无 - 纯软件修改
- Quality: 🟢 中 - 提升现场适应性
- Risk: 🟢 低 - 延长超时时间降低误报率

#### 受影响文件:

| DSN-FB1004-*.md | 修改 | 参数推荐值更新 |

- **实施日期**: 2026-04-25
- **实施人**: 张工程师
```

### 8.2 文档版本联动规则

```
版本号联动机制:

当FB源代码(.st文件)发生修改时:
  │
  ├──→ CHG文档: 新增变更记录条目 (必须!)
  │
  ├──→ DSN文档: 如涉及设计逻辑变更则升级版本号
  │
  ├──→ IFC文档: 如涉及接口变更(新增/删除/修改针脚)则升级版本号
  │
  ├──→ UM文档: 如涉及使用方式变更则升级版本号
  │
  └──→ 所有4个文档版本号必须保持同步!
  
示例:
  FB源代码: V4.0.0 → V4.0.1 (Bug修复)
      │
      ├──→ CHG-FB1004: V4.0.0 → V4.0.1 (追加变更记录)
      ├──→ DSN-FB1004: V4.0.0 → V4.0.1 (如设计有变)
      ├──→ IFC-FB1004: V4.0.0 → V4.0.1 (如接口有变)
      └──→ UM-FB1004:  V4.0.0 → V4.0.1 (如使用有变)
      
  如果只是文档修正(不影响功能):
      │
      └──→ 可以只升级文档版本号: V4.0.0 → V4.0.0.1 (修订号)
          (FB源代码版本不变)
```
