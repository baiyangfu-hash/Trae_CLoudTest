## 2. 版本变更说明 (V3.0.0 -> V4.0.0)

### 2.1 架构重构核心变化

| 对比项 | V3.0.0 (旧架构) | V4.0.0 (新架构) |
|--------|------------------|------------------|
| **架构模式** | 三层碎片化架构(Device+Logic+Main=21个FB) | 扁平化组件架构(主控+3工站+公共报警=5个组件) |
| **功能块数量** | 21个功能块 | 5个组件(1个PROGRAM + 4个FUNCTION_BLOCK) |
| **IO映射方式** | 分散在各Device层FB中 | 集中在主控程序(主控程序承担所有X/Y映射) |
| **物理地址依赖** | Device层FB直接引用X/Y/M/D地址 | 所有工站FB为纯逻辑块，无任何物理地址引用 |
| **数据耦合度** | 高(多层传递，接口复杂) | 低(主控统一协调，工站接口清晰) |
| **D区规划** | 无统一规划 | 新增D400~D425固定分配(报警系统专用) |
| **M区规划** | 无统一规划 | M200全局互锁，M100~M149 HMI控制/状态 |

## 3. D区地址规划 (数据寄存器)

> V4.0.0新增统一的D区地址规划，由主控程序负责写入。所有工站功能块不直接操作D区。

### 3.1 D区总览

| 地址范围 | 位宽 | 用途 | 写入来源 | 读取方 |
|----------|------|------|----------|--------|
| **D100~D117** | 9 x WORD (18D) | HMI工艺参数设定(REAL/INT) | HMI写入 -> 主控读取 | 主控程序 |
| **D120~D127** | 4 x WORD (8D) | HMI状态回写(报警代码/层数/步序) | 主控写入 | HMI读取 |
| **D400** | WORD (1D) | 全局报警字(HMI报警灯指示用) | FB_2001_CommonAlarm_AllStation输出 -> 主控写入 | HMI读取 |
| **D402** | INT (1D,占2D) | 当前最高优先级报警码(HMI文本显示用) | FB_2001_CommonAlarm_AllStation输出 -> 主控写入 | HMI读取 |
| **D404** | INT (1D,占2D) | MES累计报警次数 | FB_2001_CommonAlarm_AllStation输出 -> 主控写入 | MES系统读取 |
| **D406~D425** | 10 x INT (20D) | MES报警队列(最近10条去重记录) | FB_2001_CommonAlarm_AllStation输出 -> 主控写入 | MES系统读取 |

### 3.2 HMI参数设定区 (D100~D117)

| D地址 | 变量名 | 数据类型 | 说明 | HMI操作 |
|-------|--------|----------|------|---------|
| D100 | rHMI_输送速度设定 | REAL | 输送带速度设定 (Hz或m/min) | 设定写入 |
| D102 | rHMI_伺服速度设定 | REAL | 伺服轴速度设定 (mm/s或%) | 设定写入 |
| D104 | rHMI_取料速度设定 | REAL | 取料过程速度 (mm/s) | 设定写入 |
| D106 | rHMI_放料速度设定 | REAL | 放料过程速度 (mm/s) | 设定写入 |
| D108 | iHMI_分料时间设定 | INT | 分料动作保持时间 (ms) | 设定写入 |
| D110 | iHMI_夹紧确认时间设定 | INT | 夹爪夹紧确认等待时间 (ms) | 设定写入 |
| D112 | iHMI_升降动作时间设定 | INT | 升降气缸动作超时时间 (ms) | 设定写入 |

### 3.3 HMI状态回写区 (D120~D127)

| D地址 | 变量名 | 数据类型 | 说明 | 方向 |
|-------|--------|----------|------|------|
| D120 | iHMI_全局报警代码 | INT | 当前最高优先级报警码 | 主控->HMI |
| D122 | wHMI_全局报警字 | INT(WORD转) | 全局报警字(INT形式显示) | 主控->HMI |
| D124 | iHMI_当前取料层数 | INT | 当前正在处理的层数 (0~4) | 主控->HMI |
| D126 | iHMI_取放料步序 | INT | 取放料状态机步序 (0~10) | 主控->HMI |

### 3.4 报警系统专用区 (D400~D425)

| D地址 | 位宽 | 来源变量(来自FB_2001_CommonAlarm_AllStation) | 说明 |
|-------|------|---------------------------|------|
| **D400** | WORD (16bit) | o_w全局报警字 | 各非零报警码的位或(OR)组合；HMI用于报警灯指示(非零=有报警) |
| **D402** | INT (16bit) | o_i当前报警代码 | 当前最高优先级报警码数值(0=无报警)；HMI用于文本显示具体报警内容 |
| **D404** | INT (16bit) | o_iMES报警计数 | 自上次复位以来的累计报警次数统计(含重复报警) |
| **D406, D407** | INT (32bit) | o_iMES报警队列[0] | 最新一条报警码(索引0=最新) |
| **D408, D409** | INT | o_iMES报警队列[1] | 第2新报警码 |
| **D410, D411** | INT | o_iMES报警队列[2] | 第3新报警码 |
| **D412, D413** | INT | o_iMES报警队列[3] | 第4新报警码 |
| **D414, D415** | INT | o_iMES报警队列[4] | 第5新报警码 |
| **D416, D417** | INT | o_iMES报警队列[5] | 第6新报警码 |
| **D418, D419** | INT | o_iMES报警队列[6] | 第7新报警码 |
| **D420, D421** | INT | o_iMES报警队列[7] | 第8新报警码 |
| **D422, D423** | INT | o_iMES报警队列[8] | 第9新报警码 |
| **D424, D425** | INT | o_iMES报警队列[9] | 最旧一条报警码(索引9=最旧) |

> **注意**: 三菱PLC的INT类型为16bit，占用2个连续D寄存器(小端模式：低字节在前)。每个INT报警码实际占2个D地址。

### 6.4 FB_2001_CommonAlarm_AllStation -- 公共报警集中管理纯逻辑功能块

> **文件**: `公共服务/FB_2001_CommonAlarm_AllStation_V4.0.0.st`
> **特点**: 纯逻辑块，不包含任何X/Y/M/D地址引用
> **职责**: 汇聚三工站报警，计算全局报警字，维护MES报警队列

#### VAR_INPUT (输入参数)

| 参数名 | 类型 | 说明 |
|--------|------|------|
| i_i输送机报警 | INT | 来自FB_1001_Conveyor4Layer_BufferFraming的o_i本站报警代码 (0/1~7) |
| i_i取放料报警 | INT | 来自FB_1003_PickPlace_BufferFraming的o_i本站报警代码 (0/101~108) |
| i_i送料机构报警 | INT | 来自FB_1004_GlueMachineFeeder_BufferFraming的o_i本站报警代码 (0/201~205) |
| i_b复位 | BOOL | HMI复位按钮(电平有效)，按下时清除所有历史和队列 |

#### VAR_OUTPUT (输出参数)

| 参数名 | 类型 | 写入目标 | 说明 |
|--------|------|----------|------|
| o_w全局报警字 | WORD | D400(由主控写入) | 各非零报警码的OR组合，非零即有报警 |
| o_i当前报警代码 | INT | D402(由主控写入) | 最高优先级报警码(数值最小优先) |
| o_b任何报警激活 | BOOL | M200(由主控写入) | 至少一站有报警时为TRUE |
| o_iMES报警计数 | INT | D404(由主控写入) | 自上次复位以来的累计报警次数 |
| o_iMES报警队列 | ARRAY[0..9] OF INT | D406~D425(由主控写入) | 最近10条去重报警记录 |
| o_b新报警标志 | BOOL | (主控直接读取) | 新报警单周期脉冲标志，触发MES上传 |

#### 核心算法说明

**优先级规则**: 数值越小优先级越高
- 四层输送机 (1~7) > 取放料机构 (101~108) > 送料机构 (201~205)

**全局报警字计算**: 将每个非零INT报警码转换为WORD后做位或(OR)运算

**MES队列管理策略**:
- 仅在有新报警且o_b任何报警激活=TRUE时入队
- 入队前去重: 相同报警码已在队列中则不入队
- 队列满(10条)时覆盖最旧记录
- 复位时整个队列清零

**新报警脉冲特性**: o_b新报警标志仅保持1个扫描周期为TRUE
