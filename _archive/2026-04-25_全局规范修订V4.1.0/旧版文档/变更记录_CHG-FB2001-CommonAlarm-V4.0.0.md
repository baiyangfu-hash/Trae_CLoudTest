## 4. V4.0.0 版本详细变更说明

### 4.1 变更基本信息

| 属性 | 值 |
|------|-----|
| **变更编号** | CHG-DOCU-2026-003 |
| **技术领域** | ☑ DOCU (工程文档) |
| **业务性质** | ☑ OPT (优化改进) - 文档体系建立 |
| **影响范围** | ☑ MODULE (模块级) - 单个FB(公共报警管理器)文档体系 |
| **关联变更单** | 无（首次建立） |

### 4.2 变更原因

#### 4.2.1 背景

FB_2001_CommonAlarm_AllStation 作为边框缓存机的**公共服务功能块**，承担着三工站报警汇聚、全局互锁控制、MES数据接口等核心职责，但此前缺少完整的规范化文档体系：

1. **仅有ST源代码和报警码定义文档**: 缺少设计层面的算法说明、接口层面的完整定义、使用层面的调用指南
2. **作为公共服务FB影响面广**: 本FB的任何变更都会同时影响FB_1001/FB_1003/FB_1004三个工站FB以及主控程序、HMI界面、MES系统
3. **调试和维护困难**: 缺乏文档导致后续维护人员需要阅读全部ST源代码才能理解其行为
4. **不符合项目规范**: 项目要求每个FB都应具备DSN/IFC/UM/CHG四类独立文档

#### 4.2.2 必要性

用户明确要求为FB_2001创建完整的4个规范化文档，且特别强调：
- "基于实际ST源代码分析, 不要虚构"
- "格式规范统一(与其他3个FB文档风格完全一致)"
- "特别体现公共服务FB的特殊性(汇聚3个工站、MES接口、全局互锁)"

#### 4.2.3 参考

- 用户需求: 2026-04-24 直接指令
- 项目规范: 文档分层标准（DSN设计、IFC接口、UM使用、CHG变更）
- 参考文档: 打胶机送料机构(FB_1004)的四类文档作为格式模板

| MES报警队列 | 最近10条去重记录 → D406~D425 | 循环缓冲区 |
| 新报警脉冲 | 单周期脉冲触发MES上传 | 变化检测 |
| 多报警状态 | 分站状态数组 + 分站BOOL量 | 透传输出 |

### 8.2 文档版本联动规则

```
版本号联动机制:

当FB_2001源代码(.st文件)发生修改时:
  │
  ├──→ CHG文档: 新增变更记录条目 (必须!)
  │
  ├──→ DSN文档: 如涉及设计逻辑变更则升级版本号
  │   (如: 算法修改/处理流程变化/复位机制调整)
  │
  ├──→ IFC文档: 如涉及接口变更则升级版本号
  │   (如: 新增/删除/修改输入输出针脚)
  │
  ├──→ UM文档: 如涉及使用方式变更则升级版本号
  │   (如: 调用方式改变/配置参数变化)
  │
  └──→ 所有4个文档版本号必须保持同步!

⚠️ 公共服务FB特殊规则:
  任何接口变更都必须通知以下关联方:
  → FB_1001/FB_1003/FB_1004 (数据源FB)
  → 主控程序开发人员 (IO映射)
  → HMI工程师 (画面配置)
  → MES工程师 (数据采集)

示例:
  FB源代码: V4.0.0 → V4.0.1 (队列容量10→20)
      │
      ├──→ CHG-FB2001: V4.0.0 → V4.0.1 (追加变更记录)
      ├──→ DSN-FB2001: V4.0.0 → V4.0.1 (队列算法描述更新)
      ├──→ IFC-FB2001: V4.0.0 → V4.0.1 (地址映射D406~D425→D445)
      └──→ UM-FB2001:  V4.0.0 → V4.0.1 (调试步骤+HMI配置更新)
      
  如果只是文档修正(不影响功能):
      │
      └──→ 可以只升级文档版本号: V4.0.0 → V4.0.0.1 (修订号)
          (FB源代码版本不变)
```
