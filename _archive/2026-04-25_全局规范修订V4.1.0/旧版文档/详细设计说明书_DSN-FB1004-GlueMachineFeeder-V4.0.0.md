## 3. 功能描述

### 3.1 核心功能

FB_1004_GlueMachineFeeder_BufferFraming 是一个**纯逻辑功能块**，实现边框缓存机打胶机送料机构的完整控制逻辑：

- **6步自动状态机**: 完整的送料循环控制（空闲 → 等待放料完成 → 移动到取料位 → 等待允许 → 发送允许抓料 → 返回待机）
- **自动/手动双模式**: 支持自动循环运行和手动点动调试
- **初始化流程**: 上电或复位后自动执行伺服状态检查和初始化
- **内置报警机制**: 5种报警类型的检测、锁存与输出（201~299段）
- **安全区管理**: 与取放料机构的区域互锁逻辑，防止机械碰撞
- **完全解耦设计**: 无任何X/Y/M/D物理地址引用，所有数据通过逻辑变量交互

自动循环特点:
┌─────────────────────────────────────────────────────────────┐

### 4.3 设计原则

| 原则 | 说明 | 实现方式 |
|------|------|----------|
| **纯逻辑** | 不包含X/Y/M/D地址引用 | 所有接口为VAR_INPUT/VAR_OUTPUT逻辑变量 |
| **数据解耦** | 通过变量与主控交互 | 主控负责IO映射，本FB只处理业务逻辑 |
| **故障安全** | 使能丢失时所有输出归零 | `i_b使能=FALSE` 时强制清零所有输出 |
| **自包含** | 内部完成全部控制逻辑 | 无需外部辅助FB，独立完成6步状态机 |
| **安全区优先** | 与取放料机构互锁 | 通过`o_安全区信号`防止机械碰撞 |
| **通讯保护** | 打胶机交互带超时检测 | 步骤3/4设置10秒通讯超时保护 |

### 5.2 各步骤详细说明

| 步骤 | 名称 | 输入条件 | 动作输出 | 转出条件 | 转出目标 | 超时报警 |
|------|------|----------|----------|----------|----------|----------|
| 0 | 空闲/待机位置 | - | 全部OFF | 启动触发(上升沿) | 步骤1 | - |
| 1 | 等待取放料放料完成 | 进入步骤0 | 无 | 放料完成 AND 安全区开放 | 步骤2 | - |
| 2 | 移动到打胶机取料位置 | 放料完成 | X2轴前移请求=TRUE | 到达取料位(主控反馈) | 步骤3 | ALM_204 (5秒) |
| 3 | 等待打胶机允许送料 | 到达取料位 | 无 | 允许送料=TRUE | 步骤4 | ALM_201 (10秒) |
| 4 | 发送允许抓料+等待取料完成 | 允许送料 | 允许抓料=TRUE | 取料完成=TRUE | 步骤5 | ALM_201 (10秒) |

### 5.3 安全区管理逻辑

```
安全区管理机制:

目的: 防止取放料机构和打胶机同时在工作区域造成机械碰撞
原则: 取放料机构优先(它是主动方, 先完成放料后打胶机才能进来)

信号流向:
┌──────────────┐                    ┌──────────────────┐
│ 取放料机构    │──i_取放料_放料完成─>│                  │
│ (上游设备)    │                    │  FB_1004          │
└──────────────┘                    │  安全区判断逻辑:   │
                                    │  IF 放料完成 THEN  │
│<──o_安全区信号──│  o_安全区:=TRUE │
│                  │  ELSE           │
│    打胶机       │  o_安全区:=FALSE │
│  (外部设备)     │                  │
│  根据此信号决定 │                  │
│  是否进入工作区 │                  │
└──────────────────┘                  └──────────────────┘

时序说明:
1. 取放料机构工作中: i_取放料_放料完成=FALSE -> o_安全区信号=FALSE
   -> 打胶机禁止进入工作区

2. 取放料机构完成放料并移开: i_取放料_放料完成=TRUE -> o_安全区信号=TRUE
   -> 打胶机可以进入工作区

3. 本功能块步骤1会等待此信号为TRUE后才继续后续动作
   -> 保证不会在取放料机构还在工作时就开始移动X2轴
```

## 6. 报警机制设计

### 6.1 报警码定义 (201~299段)

| 报警码 | 含义 | 触发条件 | 触发步骤 | 清除条件 | 优先级 |
|--------|------|----------|----------|----------|--------|
| 0 | 无报警 | - | - | - | - |
| 201 | 通讯超时(打胶机无响应) | 打胶机未在规定时间(10s)内响应 | 步骤3/步骤4 | 复位按钮 | 高 |
| 202 | 安全区冲突(取放料未离开) | *(预留扩展)* 当前版本通过o_安全区信号体现 | - | - | 中 |
| 203 | X2轴伺服故障 | X2轴伺服驱动器ALM输出有效 | 所有步骤(持续检测) | 故障消除+复位 | 最高 |
| 204 | X2轴移动超时 | X2轴未在规定时间(5s)内到达目标位置 | 步骤2/步骤5 | 复位按钮 | 高 |
| 205 | 取料位置到达超时 | *(预留扩展)* 步骤2专用超时 | 步骤2 | 复位按钮 | 中 |

### 6.2 报警优先级说明

**优先级规则**: 数值越小优先级越高

1. **伺服故障(203)** - 最高优先级
   - 会立即停止X2轴的运动请求
   - 需要排除驱动器故障后复位

2. **通讯超时(201)** - 次高优先级
   - 保护打胶机交互流程不死等
   - 可能是打胶机侧问题或接线问题

3. **移动超时(204)** - 中等优先级
   - 保护X2轴不会一直运动
   - 可能是机械卡死或到位信号未反馈

### 6.3 报警处理流程

```
报警检测与处理流程:

状态机运行中
     │
     ├─ 持续检测: X2轴伺服故障(i_X2轴_伺服故障)
     │   └─ 若=TRUE → 立即触发ALM_203, 停止所有运动请求
     │
     ├─ 步骤2/5中: 检查X2轴是否到达目标位置
     │   └─ 超时(5s)未到达 → 触发ALM_204, 停止当前步骤
     │
     └─ 步骤3/4中: 检查打胶机响应
         └─ 超时(10s)无响应 → 触发ALM_201, 停止等待

报警激活后:
     │
     ├─ o_i本站报警代码 := 最高优先级活跃报警码
     ├─ o_b故障 := TRUE
     ├─ 状态机暂停在当前步骤
     └─ 所有输出保持当前状态(不归零, 由主控决定如何处理)

报警清除:
     │
     └─ 操作员按下 i_b复位 按钮
         ├─ 清除所有报警锁存标志
         ├─ o_i本站报警代码 := 0
         ├─ o_b故障 := FALSE
         └─ 状态机回到步骤0(空闲), 可重新启动
```

## 8. 手动模式设计

### 8.1 手动操作功能

在手动模式下(`i_b手动模式=TRUE`)，提供X2轴的点动控制：

| 手动操作 | 功能描述 | 互锁关系 | 安全限制 |
|---------|---------|----------|----------|
| `i_bX2轴点动前` | X2轴向前点动(去打胶机取料位置方向) | 与X2轴点动后互斥 | 受正向限位限制 |
| `i_bX2轴点动后` | X2轴向后点动(返回待机位置方向) | 与X2轴点动前互斥 | 受反向限位限制 |

### 8.2 手动模式约束

1. **生效条件**: 仅在`i_b手动模式=TRUE`且该站无故障时生效
2. **输出方式**: 通过内部请求标志(`s_bX2轴请求_*`)输出，由主控统一处理脉冲输出
3. **安全限制**: 受限位开关保护，防止超出行程范围
4. **模式切换**: 从手动切到自动时，必须先停止所有手动操作
