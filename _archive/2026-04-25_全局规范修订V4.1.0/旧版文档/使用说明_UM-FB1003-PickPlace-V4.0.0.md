## 2. 前置条件与依赖项

### 2.1 硬件环境要求

| 项目 | 要求 | 说明 |
|------|:----:|:-----|
| **PLC型号** | 支持ST语言的控制器（如西门子S7-1200/1500、三菱FX5U等） | 必须支持结构化文本编程 |
| **Z轴伺服系统** | 已完成参数配置和原点回归 | 包含驱动器、电机、限位开关(X3/X4)、原点传感器(X0) |
| **X1轴伺服系统** | 已完成参数配置和原点回归 | 包含驱动器、电机、限位开关(X5/X6)、原点传感器(X1) |
| **远程IO_1模块** | 已配置并通信正常 | X0~XF(输入), Y10~Y15(输出) |
| **升降气缸** | 已安装磁性开关(X0/X1对应远程IO) | 双作用气缸, 电磁阀YV1/YV2 |
| **前/后夹紧气缸** | 各2组, 共4个夹爪 | 每组有夹紧/松开电磁阀 + 动点/原点磁开关 |
| **产品检测传感器** | 4个光电传感器(SICK) | 安装于取料位置, 对应XC/XD/XE/XF |
| **HMI界面** | 已配置操作按钮和状态显示 | 启动/停止/复位、14个手动操作按钮、状态显示 |

### 2.2 软件依赖

| 依赖项 | 版本要求 | 用途 |
|--------|:-------:|:-----|
| 主控程序 (Main) | V4.0.0+ | IO映射、调用本FB、处理X1轴运动请求 |
| 公共报警FB | V4.0.0+ | 接收101~108段报警代码 |
| 四层输送机FB (FB_1001) | V4.0.0+ | 提供上游信号`o_Lx_放料完成[1..4]` |

### 2.3 必须连接的信号清单（最小化调用）

> 即使是测试，以下信号也**必须正确连接**，否则功能块无法正常工作：

#### 必接输入（不连则无法运行）

| 针脚名称 | 连接建议 | 原因 |
|---------|:---------|:-----|
| `i_b使能` | `gb系统就绪` | 总开关，FALSE时全部输出归零 |
| `i_b自动模式` | HMI M变量 | 模式选择必需 |
| `i_b手动模式` | HMI M变量 | 与自动模式互斥 |
| `i_b启动` | HMI启动按钮(R_TRIG边沿检测) | 触发11步状态机 |
| `i_b停止` | HMI停止按钮 | 安全停止必需 |
| `i_b复位` | HMI复位按钮 | 故障清除必需 |
| `i_Z轴_伺服故障` | IO映射(X11) | 最高优先级报警源(ALM_101) |
| `i_X1轴_伺服故障` | IO映射(X12) | 最高优先级报警源(ALM_102) |
| `i_升降_动点` | IO映射(远程IO_1:X0) | 步骤2/7转出条件 |
| `i_升降_原点` | IO映射(远程IO_1:X1) | 步骤5/9转出条件 |
| `i_前夹紧_动点` | IO映射(远程IO_1:X2) | 步骤3夹紧确认 |
| `i_后夹紧_动点` | IO映射(远程IO_1:X4) | 步骤3夹紧确认 |
| `i_长边1检测` | IO映射(远程IO_1:XC) | 步骤4产品检测AND判断 |
| `i_长边2检测` | IO映射(远程IO_1:XD) | 步骤4产品检测AND判断 |
| `i_短边1检测` | IO映射(远程IO_1:XE) | 步骤4产品检测AND判断 |
| `i_短边2检测` | IO映射(远程IO_1:XF) | 步骤4产品检测AND判断 |
| `i_输送机_L1_放料完成` | 四层输送机FB输出 | 步骤1转换条件(第1次取料) |
| `i_输送机_L2_放料完成` | 四层输送机FB输出 | 步骤1转换条件(第1次取料) |
| `i_输送机_L3_放料完成` | 四层输送机FB输出 | 步骤1转换条件(第2次取料) |
| `i_输送机_L4_放料完成` | 四层输送机FB输出 | 步骤1转换条件(第2次取料) |

#### 强烈建议连接（不连则功能受限）

| 针脚名称 | 连接建议 | 不连的影响 |
|---------|:---------|:----------|
| `i_Z轴_原点` | IO映射(CPU:X0) | 无法判断Z轴初始位置 |
| `i_X1轴_原点` | IO映射(CPU:X1) | 无法判断X1轴初始位置 |
| `i_Z轴正向/反向限位` | IO映射(X3/X4) | Z轴无限位保护 |
| `i_X1轴正向/反向限位` | IO映射(X5/X6) | X1轴无手动点动保护 |
| `i_r取料速度` ~ `i_i升降动作时间` | HMI参数设定(6个) | 使用内部默认值 |
| 14个手动操作信号 | HMI手动操作按钮 | 无法进行手动调试 |

## 3. 完整ST调用示例

### 3.1 生产环境完整调用（含52输入/18输出映射）

```st
(*==============================================================================
  主程序中调用 FB_1003_PickPlace_BufferFraming 的完整示例
  包含: 52个输入连接 / 18个输出连接 / 边沿检测 / X1轴运动处理
  ==============================================================================*)

PROGRAM Main
VAR
    (* ========== 功能块实例 ========== *)
    fb取放料机构 : FB_1003_PickPlace_BufferFraming;
    
    (* ========== 边沿检测 ========== *)
    rtrig_Start : R_TRIG;          (* 启动按钮上升沿检测 *)
    
    (* ========== X1轴运动处理(主控侧) ========== *)
    bX1AxisReqForward : BOOL;      (* X1轴前移请求 *)
    bX1AxisReqBackward : BOOL;     (* X1轴后移请求 *)
    
    (* ========== 报警上报 ========== *)
    iPickPlaceAlarmCode : INT;     (* 本站报警代码暂存 *)
END_VAR


(* ========================================================================== *)
(* =                          边沿检测                                      = *)
(* ========================================================================== *)
(* 重要! i_b启动必须用上升沿触发, 否则每次扫描都会重复触发启动             *)
rtrig_Start(CLK := di_M0_StartButton);


(* ========================================================================== *)
(* =                    调用取放料机构FB (52输入 / 18输出)                   =*)
(* ========================================================================== *)
fb取放料机构(
    (*=======================================================================*)
    (*=                        输入连接 (52个)                              =*)
    (*=======================================================================*)

    (*--- 一、系统控制 (3个) ---*)
    i_b使能               := NOT(di_EmergencyStop OR di_SafeDoorOpen),
    (* 说明: 急停解除且安全门关闭后才允许运行                              *)
    
    i_b自动模式           := mb_M5_AutoMode,
    i_b手动模式           := mb_M6_ManualMode,

    (*--- 二、操作按钮 (3个, 来自HMI) ---*)
    i_b启动               := rtrig_Start.Q,
    (* 注意! 必须使用R_TRIG做上升沿检测                                    *)
    
    i_b停止               := di_M1_StopButton OR di_EmergencyStop,
    (* 停止按钮或急停都可触发停止                                           *)
    
    i_b复位               := di_M2_ResetButton,

    (*--- 三、手动操作-伺服点动 (4个, 来自HMI) ---*)
    i_bZ轴点动上          := mb_M7_ZJogUp,
    i_bZ轴点动下          := mb_M8_ZJogDown,
    i_bX1轴点动前         := mb_M9_X1JogFwd,
    i_bX1轴点动后         := mb_M10_X1JogBwd,

    (*--- 四、手动操作-气缸 (10个, 来自HMI) ---*)
    i_b升降上升           := mb_M11_LiftUp,
    i_b升降下降           := mb_M12_LiftDown,
    i_b前夹紧夹紧         := mb_M13_FrontGripClose,
    i_b前夹紧松开         := mb_M14_FrontGripOpen,
    i_b后夹紧夹紧         := mb_M15_RearGripClose,
    i_b后夹紧松开         := mb_M16_RearGripOpen,
    i_b前夹紧2夹紧        := mb_M17_Front2GripClose,   (* 预留 *)
    i_b前夹紧2松开        := mb_M18_Front2GripOpen,    (* 预留 *)
    i_b后夹紧2夹紧        := mb_M19_Rear2GripClose,    (* 预留 *)
    i_b后夹紧2松开        := mb_M20_Rear2GripOpen,     (* 预留 *)

    (*--- 五、工艺参数 (6个, 来自HMI设定) ---*)
    i_r取料速度           := md_D30_PickSpeed,         (* mm/s, 默认100 *)
    i_r放料速度           := md_D31_PlaceSpeed,        (* mm/s, 默认80  *)
    i_rZ轴速度            := md_D32_ZAxisSpeed,        (* mm/s, 默认100 *)
    i_rX1轴速度           := md_D33_X1AxisSpeed,       (* mm/s, 默认150 *)
    i_i夹紧确认时间       := mi_D34_GripConfirmTime,   (* ms,   默认800 *)
    i_i升降动作时间       := mi_D35_LiftActionTime,    (* ms,   默认2000*)

    (*--- 六、传感器-气缸位置 (10个, 来自IO映射, 远程IO_1) ---*)
    i_升降_动点           := di_Lift_DownSensor,       (* 远程IO_1: X0 *)
    i_升降_原点           := di_Lift_UpSensor,         (* 远程IO_1: X1 *)
    i_前夹紧_动点         := di_FrontGrip_CloseSen,    (* 远程IO_1: X2 *)
    i_前夹紧_原点         := di_FrontGrip_OpenSen,     (* 远程IO_1: X3 *)
    i_后夹紧_动点         := di_RearGrip_CloseSen,     (* 远程IO_1: X4 *)
    i_后夹紧_原点         := di_RearGrip_OpenSen,      (* 远程IO_1: X5 *)
    i_前夹紧2_动点        := di_Front2Grip_CloseSen,   (* 远程IO_1: X6, 预留 *)
    i_前夹紧2_原点        := di_Front2Grip_OpenSen,    (* 远程IO_1: X7, 预留 *)
    i_后夹紧2_动点        := di_Rear2Grip_CloseSen,    (* 远程IO_1: X8, 预留 *)
    i_后夹紧2_原点        := di_Rear2Grip_OpenSen,     (* 远程IO_1: X9, 预留 *)

    (*--- 七、传感器-产品检测 (4个, 来自IO映射, 远程IO_1) ---*)
    i_长边1检测           := di_LongEdge1_Detect,      (* 远程IO_1: XC, SICK *)
    i_长边2检测           := di_LongEdge2_Detect,      (* 远程IO_1: XD, SICK *)
    i_短边1检测           := di_ShortEdge1_Detect,     (* 远程IO_1: XE, SICK *)
    i_短边2检测           := di_ShortEdge2_Detect,     (* 远程IO_1: XF, SICK *)

    (*--- 八、传感器-伺服轴状态 (12个, 来自CPU本体DI) ---*)
    i_Z轴_原点            := di_Z_HomeSwitch,          (* CPU: X0 *)
    i_X1轴_原点           := di_X1_HomeSwitch,         (* CPU: X1 *)
    i_X2轴_原点           := di_X2_HomeSwitch,         (* CPU: X2, 预留 *)
    i_Z轴_伺服故障        := di_Z_ServoAlarm,          (* CPU: X11 *)
    i_X1轴_伺服故障       := di_X1_ServoAlarm,         (* CPU: X12 *)
    i_X2轴_伺服故障       := di_X2_ServoAlarm,         (* CPU: X13, 预留 *)
    i_Z轴_正向限位        := di_Z_FwdLimit,            (* CPU: X3, 上限位 *)
    i_Z轴_反向限位        := di_Z_BwdLimit,            (* CPU: X4, 下限位 *)
    i_X1轴_正向限位       := di_X1_FwdLimit,           (* CPU: X5, 取料侧 *)
    i_X1轴_反向限位       := di_X1_BwdLimit,           (* CPU: X6, 放料侧 *)
    i_X2轴_正向限位       := di_X2_FwdLimit,           (* CPU: X7, 预留 *)
    i_X2轴_反向限位       := di_X2_BwdLimit,           (* CPU: X10, 预留 *)

    (*--- 九、上游信号 (4个, 来自四层输送机 FB_1001) ---*)
    i_输送机_L1_放料完成  := fb四层输送机.o_Lx_放料完成[1],
    (* 第1层输送机放料完成, 第1次取料时等待                                   *)
    
    i_输送机_L2_放料完成  := fb四层输送机.o_Lx_放料完成[2],
    (* 第2层输送机放料完成, 第1次取料时等待(L1+L2 AND)                       *)
    
    i_输送机_L3_放料完成  := fb四层输送机.o_Lx_放料完成[3],
    (* 第3层输送机放料完成, 第2次取料时等待                                   *)
    
    i_输送机_L4_放料完成  := fb四层输送机.o_Lx_放料完成[4],
    (* 第4层输送机放料完成, 第2次取料时等待(L3+L4 AND)                       *)

    (*=======================================================================*)
    (*=                        输出连接 (18个)                               =*)
    (*=======================================================================*)

    (*--- 执行器请求-气缸 -> 映射到远程IO_1 Y10~Y15 ---*)
    o_升降_上升           => do_Y10_LiftUp,           (* Y10 / YV1 *)
    o_升降_下降           => do_Y11_LiftDown,         (* Y11 / YV2 *)
    o_前夹紧_夹紧         => do_Y12_FrontGripClose,   (* Y12 / YV3 *)
    o_前夹紧_松开         => do_Y13_FrontGripOpen,    (* Y13 / YV4 *)
    o_后夹紧_夹紧         => do_Y14_RearGripClose,    (* Y14 / YV5 *)
    o_后夹紧_松开         => do_Y15_RearGripOpen,     (* Y15 / YV6 *)

    (*--- 执行器请求-伺服(Z轴) -> 映射到CPU Y0/Y4/Y10 ---*)
    o_Z轴_脉冲输出        => do_Y0_ZPulse,            (* CPU: Y0, PULSE+ *)
    o_Z轴_方向输出        => do_Y4_ZDirection,        (* CPU: Y4, SIGN+, FALSE=上升 *)
    o_Z轴_伺服使能        => do_Y10_ZServoEnable,     (* CPU: Y10 via KA10, SON *)

    (*--- 下游信号 -> 给打胶机送料机构(FB_1004) ---*)
    o_放料完成_给送料机构 => go_PickPlaceDoneToFeeder,

    (*--- 状态/HMI显示 ---*)
    o_b运行中             => mb_M21_PickPlaceRunning,
    o_b故障               => mb_M22_PickPlaceFault,
    o_i当前状态           => mi_PickPlaceStepCode,     (* 0~10, 99 *)
    o_rZ轴当前位置        => mr_ZAxisPosition,
    o_rX1轴当前位置       => mr_X1AxisPosition,
    o_rX2轴当前位置       => mr_X2AxisPosition,        (* 预留 *)
    o_i当前取料层数       => mi_CurrentLayerCount,     (* 0/1/3 *)

    (*--- 报警 -> 给公共报警管理器 ---*)
    o_i本站报警代码       => iPickPlaceAlarmCode        (* 101~108 *)
);


(* ========================================================================== *)
(* =                    主控侧: X1轴运动请求处理                             =*)
(* =  重要: X1轴的脉冲输出不由FB_1003直接输出, 由主控统一处理              =*)
(* ========================================================================== *)

(* 读取内部请求标志 *)
bX1AxisReqForward := fb取放料机构.s_bX1轴请求前移;
bX1AxisReqBackward := fb取放料机构.s_bX1轴请求后移;

(* 根据请求标志处理X1轴运动 *)
IF bX1AxisReqBackward THEN
    (* 步骤6: 移动到放料位置 -> X1轴向放料方向(后)移动 *)
    (* 调用X1轴伺服定位指令 - 具体指令取决于PLC品牌 *)
    MC_MoveAbsolute(
        Axis    := Axis_X1,
        Position := mr_X1PlacePosition,     (* 放料位置坐标 *)
        Velocity := fb取放料机构.i_rX1轴速度,
        Execute := TRUE
    );
    
ELSIF bX1AxisReqForward THEN
    (* 手动模式: X1轴向取料方向(前)移动 *)
    MC_MoveAbsolute(
        Axis    := Axis_X1,
        Position := mr_X1PickPosition,      (* 取料位置坐标 *)
        Velocity := fb取放料机构.i_rX1轴速度,
        Execute := TRUE
    );
    
ELSE
    (* 无请求时停止脉冲输出 *)
    MC_Stop(Axis := Axis_X1);
END_IF;


(* ========================================================================== *)
(* =                    主控侧: X1轴到位反馈给本FB                           =*)
(* =  当X1轴到达目标位置时, 需要通知本FB(通过设置内部变量或步序判断)       =*)
(* ========================================================================== *)
(* 方案A: 通过实例访问设置内部标志(如果暴露了的话)
IF Axis_X1.StatusDone THEN
    IF ABS(Axis_X1.ActPosition - mr_X1PlacePosition) < 1.0 THEN
        fb取放料机构.s_bX1轴在放料位 := TRUE;   (* 反馈给步骤6 *)
    ELSE
        fb取放料机构.s_bX1轴在放料位 := FALSE;
    END_IF;
ELSE
    fb取放料机构.s_bX1轴在放料位 := FALSE;
END_IF;
*)

(* 方案B(推荐): 通过步序号间接判断 - 主控在步骤6时持续发送脉冲直到到位
   步骤6会保持s_bX1轴请求后移=TRUE直到收到到位反馈或超时
   具体实现取决于FB内部的到位判断逻辑
*)


(* ========================================================================== *)
(* =                    报警上报到全局系统                                  =*)
(* ========================================================================== *)
IF iPickPlaceAlarmCode <> 0 THEN
    (* 上报到公共报警管理器 *)
    fbCommonAlarm.i_iStationID := 3;           (* 工站编号3=取放料机构 *)
    fbCommonAlarm.i_iAlarmCode := iPickPlaceAlarmCode;  (* 101~108 *)
END_IF;


(* ========================================================================== *)
(* =                    HMI状态更新                                         =*)
(* ========================================================================== *)
mb_M23_PickPlaceStepDisplay := fb取放料机构.o_i当前状态;
mr_D36_ZPosDisplay := fb取放料机构.o_rZ轴当前位置;
mr_D37_X1PosDisplay := fb取放料机构.o_rX1轴当前位置;
mi_D38_LayerDisplay := fb取放料机构.o_i当前取料层数;
```

(* 目的: 验证11步状态机和基本接口连通性          *)
(* ============================================ *)

VAR

    bX1Bwd : BOOL;
    iAlm : INT;

rtrig_Start(CLK := gbStartButton);

fbPP(
    (* 系统控制 *)
    i_b使能     := gbSystemReady,
    i_b自动模式 := gbAutoMode,
    i_b手动模式 := gbManualMode,

    i_b复位     := gbResetButton,

    i_b前夹紧夹紧 := FALSE, i_b前夹紧松开 := FALSE,
    i_b后夹紧夹紧 := FALSE, i_b后夹紧松开 := FALSE,
    i_b前夹紧2夹紧 := FALSE, i_b前夹紧2松开 := FALSE,
    i_b后夹紧2夹紧 := FALSE, i_b后夹紧2松开 := FALSE,
    
    (* 工艺参数 - 使用默认值(全0则用内部常量) *)

## 4. X1轴运动请求处理详解（主控侧）

### 4.1 设计原则

**重要**: 本功能块**不直接输出X1轴伺服脉冲**，而是通过内部请求标志通知主控程序，由主控统一处理所有伺服运动。

```
设计理由:
┌─────────────────┐    s_bX1轴请求_*    ┌─────────────────┐
│  FB_1003        │ ──────────────────> │  主控程序(Main)  │
│  (纯逻辑)       │                     │                 │
│  决定"何时移动"  │                     │  处理"如何移动"   │
│  和"移动方向"    │                     │  (脉冲输出/定位)  │
└─────────────────┘                     └─────────────────┘

与Z轴的区别:
  Z轴: FB直接输出 o_Z轴_脉冲输出 + o_Z轴_方向输出 (简单, 单轴独占)
  X1轴: FB只输出请求标志, 由主控处理 (复杂, 可能多FB共享/需协调)
```

### 4.2 内部请求标志说明

| 变量名 | 类型 | 含义 | 有效时机 | 触发来源 |
|:-------|:----:|:-----|:--------:|:--------|
| `s_bX1轴请求前移` | BOOL | 请求X1轴向取料方向移动(正向) | 手动模式下`i_bX1轴点动前`=TRUE | 手动操作 |
| `s_bX1轴请求后移` | BOOL | 请求X1轴向放料方向移动(反向) | **步骤6**期间为TRUE | 自动状态机 |

**访问方式**: 在主控中通过实例名访问：
```st
bX1ForwardReq := fb取放料机构.s_bX1轴请求前移;
bX1BackwardReq := fb取放料机构.s_bX1轴请求后移;
```

### 4.3 完整X1轴处理示例代码

```st
(* ========================================================================== *)
(* =              主控程序中X1轴运动的完整处理逻辑                            =*)
(* ========================================================================== *)

(* 1. 读取请求标志 *)
bX1ReqFwd := fb取放料机构.s_bX1轴请求前移;
bX1ReqBwd := fb取放料机构.s_bX1轴请求后移;

(* 2. 根据请求执行X1轴运动 *)
IF bX1ReqBwd THEN
    (* ====== 步骤6: X1轴后移到放料位置 ====== *)
    
    (* 判断是否已在运动中, 避免重复启动 *)
    IF NOT bX1Moving THEN
        (* 启动X1轴定位运动到放料位置 *)
        MC_MoveAbsolute(
            Axis     := Axis_X1,
            Position := mr_X1PlacePositionCoord,   (* 放料位置坐标(mm) *)
            Velocity := fb取放料机构.i_rX1轴速度,   (* 使用FB设定的速度 *)
            Acceleration := 1000,                   (* 加速度, 根据实际调整 *)
            Deceleration := 1000,                   (* 减速度 *)
            Execute  := TRUE
        );
        bX1Moving := TRUE;
    END_IF;
    
    (* 检查是否到达目标位置 *)
    IF Axis_X1.StatusDone AND 
       ABS(Axis_X1.ActPosition - mr_X1PlacePositionCoord) < 1.0 THEN
        (* 到达放料位置 - 通知本FB(如果支持内部反馈标志) *)
        (* fb取放料机构.s_bX1轴在放料位 := TRUE;  *)  (* 方案A: 直接设置 *)
        
        (* 或者: 保持请求为TRUE, FB通过其他方式判断到位 *)  (* 方案B: 推荐 *)
        
        bX1Moving := FALSE;
    END_IF;
    
ELSIF bX1ReqFwd THEN
    (* ====== 手动模式: X1轴前移到取料位置 ====== *)
    
    IF NOT bX1Moving THEN
        MC_MoveAbsolute(
            Axis     := Axis_X1,
            Position := mr_X1PickPositionCoord,    (* 取料位置坐标(mm) *)
            Velocity := fb取放料机构.i_rX1轴速度,
            Acceleration := 1000,
            Deceleration := 1000,
            Execute  := TRUE
        );
        bX1Moving := TRUE;
    END_IF;
    
    (* 手动模式下到位后不需要特别通知FB, 保持点动即可 *)
    IF NOT bX1ReqFwd THEN
        (* 按钮释放, 停止运动 *)
        MC_Stop(Axis := Axis_X1);
        bX1Moving := FALSE;
    END_IF;
    
ELSE
    (* ====== 无请求: 停止X1轴运动 ====== *)
    IF bX1Moving THEN
        MC_Stop(Axis := Axis_X1);
        bX1Moving := FALSE;
    END_IF;
END_IF;
```

### 4.4 步骤6的超时保护配合

```
步骤6的X1轴运动超时保护机制:

fb取放料机构(步骤6)          主控程序                X1轴伺服
     │                          │                      │
     │── s_bX1轴请求后移=ON ───>│                      │
     │                          │── 启动定位运动 ───────>│
     │                          │                      │
     │   (同时启动内部超时定时器)│                      │
     │   T_DEF_X1轴移动超时=5s  │                      │
     │                          │                      │
     │<── 5秒内未收到到位反馈 ──│<── 运动中... ─────────│
     │                          │                      │
     │ [超时! 触发ALM_108]      │                      │
     │ s_bX1轴请求后移:=OFF     │                      │
     │ o_i本站报警代码:=108     │                      │
     │ s_i步序:=99(故障)        │── 收到停止(可选) ────>│
     │                          │                      │

注意:
- 如果主控在5秒内让X1轴到达目标并反馈给FB, 则不会触发ALM_108
- 如果X1轴因机械卡死等原因无法到达, 5秒后FB自动报警
- 超时后主控应检测到请求变为FALSE, 停止X1轴运动
```

## 7. 双夹爪调试要点

### 7.1 双夹爪机械结构理解

```
双夹爪结构示意:

          取料位置俯视图:
          
     ┌─────────────────────────────┐
     │        边框缓存机工作台      │
     │                             │
     │  ┌─────┐          ┌─────┐  │
     │  │ L1   │          │ L2   │  │  ← 第1、2层边框
     │  │(前夹爪)│          │(后夹爪)│  │
     │  └──┬──┘          └──┬──┘  │
     │     │  前夹紧气缸    │ 后夹紧气缸
     │  ┌─────┐          ┌─────┐  │
     │  │ L3   │          │ L4   │  │  ← 第3、4层边框
     │  │(前夹爪)│          │(后夹爪)│  │
     │  └─────┘          └─────┘  │
     │                             │
     └─────────────────────────────┘

关键点:
- 前夹爪负责抓取奇数层(L1, L3)的边框
- 后夹爪负责抓取偶数层(L2, L4)的边框
- 两个夹爪同时动作, 一次取两根!
- 如果只有一个夹爪动作, 只能取一根边框 → 产品检测会失败(ALM_107)
```

### 7.2 双夹爪调试步骤

```

阶段1: 单独测试前夹爪 [预计15分钟]

│  │   └─ 观察: Y12(前夹紧_夹紧)是否输出? 气缸是否动作?

│  │   └─ 观察: Y13(前夹紧_松开)是否输出? 气缸是否复位?
│  └─ Step 1.4: 检查前夹紧原点信号(i_前夹紧_原点=X3)
│      └─ 松开到位后X3应为TRUE
│

│  ├─ (同阶段1, 测试后夹紧的夹紧/松开和磁开关)
│  └─ 确认Y14/Y15输出和X4/X5信号正确

阶段3: 双夹爪同步测试 [预计15分钟]
│  ├─ Step 3.1: 进入步骤3(自动模式下的夹紧步骤)
│  │   └─ 观察: Y12和Y14是否**同时**输出TRUE?
│  ├─ Step 3.2: 检查两个夹爪是否**同时**夹紧
│  │   └─ 时间差应<100ms(同一扫描周期内输出)
│  ├─ Step 3.3: 进入步骤8(松开步骤)
│  │   └─ 观察: Y13和Y15是否**同时**输出TRUE?
│  └─ Step 3.4: 检查两个夹爪是否**同时**松开
│
阶段4: 带产品验证 [预计20分钟]
   ├─ Step 4.1: 放置两根边框在取料位置(L1+L2位置)
   ├─ Step 4.2: 执行自动取料, 观察双夹爪同时夹紧
   ├─ Step 4.3: 检查4个产品检测传感器(应全为TRUE)
   │   └─ 任一为FALSE都会触发ALM_107
   └─ Step 4.4: 确认两根边框都被可靠抓取
```

### 7.3 双夹爪常见问题

| 现象 | 可能原因 | 解决方法 |
|:-----|:---------|:---------|
| 只有前夹爪动作, 后夹爪不动 | 后夹紧电磁阀(YV5)接线错误或线圈烧毁 | 检查Y14输出和YV5接线 |
| 夹紧不同步(时间差>500ms) | 气路气压不平衡或气管长度差异太大 | 调整气压平衡阀, 缩短气管差 |
| 一个夹爪夹不紧 | 该夹爪气缸漏气或磁开关位置偏移 | 检查气缸密封性和调整磁开关 |
| 产品检测总是失败(ALM_107) | 其中一个夹爪没有真正夹到边框 | 检查夹爪位置是否对准边框 |
| 松开后边框掉落 | 松开动作太快或夹爪磨损 | 调整松开速度或更换夹爪垫片 |

## 8. 参数调优建议

### 8.1 关键参数推荐值

| 参数名 | 默认值 | 推荐范围 | 调优依据 | 影响因素 |
|:-------|:------:|:--------:|:---------|:---------|
| `i_i升降动作时间` | 2000ms | 1500~3000ms | 升降行程和气压 | 行程长/气压低 → 加大 |
| `i_i夹紧确认时间` | 800ms | 500~1500ms | 夹紧行程和气压 | 确保可靠夹紧 |
| `i_r取料速度` | 100.0 mm/s | 80~200 mm/s | 取料定位精度 | 过快影响精度 |
| `i_r放料速度` | 80.0 mm/s | 60~150 mm/s | 放料平稳性 | 过快导致冲击 |
| `i_rZ轴速度` | 100.0 mm/s | 100~200 mm/s | Z轴通用速度 | 兼顾效率和安全 |
| `i_rX1轴速度` | 150.0 mm/s | 150~300 mm/s | X1轴水平速度 | 行程较长可加快 |

### 8.2 超时时间调优方法

**升降动作时间 (i_i升降动作时间)**:
```
计算公式:
超时时间 = 正常动作时间 x 安全系数(1.5~2.0)

示例:
如果升降气缸正常动作时间约1.2秒:
  推荐超时时间 = 1200ms x 1.5 = 1800ms (可设为2000ms)
  
如果经常出现ALM_103/104误报警:
  可适当增加到 2500~3000ms
  
如果需要快速故障检测(节拍敏感):
  可缩短到 1500ms (但需确保机械状态良好)
```

**夹紧确认时间 (i_i夹紧确认时间)**:
```
计算公式:
超时时间 = 正常夹紧时间 x 安全系数(1.5~2.0)

示例:
如果夹爪正常夹紧时间约400ms:
  推荐超时时间 = 400ms x 2.0 = 800ms (默认值)
  
注意: 此参数同时用于夹紧(步骤3)和松开(步骤8)!
如果夹紧和松开时间差异较大, 可能需要在代码中分开设置
```

## 10. 常见问题排查指南

### 10.1 问题诊断表 (9种典型故障)

| 序号 | 现象 | 可能原因 | 解决方案 | 相关报警 |
|:----:|:-----|:---------|:---------|:--------|
| 1 | **启动后卡在步骤1** | 输送机未发送L1+L2(或L3+L4)放料完成信号 | 检查四层输送机运行状态和信号连线 | - |
| 2 | **升降下降报警(ALM_103)** | 气缸动作太慢、气压不足(<0.4MPa)、传感器松动或接线不良 | 加大`i_i升降动作时间`; 检查气压和传感器 | 103 |
| 3 | **产品检测失败(ALM_107)** | 边框位置偏移、传感器脏污/偏移、夹爪未真正夹紧、某传感器损坏 | 调整边框位置; 清洁传感器; 检查夹爪对中度; 用真值表分析哪个传感器为FALSE | 107 |
| 4 | **夹紧超时(ALM_105)** | 夹紧气缸漏气、磁开关位置偏移(离夹紧位太远)、气压不足 | 检查气缸密封性; 调整磁开关位置 closer to 夹紧位; 加大`i_i夹紧确认时间` | 105 |
| 5 | **松开超时(ALM_106)** | 与夹紧超时类似, 但发生在松开方向 | 检查松开电磁阀和气缸复位弹簧; 调整松开侧磁开关位置 | 106 |
| 6 | **移动超时(ALM_108)** | X1轴伺服参数不当(速度/加速度太小)、机械卡死、到位反馈未正确设置 | 检查伺服参数; 手动摇动X1轴确认无卡死; 检查主控到位判断逻辑 | 108 |
| 7 | **伺服故障(101/102)** | 驱动器过载/过热/过压、编码器异常、通信丢失、电源缺相 | 查阅驱动器手册排除故障; 检查电机电缆和编码器线; 确认电源电压稳定 | 101/102 |
| 8 | **手动操作无效** | 未选手动模式(`i_b手动模式`=FALSE); 存在未清除故障(`o_b故障`=TRUE); 限位已触发 | 切换到手动模式; 先按复位清除故障; 手动摇离限位点 | - |
| 9 | **4层循环不正常** | 启动信号未做边沿检测(每扫描周期重复触发或无法触发); `i_b启动`信号不稳定 | 使用R_TRIG检测上升沿; 检查HMI按钮配置; 用示波器观察启动信号 | - |

## 11. 使用约束与禁忌

### 11.1 禁止事项

| 序号 | 禁止操作 | 后果 | 正确做法 |
|:----:|:--------|:-----|:--------|
| 1 | **同时设置`i_b自动模式`和`i_b手动模式`为TRUE** | 功能块进入不确定状态，可能全部输出归零 | 确保两者互斥（HMI应做联锁） |
| 2 | **在`i_b使能`=FALSE时尝试启动** | 启动信号被忽略，状态机不会离开步骤0 | 先确保系统就绪条件满足 |
| 3 | **忽略ALM_101/102(伺服故障)继续运行** | 可能损坏伺服驱动器或造成机械碰撞 | 必须先排除伺服故障并复位 |
| 4 | **手动修改内部变量(`s_*`)绕过状态机** | 破坏11步状态机逻辑和4层循环一致性 | 只通过标准输入接口操作 |
| 5 | **在步骤3/8(夹爪动作期间)突然切断`i_b使能`** | 夹爪可能保持当前状态(夹紧!), 造成安全隐患 | 应先按停止按钮让状态机正常结束到安全态 |
| 6 | **将上游信号硬接线为常TRUE** | 失去真实的层间协调, 可能在边框未准备好时就尝试取料 | 必须由四层输送机真实状态决定 |
| 7 | **不处理X1轴请求标志就调用本FB** | 步骤6会永远等待X1轴到位, 最终触发ALM_108 | 主控必须实现X1轴运动处理逻辑 |

### 11.2 注意事项

| 序号 | 注意事项 | 说明 | 建议 |
|:----:|:--------|:-----|:-----|
| 1 | **启动信号必须做边沿检测** | 否则每个扫描周期都会重复触发启动(或无法触发) | 使用R_TRIG指令检测上升沿 |
| 2 | **上游信号的稳定性** | L1~L4放料完成信号跳变会导致步骤1条件判断不稳定 | 上游应保证至少200ms稳定 |
| 3 | **X1轴运动必须由主控处理** | 本FB只输出请求标志, 不直接输出脉冲 | 不要在本FB内写X1轴脉冲输出代码 |
| 4 | **报警代码需上报全局系统** | `o_i本站报警代码`要连接到公共报警FB(101~108段) | 否则HMI看不到此站的报警 |
| 5 | **复位后需等待初始化完成** | 初始化最多需要10秒(伺服检查) | HMI应有"初始化中..."提示 |
| 6 | **4层循环的断点续接** | 中途停止后再次启动会从断点继续(不会从头开始) | 这是设计特性, 不是bug |
| 7 | **双夹爪的同步性** | 前后夹紧应在同一扫描周期内同时输出 | 如果发现明显不同步, 检查气路平衡 |
| 8 | **产品检测的4传感器AND** | 只要1个传感器为FALSE就会报ALM_107 | 调试时可先用3传感器AND测试(改代码) |

### 11.3 推荐做法

| 序号 | 推荐做法 | 好处 |
|:----:|:--------|:-----|
| 1 | HMI增加"单层测试"模式 | 方便单独测试某一层的取放料而不必等4层 |
| 2 | 记录每个完整4层循环的周期时间 | 用于评估节拍是否符合产能要求(目标: <60秒/4层) |
| 3 | 在HMI显示4个产品检测传感器的实时状态 | 快速定位ALM_107的具体原因 |
| 4 | 为上下游信号增加状态指示灯 | 快速定位联调时的通讯问题 |
| 5 | 定期测试急停和安全门功能 | 确保安全系统始终可靠(特别是6个气缸和2个伺服) |
| 6 | 建立完整的报警日志系统 | 记录每次ALM_101~108的时间、原因、持续时间 |
| 7 | 为X1轴运动增加到位指示 | 方便调试时确认主控和FB之间的到位反馈 |

## 12. 扩展应用场景

### 12.1 从2层扩展到N层取料

如果未来工艺变更需要一次取N根边框（而非当前的2根），需要修改:

```st
(* 当前设计: 一次取2根, 4层循环2次 *)
(* 扩展为: 一次取N根, M层循环(M/N)次 *)

(* 需要修改的地方: *)
1. LAYER_TOTAL 常量: 4 → M (新的总层数)
2. STEP_PER_PICK 常量: 2 → N (每次取的层数)
3. 步骤1的条件判断: 分组逻辑从(L1+L2)/(L3+L4)改为动态分组
4. 步骤10的计数器递增: += 2 → += N
5. 夹爪数量: 2个 → N个 (新增N-2个夹紧气缸接口)
6. 产品检测传感器: 4个 → 2*N个 (每根边框2个检测)
7. IFC接口: 输入增加(N-2)*2个夹紧信号 + (2*N-4)个检测信号
```

### 12.2 与MES系统集成

可以将关键状态上传到MES系统:

```st
(* MES数据采集示例 *)
fbMES.i_Data[1]  := INT_TO_WORD(fb取放料机构.o_i当前状态);      (* 当前步序0~10,99 *)
fbMES.i_Data[2]  := INT_TO_WORD(fb取放料机构.o_i本站报警代码);  (* 报警代码101~108 *)
fbMES.i_Data[3]  := INT_TO_WORD(BOOL_TO_INT(fb取放料机构.o_b运行中)); (* 运行状态 *)
fbMES.i_Data[4]  := INT_TO_WORD(BOOL_TO_INT(fb取放料机构.o_b故障));    (* 故障状态 *)
fbMES.i_Data[5]  := INT_TO_WORD(fb取放料机构.o_i当前取料层数);     (* 层数进度0/1/3 *)
fbMES.r_Data[1]  := fb取放料机构.o_rZ轴当前位置;                 (* Z轴位置 *)
fbMES.r_Data[2]  := fb取放料机构.o_rX1轴当前位置;                (* X1轴位置 *)
```

### A.2 11步状态速查

```
状态机11步 + 故障态:

  0  = 空闲待机       [等待启动]
  1  = 等待放料完成   [L1+L2 或 L3+L4]
  2  = 取料下降       [Z↓+升降↓]     → ALM_103
  3  = 夹爪夹紧       [前后同时]      → ALM_105
  4  = 产品检测       [4传感器AND]    → ALM_107
  5  = 取料上升       [Z↑+升降↑]     → ALM_104
  6  = 移动到放料位   [X1后移请求]    → ALM_108
  7  = 放料下降       [Z↓+升降↓]     → ALM_103
  8  = 夹爪松开       [前后同时]      → ALM_106
  9  = 放料上升       [Z↑+升降↑]     → ALM_104
  10 = 放料完成通知   [500ms脉冲]
  99 = 故障           [有活跃报警]

循环: 10完成 -> 层数<4? -> 回1 : 回0
```

### A.3 报警速查

```
报警码 (101~108):

  101 = Z轴伺服故障  [最高优先! 立即停Z轴]
  102 = X1轴伺服故障  [最高优先! 立即停X1]
  103 = 升降下降超时  [步骤2/7, 检查气缸/气压]
  104 = 升降上升超时  [步骤5/9, 检查气缸/气压]
  105 = 夹紧超时      [步骤3, 检查夹紧气缸/磁开关]
  106 = 松开超时      [步骤8, 检查松开气缸/磁开关]
  107 = 产品检测失败  [步骤4, 检查4传感器+边框位置]
  108 = X1轴移动超时  [步骤6, 检查伺服/机械/主控处理]

处理: 排除原因 -> 按复位 -> 重新初始化
```
