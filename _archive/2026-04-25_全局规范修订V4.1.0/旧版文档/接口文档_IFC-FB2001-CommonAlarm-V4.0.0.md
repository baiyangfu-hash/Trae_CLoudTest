## 3. 输入接口定义 (VAR_INPUT)

### 3.1 工站报警代码输入组 (3个INT)

> 这三个输入分别来自三个工站功能块的`o_i本站报警代码`输出。本FB**不直接读取任何物理IO地址**，所有数据通过主控程序中转。

#### 3.1.1 i_i输送机报警

| 属性 | 值 |
|------|-----|
| **针脚名称** | `i_i输送机报警` |
| **数据类型** | INT |
| **功能描述** | 四层输送机的当前报警代码 |
| **来源** | `FB_1001_Conveyor4Layer_BufferFraming.o_i本站报警代码` |
| **取值范围** | 0（无报警）或 1~7（具体报警类型，见[报警码定义](./报警码定义_ALM-DJ-2026-005-V4.0.0.md#2-四层输送机报警码-1-7-段)） |
| **优先级段** | 高优先级（1~7为全系统最小编号范围） |
| **更新周期** | 每扫描周期（由主控从FB_1001读取后传入） |
| **默认行为** | 值为0时表示该工站无报警，不参与OR运算和优先级比较 |

**完整报警码列表**:

| 码值 | 名称 | 触发条件简述 |
|:----:|:-----|:------------|
| 0 | 无报警 | 正常状态 |
| 1 | 阻挡下降超时 | 阻挡气缸下降未到位 |
| 2 | 上升超时 | 阻挡气缸上升未到位 |
| 3 | 分料推出超时 | 分料气缸推出未到位 |
| 4 | 复位超时 | 复位后未完成初始化 |
| 5 | 输送超时 | 材料未在规定时间到达 |
| 6 | 传感器冲突 | 同一气缸上下位同时为TRUE |
| 7 | 变频器故障 | 变频器反馈ALM信号 |

### 3.2 操作控制输入组 (1个BOOL)

#### 3.2.1 i_b复位

| 属性 | 值 |
|------|-----|
| **针脚名称** | `i_b复位` |
| **数据类型** | BOOL |
| **功能描述** | HMI复位按钮信号（电平有效） |
| **有效电平** | TRUE = 执行复位操作 |
| **触发方式** | **电平有效**（按下期间持续执行复位） |
| **处理优先级** | **最高**（在本FB所有逻辑中最先执行，Step 2） |
| **来源** | HMI M区变量映射（操作员按下复位按钮） |
| **行为描述** | 清除所有历史报警、清空MES队列、重置所有内部状态和计数器 |

**复位后的完整动作序列**:
1. `o_w全局报警字 := 0`
2. `o_i当前报警代码 := 0`
3. `o_b任何报警激活 := FALSE`
4. `o_b新报警标志 := FALSE`
5. `o_iMES报警计数 := 0`
6. `s_i累计报警次数 := 0`
7. `o_iMES报警队列[0..9]` 全部归零
8. `s_i队尾索引 := 0`
9. 内部状态变量全部重置
10. **RETURN**（不处理本周期的报警输入）

**重要约束**: 复位操作仅清除本FB内部的报警管理状态。各工站FB的内部报警需要各自工站在下一周期自行检测到复位信号后清除。

## 4. 输出接口定义 (VAR_OUTPUT)

### 4.1 HMI显示输出组 (2个)

> 这些输出由**主控程序写入D区寄存器**，供HMI界面读取显示。

#### 4.1.1 o_w全局报警字

| 属性 | 值 |
|------|-----|
| **针脚名称** | `o_w全局报警字` |
| **数据类型** | WORD |
| **映射目标** | D400 |
| **功能描述** | 所有非零报警代码的位或(OR)组合结果 |
| **计算方式** | 对每个非零的INT报警码做`INT_TO_WORD()`后进行OR运算 |
| **有效值域** | 0x0000（无报警）或 非零WORD值（有报警） |
| **消费方** | HMI报警灯指示（非零=点亮报警灯） |
| **典型用法** | HMI通过判断`D400 <> 0`来控制报警灯的亮灭和闪烁 |

**运算规则**:

```st
(* 计算公式 *)
o_w全局报警字 = 0;
IF i_i输送机报警 <> 0 THEN
    o_w全局报警字 := o_w全局报警字 OR INT_TO_WORD(i_i输送机报警);
END_IF;
IF i_i取放料报警 <> 0 THEN
    o_w全局报警字 := o_w全局报警字 OR INT_TO_WORD(i_i取放料报警);
END_IF;
IF i_i送料机构报警 <> 0 THEN
    o_w全局报警字 := o_w全局报警字 OR INT_TO_WORD(i_i送料机构报警);
END_IF;
```

#### 4.1.2 o_i当前报警代码

| 属性 | 值 |
|------|-----|
| **针脚名称** | `o_i当前报警代码` |
| **数据类型** | INT |
| **映射目标** | D402 |
| **功能描述** | 当前所有活跃报警中数值最小的那个（即最高优先级报警码） |
| **选取算法** | 三段顺序比较法（详见[DSN文档第5.2节](./详细设计说明书_DSN-FB2001-CommonAlarm-V4.0.0.md#52-最高优先级选取算法)） |
| **取值范围** | 0（无报警）或 1~7 / 101~108 / 201~205 |
| **消费方** | HMI文本显示（根据此值查表显示对应报警描述文字） |
| **典型用法** | HMI用此值作为索引查找报警文本消息 |

**选取示例**:

| 输送机 | 取放料 | 送料机构 | o_i当前报警代码 | 显示含义 |
|:------:|:------:|:--------:|:--------------:|:---------|
| 3 | 0 | 0 | **3** | 分料推出超时 |
| 5 | 101 | 0 | **5** | 上升超时（5 < 101） |
| 0 | 105 | 201 | **105** | 夹紧超时（105 < 201） |
| 7 | 102 | 203 | **7** | 变频器故障（7最小） |
| 0 | 0 | 0 | **0** | 无报警 |

### 4.2 全局互锁输出组 (1个BOOL)

#### 4.2.1 o_b任何报警激活

| 属性 | 值 |
|------|-----|
| **针脚名称** | `o_b任何报警激活` |
| **数据类型** | BOOL |
| **映射目标** | M200 |
| **功能描述** | 全局报警激活标志（至少一个工站有报警时为TRUE） |
| **计算方式** | `o_b任何报警激活 := s_b输送机有报警 OR s_b取放料有报警 OR s_b送料机构有报警` |
| **有效电平** | TRUE = 存在报警（禁止自动运行），FALSE = 无报警（允许运行） |
| **消费方** | 主控自动运行逻辑（作为运行前置条件的否定条件） |
| **安全等级** | 高（直接影响设备安全运行） |

**使用场景**:
```st
(* 主控侧典型用法: 自动运行的前置条件检查 *)
IF NOT fbCommonAlarm.o_b任何报警激活 AND gbAutoMode THEN
    (* 允许执行自动运行逻辑 *)
END_IF;
```

### 4.3 MES数据接口输出组 (3个)

> 这些输出用于与MES制造执行系统的数据交互。主控负责将数据写入D区供MES采集。

#### 4.3.1 o_iMES报警计数

| 属性 | 值 |
|------|-----|
| **针脚名称** | `o_iMES报警计数` |
| **数据类型** | INT |
| **映射目标** | D404 |
| **功能描述** | 自上次复位以来的累计报警次数（含重复报警） |
| **累加时机** | 每个有报警的扫描周期都+1（不论是否去重成功） |
| **归零时机** | 仅在`i_b复位=TRUE`时归零 |
| **消费方** | MES质量统计分析模块 |
| **用途** | 统计设备稳定性指标（报警频率） |

**计数特性**:

| 场景 | 计数行为 |
|:-----|:--------|
| ALM_3持续存在10个扫描周期 | 计数+10（每个周期都计） |
| ALM_3去重不入队 | 仍计数+1（计数与去重独立） |
| 复位后重新出现ALM_3 | 从0开始重新累计 |
| 无报警期间 | 保持不变（不归零） |

#### 4.3.2 o_iMES报警队列

| 属性 | 值 |
|------|-----|
| **针脚名称** | `o_iMES报警队列` |
| **数据类型** | ARRAY[0..9] OF INT |
| **映射目标** | D406 ~ D425（每个INT占2个D地址，共20个D地址） |
| **功能描述** | 最近10条去重报警记录（循环缓冲区） |
| **存储格式** | 小端模式（三菱PLC标准） |
| **索引语义** | [0]=最新一条, [9]=最旧一条 |
| **空位标记** | 0表示该位置未被使用或已无效 |
| **消费方** | MES系统通过通信口批量读取后上传 |
| **去重策略** | 入队前遍历检查，相同报警码不重复入队 |

**D地址映射明细**:

| 队列索引 | D地址范围 | 含义 |
|:-------:|:--------:|:-----|
| [0] | D406, D407 | 最新一条去重报警码 |
| [1] | D408, D409 | 第2新报警码 |
| [2] | D410, D411 | 第3新报警码 |
| [3] | D412, D413 | 第4新报警码 |
| [4] | D414, D415 | 第5新报警码 |
| [5] | D416, D417 | 第6新报警码 |
| [6] | D418, D419 | 第7新报警码 |
| [7] | D420, D421 | 第8新报警码 |
| [8] | D422, D423 | 第9新报警码 |
| [9] | D424, D425 | 最旧一条报警码 |

**队列行为特性**:

| 特性 | 说明 |
|:-----|:-----|
| 写入策略 | 循环覆盖（队满时覆盖最旧记录） |
| 去重机制 | 相同报警码不重复入队 |
| 排序方式 | 按入队时间排序（非按优先级） |
| 容量固定 | 始终保持10个元素（不足则补0） |
| 清空条件 | 仅在复位时全部归零 |

#### 4.3.3 o_b新报警标志

| 属性 | 值 |
|------|-----|
| **针脚名称** | `o_b新报警标志` |
| **数据类型** | BOOL |
| **功能描述** | 新报警检测脉冲（单扫描周期有效） |
| **脉冲宽度** | 仅1个扫描周期为TRUE |
| **触发条件** | 本周期任一工站报警输入发生变化 且 当前有活跃报警 |
| **消费方** | 主控程序（检测到上升沿后立即执行MES上传） |
| **典型用法** | 主控用R_TRIG检测此信号的上升沿 |

**脉冲时序图**:
```
时间轴:    T1    T2      T3    T4    T5        T6
            │     │       │     │     │         │
报警输入:  无    出现ALM3  保持  保持  消失      出现ALM102
            │     │       │     │     │         │
新报警标志:      ┌──┐                         ┌──┐
                │██│                         │██│
                └──┘                         └──┘
                  ↑                             ↑
              仅T2为TRUE                    仅T6为TRUE
             (单周期脉冲)                   (单周期脉冲)
```

**检测算法详情**:

```st
(* 变化检测: 比较本周 vs 上周快照 *)
IF (i_i输送机报警   <> s_i上周输送机报警) OR
   (i_i取放料报警   <> s_i上周取放料报警) OR
   (i_i送料机构报警 <> s_i上周送料机构报警) THEN
    s_b检测到新报警 := TRUE;
END_IF;

(* 脉冲生成: 有变化且有实际报警时才输出TRUE *)
o_b新报警标志 := s_b检测到新报警 AND o_b任何报警激活;
```

### 4.4 多报警状态输出组 (4个)

> 这些输出提供更细粒度的报警状态信息，便于HMI分站显示和调试诊断。

#### 4.4.1 o_i工站报警状态

| 属性 | 值 |
|------|-----|
| **针脚名称** | `o_i工站报警状态` |
| **数据类型** | ARRAY[1..3] OF INT |
| **功能描述** | 各工站当前的原始报警代码（透传输入值） |
| **索引分配** | [1]=输送机, [2]=取放料, [3]=送料机构 |
| **用途** | HMI可同时显示所有工站的报警状态，不仅限于最高优先级 |

**数组元素说明**:

| 索引 | 元素名 | 来源 | 含义 |
