## 2. 前置条件与依赖项

### 2.1 硬件环境要求

| 项目 | 要求 | 说明 |
|------|:----:|:-----|
| **PLC型号** | 支持ST语言的控制器（如西门子S7-1200/1500、三菱FX5U等） | 必须支持结构化文本编程 |
| **X2轴伺服系统** | 已完成参数配置和原点回归 | 包含驱动器、电机、限位开关、原点传感器 |
| **打胶机接口** | X76(允许送料)、X102(取料完成)、Y44(允许抓料)、Y47(安全区) | 硬接线或通讯已连接 |
| **HMI界面** | 已配置操作按钮和状态显示 | 启动/停止/复位按钮、状态指示灯 |

### 2.2 软件依赖

| 依赖项 | 版本要求 | 用途 |
|--------|:-------:|:-----|
| 主控程序 (Main) | V4.0.0+ | IO映射和调用本FB |
| 公共报警FB | V4.0.0+ | 接收201~299段报警代码 |
| 取放料机构FB | V4.0.0+ | 提供上游信号`i_取放料_放料完成` |

### 2.3 必须连接的信号清单（最小化调用）

> 即使是测试，以下信号也**必须正确连接**，否则功能块无法正常工作：

#### 🔴 必须连接（不连则无法运行）

| 针脚名称 | 连接建议 | 原因 |
|---------|:---------|:-----|
| `i_b使能` | `gb系统就绪` | 总开关，FALSE时全部输出归零 |
| `i_b自动模式` | HMI M变量 | 模式选择必需 |
| `i_b手动模式` | HMI M变量 | 与自动模式互斥 |
| `i_b启动` | HMI启动按钮(边沿检测) | 触发自动循环 |
| `i_b停止` | HMI停止按钮 | 安全停止必需 |
| `i_b复位` | HMI复位按钮 | 故障清除必需 |
| `i_X2轴_伺服故障` | IO映射(X13) | 最高优先级报警源 |
| `i_打胶机允许送料` | IO映射(X76) | 步骤3转换条件 |
| `i_打胶机取料完成` | IO映射(X102) | 步骤4转换条件 |
| `i_取放料_放料完成` | 取放料FB输出 | 步骤1转换条件+安全区联动 |

#### 🟡 强烈建议连接（不连则功能受限）

| 针脚名称 | 连接建议 | 不连的影响 |
|---------|:---------|:----------|
| `i_X2轴_原点` | IO映射(X2) | 无法判断初始化状态 |
| `i_X2轴_正向限位` | IO映射(X7) | 手动点动无正向保护 |
| `i_X2轴_反向限位` | IO映射(X10) | 手动点动无反向保护 |
| `i_r送料速度` | HMI参数设定 | 速度显示为0 |
| `i_r待机位置` | HMI参数设定 | 位置显示为0 |
| `i_r取料位置` | HMI参数设定 | 位置显示为0 |
| `i_bX2轴点动前` | HMI手动按钮 | 无法手动前移X2轴 |
| `i_bX2轴点动后` | HMI手动按钮 | 无法手动后移X2轴 |

#### 🟢 可选连接（用于增强功能）

| 针脚名称 | 用途 |
|---------|:-----|
| `o_允许抓料` → Y44 | 映射到物理输出给打胶机 |
| `o_安全区信号` → Y47 | 映射到物理输出给打胶机 |
| `o_b运行中` → HMI | 显示运行状态 |
| `o_b故障` → HMI | 显示故障状态 |
| `o_i当前状态` → HMI | 显示步序码(0~5,99) |
| `o_i本站报警代码` → 报警管理器 | 上报全局报警系统 |

## 3. 快速开始指南

### 3.1 最小化调用示例（仅验证基本逻辑）

```st
(* ============================================ *)
(* FB_1004_GlueMachineFeeder 最小化调用示例      *)
(* 目的: 快速验证FB的基本功能和状态机运行        *)
(* ============================================ *)

VAR
    (* 功能块实例 *)
    fbGlueMachineFeeder : FB_1004_GlueMachineFeeder_BufferFraming;
    
    (* 边沿检测 - 启动按钮 *)
    rtrig_Start : R_TRIG;
    
    (* 内部请求标志 - X2轴运动 *)
    bX2AxisReqForward : BOOL;  (* X2轴前移请求 *)
    bX2AxisReqBackward : BOOL; (* X2轴后移请求 *)
END_VAR


(* ========== 边沿检测 ==========
   重要! i_b启动必须用上升沿触发,
   否则每次扫描都会重复触发启动 *)
rtrig_Start(CLK := gbStartButton);


(* ========== 调用功能块 ==========
   将所有输入连接到对应的IO或内部变量 *)
fbGlueMachineFeeder(
    (* --- 系统控制组 --- *)
    i_b使能 := gbSystemReady,           (* 系统全局就绪标志 *)
    i_b自动模式 := gbAutoMode,          (* 自动模式选择 *)
    i_b手动模式 := gbManualMode,        (* 手动模式选择 *)
    
    (* --- 操作按钮组 --- *)
    i_b启动 := rtrig_Start.Q,            (* 启动按钮上升沿 *)
    i_b停止 := gbStopButton,             (* 停止按钮电平 *)
    i_b复位 := gbResetButton,            (* 复位按钮电平 *)
    
    (* --- 手动操作组 --- *)
    i_bX2轴点动前 := gbJogForward,       (* X2轴向前点动 *)
    i_bX2轴点动后 := gbJogBackward,      (* X2轴向后点动 *)
    
    (* --- 工艺参数组 --- *)
    i_r送料速度 := grFeedSpeed,          (* X2轴速度设定(mm/s) *)
    i_r待机位置 := grStandbyPosition,    (* 待机位置坐标(mm) *)
    i_r取料位置 := grPickPosition,       (* 取料位置坐标(mm) *)
    
    (* --- 传感器输入组 --- *)
    i_X2轴_原点 := gdi_X2_Home,         (* X2轴原点信号 *)
    i_X2轴_正向限位 := gdi_X2_FwdLimit, (* X2轴正向限位 *)
    i_X2轴_反向限位 := gdi_X2_BwdLimit, (* X2轴反向限位 *)
    i_X2轴_伺服故障 := gdi_X2_ServoAlarm,(* X2轴伺服故障 *)
    
    (* --- 打胶机交互信号组 --- *)
    i_打胶机允许送料 := gdi_Glue_AllowFeed,  (* X76: 允许送料 *)
    i_打胶机取料完成 := gdi_Glue_PickDone,    (* X102: 取料完成 *)
    
    (* --- 上游信号组 --- *)
    i_取放料_放料完成 := fbPickPlace.o_放料完成_给送料机构, (* 取放料FB的输出 *)
    
    (* --- 输出接口 --- *)
    o_允许抓料 => gdq_Y44_AllowPick,     (* -> Y44: 允许抓料 *)
    o_安全区信号 => gdq_Y47_SafeZone,   (* -> Y47: 安全区信号 *)
    o_b运行中 => gbRunning,              (* 运行状态 *)
    o_b故障 => gbFault,                  (* 故障状态 *)
    o_i当前状态 => giCurrentStep,        (* 当前步序 *)
    o_rX2轴当前位置 => grX2Position,     (* X2轴位置反馈 *)
    o_i本站报警代码 => giAlarmCode       (* 报警代码201~299 *)
);

(* ========== 主控侧处理 ==========
   从FB获取内部请求标志并处理X2轴运动 *)
bX2AxisReqForward := fbGlueMachineFeeder.s_bX2轴请求前移;
bX2AxisReqBackward := fbGlueMachineFeeder.s_bX2轴请求后移;
```

### 3.2 完整生产环境调用示例

```st
(* ============================================ *)
(* 完整生产环境调用示例                           *)
(* 包含: IO映射、报警上报、HMI更新               *)
(* ============================================ *)

PROGRAM Main
VAR
    (* 功能块实例 *)
    fbGlueMachineFeeder : FB_1004_GlueMachineFeeder_BufferFraming;
    
    (* 边沿检测 *)
    rtrig_Start : R_TRIG;
    ftrig_Stop : F_TRIG;
    
    (* X2轴运动处理 *)
    bX2ReqFwd : BOOL;
    bX2ReqBwd : BOOL;
    
    (* 报警上报 *)
    iAlarmCode : INT;
END_VAR


(* ---------- 边沿检测 ---------- *)
rtrig_Start(CLK := di_M0_StartButton);
ftrig_Stop(CLK := di_M1_StopButton);


(* ---------- 调用打胶机送料机构FB ---------- *)
fbGlueMachineFeeder(
    (* 系统控制 *)
    i_b使能 := NOT(di_EmergencyStop OR di_SafeDoorOpen),
    i_b自动模式 := mb_M5_AutoMode,
    i_b手动模式 := mb_M6_ManualMode,
    
    (* 操作按钮 *)
    i_b启动 := rtrig_Start.Q,
    i_b停止 := ftrig_Stop.Q OR di_EmergencyStop,
    i_b复位 := di_M2_ResetButton,
    
    (* 手动操作 *)
    i_bX2轴点动前 := mb_M10_JogX2Fwd,
    i_bX2轴点动后 := mb_M11_JogX2Bwd,
    
    (* 工艺参数 *)
    i_r送料速度 := md_D20_FeedSpeed,
    i_r待机位置 := md_D21_StandbyPos,
    i_r取料位置 := md_D22_PickPos,
    
    (* 传感器 *)
    i_X2轴_原点 := di_X2_HomeSwitch,
    i_X2轴_正向限位 := di_X2_ForwardLimit,
    i_X2轴_反向限位 := di_X2_BackwardLimit,
    i_X2轴_伺服故障 := di_X2_ServoAlarm,
    
    (* 打胶机交互 *)
    i_打胶机允许送料 := di_X76_Glue_AllowFeed,
    i_打胶机取料完成 := di_X102_Glue_PickDone,
    
    (* 上游信号 *)
    i_取放料_放料完成 := fbPickPlace.o_放料完成_给送料机构,
    
    (* 输出 *)
    o_允许抓料 => dq_Y44_AllowPick,
    o_安全区信号 => dq_Y47_SafeZone,
    o_b运行中 => mb_M12_Running,
    o_b故障 => mb_M13_Fault,
    o_i当前状态 => mi_CurrentStep,
    o_rX2轴当前位置 => mr_X2ActualPos,
    o_i本站报警代码 => iAlarmCode
);


(* ---------- X2轴运动处理(主控侧) ---------- *)
bX2ReqFwd := fbGlueMachineFeeder.s_bX2轴请求前移;
bX2ReqBwd := fbGlueMachineFeeder.s_bX2轴请求后移;

IF bX2ReqFwd THEN
    (* 调用X2轴伺服前进指令 *)
    MC_MoveAbsolute(Axis := Axis_X2, 
                    Position := fbGlueMachineFeeder.i_r取料位置,
                    Velocity := fbGlueMachineFeeder.i_r送料速度,
                    Execute := TRUE);
ELSIF bX2ReqBwd THEN
    (* 调用X2轴伺服返回指令 *)
    MC_MoveAbsolute(Axis := Axis_X2,
                    Position := fbGlueMachineFeeder.i_r待机位置,
                    Velocity := fbGlueMachineFeeder.i_r送料速度,
                    Execute := TRUE);
END_IF;


(* ---------- 报警上报到全局系统 ---------- *)
IF iAlarmCode <> 0 THEN
    (* 上报到公共报警管理器 *)
    fbCommonAlarm.i_iStationID := 4;  (* 工站编号4=打胶机送料 *)
    fbCommonAlarm.i_iAlarmCode := iAlarmCode;
END_IF;


(* ---------- HMI状态更新 ---------- *)
mb_M14_StepCode := fbGlueMachineFeeder.o_i当前状态;
mr_D23_X2DisplayPos := fbGlueMachineFeeder.o_rX2轴当前位置;
```

## 4. X2轴运动处理详解（主控侧）

### 4.1 设计原则

**重要**: 本功能块**不直接输出伺服脉冲**，而是通过内部请求标志通知主控程序，由主控统一处理所有伺服运动。

```
设计理由:
┌─────────────────┐    s_bX2轴请求_*    ┌─────────────────┐
│  FB_1004        │ ──────────────────> │  主控程序(Main)  │
│  (纯逻辑)       │                     │                 │
│  决定"何时移动"  │                     │  处理"如何移动"   │
│  和"移动方向"    │                     │  (脉冲输出/定位)  │
└─────────────────┘                     └─────────────────┘
```

### 4.2 内部请求标志说明

本FB通过两个内部VAR变量传递X2轴运动请求：

| 变量名 | 类型 | 含义 | 有效时机 |
|--------|:----:|:-----|:--------|
| `s_bX2轴请求前移` | BOOL | 请求X2轴向打胶机方向移动 | 步骤2期间为TRUE |
| `s_bX2轴请求后移` | BOOL | 请求X2轴返回待机位置 | 步骤5期间为TRUE |

**访问方式**: 由于这些是内部变量(VAR)，需要在主控中通过实例名访问：
```st
bX2ForwardReq := fbGlueMachineFeeder.s_bX2轴请求前移;
bX2BackwardReq := fbGlueMachineFeeder.s_bX2轴请求后移;
```

### 4.3 到位反馈机制

主控在执行X2轴运动时，需要将到位信息反馈给本FB：

```st
(* 主控侧: X2轴到位反馈 *)
IF Axis_X2.StatusDone THEN
    (* 判断当前目标位置，设置对应的到位标志 *)
    IF ABS(Axis_X2.ActPosition - fbGlueMachineFeeder.i_r取料位置) < 1.0 THEN
        fbGlueMachineFeeder.s_bX2轴在取料位 := TRUE;  (* 反馈给步骤2 *)
    ELSIF ABS(Axis_X2.ActPosition - fbGlueMachineFeeder.i_r待机位置) < 1.0 THEN
        fbGlueMachineFeeder.s_bX2轴在待机位 := TRUE;  (* 反馈给步骤5 *)
    END_IF;
ELSE
    (* 清除到位标志 *)
    fbGlueMachineFeeder.s_bX2轴在取料位 := FALSE;
    fbGlueMachineFeeder.s_bX2轴在待机位 := FALSE;
END_IF;
```

## 5. 参数调优建议

### 5.1 关键参数推荐值

| 参数名 | 默认值 | 推荐范围 | 调优依据 | 影响因素 |
|--------|:------:|:--------:|:---------|:---------|
| `T_DEF_通讯超时时间` | 10000ms (10s) | 5000~15000ms | 打胶机响应速度 | 过短易误报，过长等待久 |
| `T_DEF_移动超时时间` | 5000ms (5s) | 3000~8000ms | X2轴行程长度和速度 | 根据实际行程计算 |
| `i_r送料速度` | 100.0 mm/s | 50~200 mm/s | 机械负载和精度要求 | 速度快效率高但冲击大 |
| `i_r待机位置` | 0.0 mm | 根据机械设计 | 安全区域边界 | 避免干涉取放料机构 |
| `i_r取料位置` | 300.0 mm | 根据机械设计 | 打胶机取料口坐标 | 需精确对准±1mm |

### 5.2 超时时间调优方法

**通讯超时时间 (T_DEF_通讯超时时间)**:
```
计算公式:
超时时间 = 正常响应时间 × 安全系数(1.5~2.0)

示例:
如果打胶机通常在3~5秒内响应:
  推荐超时时间 = 5s × 2.0 = 10秒 (默认值)
  
如果经常出现误报警:
  可适当增加到 15秒
  
如果需要快速故障检测:
  可缩短到 8秒 (但需确认打胶机能稳定响应)
```

**移动超时时间 (T_DEF_移动超时时间)**:
```
计算公式:
超时时间 = 行程距离 / 速度 × 安全系数(1.5)

示例:
X2轴行程 = 300mm, 速度 = 100mm/s:
  正常移动时间 = 300 / 100 = 3秒
  推荐超时时间 = 3s × 1.5 ≈ 5秒 (默认值)
```

## 6. 调试步骤建议

### 6.1 分阶段调试流程

```
调试阶段划分:

阶段1: 基础IO验证 [预计30分钟]
│  ├─ Step 1.1: 验证所有DI信号(传感器、打胶机信号)
│  ├─ Step 1.2: 验证DO输出(Y44/Y44)能正确输出
│  └─ Step 1.3: 验证HMI按钮输入正常
│
阶段2: 手动模式调试 [预计30分钟]
│  ├─ Step 2.1: 进入手动模式，测试X2轴点动前后
│  ├─ Step 2.2: 测试限位开关保护是否生效
│  └─ Step 2.3: 验证伺服故障报警(ALM_203)
│
阶段3: 单循环自动测试 [预计45分钟]
│  ├─ Step 3.1: 设置自动模式，按启动按钮
│  ├─ Step 3.2: 观察状态机从步骤0→1→2→...→5→1循环
│  ├─ Step 3.3: 验证安全区信号联动(i_取放料_放料完成→o_安全区信号)
│  └─ Step 3.4: 检查HMI状态显示是否正确(步序码0~5)
│
阶段4: 与打胶机联调 [预计60分钟]
│  ├─ Step 4.1: 模拟打胶机发送允许送料信号(X76)
│  ├─ Step 4.2: 验证本FB输出允许抓料(Y44)后打胶机能收到
│  ├─ Step 4.3: 模拟打胶机发送取料完成(X102)
│  ├─ Step 4.4: 测试通讯超时保护(断开X76/X102观察ALM_201)
│  └─ Step 4.5: 全流程连续运行10个循环以上
│
阶段5: 异常场景测试 [预计30分钟]
   ├─ Step 5.1: 急停测试(按下急停→恢复→复位→重启)
   ├─ Step 5.2: 伺服故障模拟(触发ALM_203→复位→重新初始化)
   └─ Step 5.3: 安全门打开/关闭测试
```

### 6.2 各阶段检查清单

#### 阶段1: 基础IO验证 ✅

- [ ] X2轴原点信号(di_X2_HomeSwitch)在原点位置为TRUE
- [ ] X2轴正向限位(di_X2_ForwardLimit)到达极限时为TRUE
- [ ] X2轴反向限位(di_X2_BackwardLimit)到达极限时为TRUE
- [ ] X2轴伺服故障(di_X2_ServoAlarm)正常时为FALSE
- [ ] 打胶机允许送料(di_X76_Glue_AllowFeed)可接收外部信号
- [ ] 打胶机取料完成(di_X102_Glue_PickDone)可接收外部信号
- [ ] Y44(允许抓料)输出能点亮LED或被外部设备检测
- [ ] Y47(安全区信号)输出能点亮LED或被外部设备检测
- [ ] HMI启动/停止/复位按钮都能正常触发
- [ ] HMI模式切换(自动/手动)正常工作

#### 阶段2: 手动模式调试 ✅

- [ ] 手动模式下X2轴点动前按钮能让X2轴向前移动
- [ ] 手动模式下X2轴点动后按钮能让X2轴向后移动
- [ ] 前后点动互斥（不能同时有效）
- [ ] 到达正向限位后无法继续前移（保护生效）
- [ ] 到达反向限位后无法继续后移（保护生效）
- [ ] 触发伺服故障后立即停止点动并报警ALM_203
- [ ] 复位按钮能清除ALM_203报警

#### 阶段3: 单循环自动测试 ✅

- [ ] 自动模式下按启动按钮，状态机进入步骤1
- [ ] HMI显示步序码从0变为1("等待放料...")
- [ ] 当`i_取放料_放料完成`=TRUE时，转到步骤2
- [ ] 步骤2期间`s_bX2轴请求前移`=TRUE，X2轴开始前移
- [ ] X2轴到达取料位后，反馈`s_bX2轴在取料位`=TRUE，进入步骤3
- [ ] 步骤3等待`i_打胶机允许送料`=TRUE（可手动置位测试）
- [ ] 收到允许后进入步骤4，`o_允许抓料`(Y44)=TRUE
- [ ] 等待`i_打胶机取料完成`=TRUE后进入步骤5
- [ ] 步骤5期间`s_bX2轴请求后移`=TRUE，X2轴返回
- [ ] X2轴回到待机位后回到步骤1（自动循环）
- [ ] 整个过程`o_b运行中`=TRUE，HMI运行灯亮
- [ ] `o_安全区信号`(Y47)与`i_取放料_放料完成`同步变化

#### 阶段4: 与打胶机联调 ✅

- [ ] 打胶机准备好后X76=TRUE，本FB能正确接收
- [ ] 本FB输出Y44=TRUE后，打胶机能检测到此信号
- [ ] 打胶机完成抓取后X102=TRUE，本FB能正确接收
- [ ] 断开X76超过10秒 → 触发ALM_201(通讯超时)
- [ ] 断开X102超过10秒 → 触发ALM_201(通讯超时)
- [ ] 超时报警后按复位能清除，重新进入步骤0
- [ ] 连续运行10个循环无异常

#### 阶段5: 异常场景测试 ✅

- [ ] 急停按下 → 所有输出立即归零，状态机回步骤0
- [ ] 急停释放 → 按`i_b复位`重新初始化
- [ ] 伺服故障 → ALM_203立即触发，X2轴停止
- [ ] 伺服故障排除 + 复位 → 重新初始化成功
- [ ] 安全门打开 → `i_b使能`=FALSE，所有输出归零
- [ ] 安全门关闭 → 恢复正常运行

| 9 | HMI状态显示一直是0 | ① 未进入自动模式<br>② 状态机一直在步骤0<br>③ `o_i当前状态`未连接到HMI | ① 切换自动模式并启动<br>② 检查启动条件和故障<br>③ 检查HMI变量映射 | - |

## 8. 使用约束与禁忌

### 8.1 ⛔ 禁止事项

| 序号 | 禁止操作 | 后果 | 正确做法 |
|:----:|:--------|:-----|:--------|
| 1 | **同时设置`i_b自动模式`和`i_b手动模式`为TRUE** | 功能块进入不确定状态，可能全部输出归零 | 确保两者互斥（HMI应做联锁） |
| 2 | **在`i_b使能`=FALSE时尝试启动** | 启动信号被忽略，状态机不会离开步骤0 | 先确保系统就绪条件满足 |
| 3 | **忽略ALM_203(伺服故障)继续运行** | 可能损坏伺服驱动器或机械结构 | 必须先排除故障并复位 |
| 4 | **手动修改内部变量(`s_*`)绕过状态机** | 破坏状态机逻辑，可能导致不可预测的行为 | 只通过标准输入接口操作 |
| 5 | **在步骤4(允许抓料)期间突然切断`i_b使能`** | Y44可能保持TRUE（取决于实现） | 应先按停止按钮让状态机正常结束 |
| 6 | **将`i_打胶机允许送料`硬接线为常TRUE** | 失去通讯超时保护，可能导致安全隐患 | 必须由打胶机真实状态决定 |

### 8.2 ⚠️ 注意事项

| 序号 | 注意事项 | 说明 | 建议 |
|:----:|:--------|:-----|:-----|
| 1 | **启动信号必须做边沿检测** | 否则每次扫描都会重复触发启动 | 使用R_TRIG指令 |
| 2 | **`i_取放料_放料完成`信号稳定性** | 此信号跳变会导致安全区信号频繁切换 | 上游应保证至少500ms稳定 |
| 3 | **X2轴运动由主控统一处理** | 本FB只负责决策，不负责执行 | 不要在本FB内写脉冲输出代码 |
| 4 | **报警代码需上报全局系统** | `o_i本站报警代码`要连接到公共报警FB | 否则HMI看不到此站的报警 |
| 5 | **复位后需等待初始化完成** | 初始化最多需要10秒（伺服检查） | HMI应有"初始化中..."提示 |
| 6 | **定期备份参数设置** | `i_r*`系列参数可能在HMI中被误修改 | 建议保存默认参数配置文件 |

### 8.3 ✅ 推荐做法

| 序号 | 推荐做法 | 好处 |
|:----:|:--------|:-----|
| 1 | HMI增加"单循环"模式 | 方便调试单个循环而不必连续运行 |
| 2 | 记录每个循环的周期时间 | 用于评估节拍是否符合产能要求 |
| 3 | 在HMI显示X2轴的实际位置曲线 | 监控伺服运动是否平稳 |
| 4 | 为打胶机交互信号增加状态指示灯 | 快速定位通讯问题 |
| 5 | 定期测试急停和安全门功能 | 确保安全系统始终可靠 |
| 6 | 建立完整的报警日志系统 | 记录每次报警的时间、原因、持续时间 |

```st
(* 示例: 同时控制两台打胶机的送料 *)

    fbGlueMachineA : FB_1004_GlueMachineFeeder_BufferFraming;
    fbGlueMachineB : FB_1004_GlueMachineFeeder_BufferFraming;

(* 分别调用两个实例，共享X2轴但独立控制打胶机交互 *)
fbGlueMachineA(
    i_打胶机允许送料 := di_X76_GlueA_AllowFeed,
    i_打胶机取料完成 := di_X102_GlueA_PickDone,
    o_允许抓料 => dq_Y44_GlueA_AllowPick,
    (* ... 其他参数 ... *)
);

fbGlueMachineB(
    i_打胶机允许送料 := di_X77_GlueB_AllowFeed,
    i_打胶机取料完成 := di_X103_GlueB_PickDone,
    o_允许抓料 => dq_Y45_GlueB_AllowPick,
    (* ... 其他参数 ... *)
);
```

### 9.2 与MES系统集成

可以将关键状态上传到MES系统：

```st
(* MES数据采集示例 *)
fbMES.i_Data[1] := fbGlueMachineFeeder.o_i当前状态;      (* 当前步序 *)
fbMES.i_Data[2] := fbGlueMachineFeeder.o_i本站报警代码;  (* 报警代码 *)
fbMES.i_Data[3] := INT_TO_WORD(BOOL_TO_INT(fbGlueMachineFeeder.o_b运行中)); (* 运行状态 *)
fbMES.i_Data[4] := INT_TO_WORD(BOOL_TO_INT(fbGlueMachineFeeder.o_b故障));    (* 故障状态 *)
```

### A.2 报警码速查

```
报警代码速查 (201~299段):

  0   = 无报警 (正常)
  201 = 通讯超时 (打胶机10秒无响应) ← 最常见!
```
