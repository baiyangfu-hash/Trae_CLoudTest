## 2. 前置条件与依赖项

### 2.1 硬件/软件环境要求

| 项目 | 要求 | 说明 |
|------|:----:|:-----|
| **PLC型号** | 支持ST语言的三菱FX5系列或兼容控制器 | 必须支持结构化文本编程和ARRAY类型 |
| **工站FB就绪** | FB_1001/FB_1003/FB_1004均已正确实例化并运行 | 本FB依赖三个上游FB的报警输出 |

### 2.3 必须连接的信号清单（最小化调用）

#### 红色 - 必须连接（不连则无法运行）

| 针脚名称 | 连接建议 | 原因 |
|:---------|:---------|:-----|
| `i_i输送机报警` | `fbConveyor.o_i本站报警代码` | 无此输入则无法感知输送机报警 |
| `i_i取放料报警` | `fbPickPlace.o_i本站报警代码` | 无此输入则无法感知取放料报警 |
| `i_i送料机构报警` | `fbGlueFeeder.o_i本站报警代码` | 无此输入则无法感知送料机构报警 |
| `i_b复位` | HMI M区变量或DI映射 | 无法清除报警状态 |

#### 黄色 - 强烈建议连接（不连则功能受限）

| 针脚名称 | 连接建议 | 不连的影响 |
|:---------|:---------|:----------|
| `o_w全局报警字` → D400 | `md_D400_AlarmWord` | HMI无法显示报警灯 |
| `o_i当前报警代码` → D402 | `md_D402_AlarmCode` | HMI无法显示报警文本 |
| `o_b任何报警激活` → M200 | `mb_M200_AnyAlarm` | 全局互锁失效，有报警时仍可能自动运行 |
| `o_b新报警标志` | 主控读取 | MES上传触发失效 |

#### 绿色 - 可选连接（用于增强功能）

| 针脚名称 | 用途 |
|:---------|:-----|
| `o_iMES报警计数` → D404 | MES累计统计 |
| `o_iMES报警队列` → D406~D425 | MES去重队列采集 |
| `o_i工站报警状态[1..3]` | HMI分站状态面板 |
| `o_b输送机报警激活` | HMI输送机站点灯 |
| `o_b取放料报警激活` | HMI取放料站点灯 |
| `o_b送料机构报警激活` | HMI送料机构站点灯 |

## 3. 快速开始指南

### 3.1 最小化调用示例（仅验证基本逻辑）

```st
(* ============================================ *)
(* FB_2001_CommonAlarm 最小化调用示例             *)
(* 目的: 快速验证FB的基本报警汇聚和优先级选取功能   *)
(* ============================================ *)

VAR
    (* 功能块实例 *)
    fbCommonAlarm : FB_2001_CommonAlarm_AllStation;
    
    (* 模拟输入 - 测试用 *)
    iTestConvAlm   : INT := 0;  (* 模拟输送机报警 *)
    iTestPickAlm    : INT := 0;  (* 模拟取放料报警 *)
    iTestFeedAlm    : INT := 0;  (* 模拟送料机构报警 *)
    bTestReset      : BOOL := FALSE; (* 模拟复位 *)
END_VAR


(* ========== 调用功能块 ========== *)
fbCommonAlarm(
    (* --- 工站报警代码输入 --- *)
    i_i输送机报警   := iTestConvAlm,     (* 来自FB_1001 *)
    i_i取放料报警   := iTestPickAlm,     (* 来自FB_1003 *)
    i_i送料机构报警 := iTestFeedAlm,     (* 来自FB_1004 *)
    
    (* --- 操作控制输入 --- *)
    i_b复位         := bTestReset,       (* HMI复位按钮 *)
    
    (* --- HMI显示输出 -> D区 --- *)
    o_w全局报警字   => md_D400_AlarmWord,  (* -> D400 *)
    o_i当前报警代码 => md_D402_AlarmCode,  (* -> D402 *)
    
    (* --- 全局互锁输出 -> M区 --- *)
    o_b任何报警激活 => mb_M200_AnyAlarm,   (* -> M200 *)
    
    (* --- MES数据接口输出 -> D区 --- *)
    o_iMES报警计数  => md_D404_MESCount,   (* -> D404 *)
    o_iMES报警队列  => mai_D406_MESQueue,  (* -> D406~D425 *)
    
    (* --- 新报警脉冲 -> 主控读取 --- *)
    o_b新报警标志   => gbNewAlarmFlag,
    
    (* --- 多报警状态输出 --- *)
    o_i工站报警状态 => mai_StationAlms,
    o_b输送机报警激活 => gbConvAlmActive,
    o_b取放料报警激活 => gbPickAlmActive,
    o_b送料机构报警激活 => gbFeedAlmActive
);


(* ========== 测试用: 手动设置模拟输入 ========== *)
(* 在调试模式下通过HMI或监控窗口修改这些值 *)
(* 示例: 设置 iTestConvAlm := 3 触发"分料推出超时" *)
(* 示例: 设置 iTestPickAlm := 103 触发"升降下降超时" *)
(* 观察输出: md_D400, md_D402, mb_M200 的变化 *)
```

### 3.2 完整生产环境调用示例

```st
(* ============================================ *)
(* 完整生产环境调用示例                             *)
(* 包含: 三站FB连接、D/M区地址映射、MES上传触发     *)
(* ============================================ *)

PROGRAM Main
VAR
    (* ====== 功能块实例 ====== *)
    fbCommonAlarm : FB_2001_CommonAlarm_AllStation;
    
    (* ====== 各工站FB实例 (已在其他位置声明) ====== *)
    fbConveyor   : FB_1001_Conveyor4Layer_BufferFraming;   (* 四层输送机 *)
    fbPickPlace  : FB_1003_PickPlace_BufferFraming;        (* 取放料机构 *)
    fbGlueFeeder : FB_1004_GlueMachineFeeder_BufferFraming; (* 送料机构 *)
    
    (* ====== 边沿检测: 新报警脉冲 ====== *)
    rtrig_NewAlarm : R_TRIG;
    
    (* ====== MES上传相关 ====== *)
    bNeedMESUpload : BOOL;
END_VAR


(* ============================================================ *)
(* 第一部分: 调用各工站FB (必须在公共报警FB之前执行)              *)
(* ============================================================ *)

(* --- 调用四层输送机FB --- *)
fbConveyor(
    (* ... 各参数连接 ... *)
    o_i本站报警代码 => gi_ConvAlarmCode  (* 关键: 输出报警码 *)
);

(* --- 调用取放料机构FB --- *)
fbPickPlace(
    (* ... 各参数连接 ... *)
    o_i本站报警代码 => gi_PickAlarmCode  (* 关键: 输出报警码 *)
);

(* --- 调用打胶机送料机构FB --- *)
fbGlueFeeder(
    (* ... 各参数连接 ... *)
    o_i本站报警代码 => gi_FeedAlarmCode  (* 关键: 输出报警码 *)
);


(* ============================================================ *)
(* 第二部分: 调用公共报警FB (汇聚三站报警)                       *)
(* ============================================================ *)

fbCommonAlarm(
    (* --- 工站报警代码输入 (来自各工站FB的输出) --- *)
    i_i输送机报警   := gi_ConvAlarmCode,    (* FB_1001 -> 1~7段 *)
    i_i取放料报警   := gi_PickAlarmCode,    (* FB_1003 -> 101~108段 *)
    i_i送料机构报警 := gi_FeedAlarmCode,    (* FB_1004 -> 201~205段 *)
    
    (* --- 操作控制输入 --- *)
    i_b复位 := di_M2_ResetButton,           (* HMI复位按钮(DI或M区) *)
    
    (* --- HMI显示输出 -> 写入D区寄存器 --- *)
    o_w全局报警字   => md_D400_AlarmWord,   (* D400: HMI报警灯 *)
    o_i当前报警代码 => md_D402_AlarmCode,   (* D402: HMI文本显示 *)
    
    (* --- 全局互锁输出 -> 写入M区辅助继电器 --- *)
    o_b任何报警激活 => mb_M200_AnyAlarm,    (* M200: 运行互锁 *)
    
    (* --- MES数据接口 -> 写入D区供MES采集 --- *)
    o_iMES报警计数  => md_D404_MESCount,    (* D404: 累计次数 *)
    o_iMES报警队列  => mai_D406_MESQueue,   (* D406~D425: 去重队列 *)
    
    (* --- 新报警脉冲 -> 主控读取用于触发MES上传 --- *)
    o_b新报警标志   => gbNewAlarmPulse,
    
    (* --- 多报警状态输出 (HMI分站显示) --- *)
    o_i工站报警状态 => mai_StationAlarmStatus,
    o_b输送机报警激活 => mb_ConvAlarmActive,
    o_b取放料报警激活 => mb_PickAlarmActive,
    o_b送料机构报警激活 => mb_FeedAlarmActive
);


(* ============================================================ *)
(* 第三部分: 主控侧处理 (基于公共报警输出的联动逻辑)              *)
(* ============================================================ *)

(* --- 3.1 新报警检测 + MES上传触发 --- *)
rtrig_NewAlarm(CLK := fbCommonAlarm.o_b新报警标志);

IF rtrig_NewAlarm.Q THEN
    (* 检测到新报警脉冲, 执行MES上传 *)
    bNeedMESUpload := TRUE;
    
    (* TODO: 在此处添加MES通信逻辑 *)
    (* 例如: 调用MES发送FB, 读取D406~D425队列数据 *)
END_IF;


(* --- 3.2 全局运行互锁 --- *)
IF fbCommonAlarm.o_b任何报警激活 THEN
    (* 有报警存在时的处理 *)
    gbAutoRunEnable := FALSE;          (* 禁止自动运行启动 *)
    gbAlarmBuzzer := TRUE;             (* 报警蜂鸣器 *)
ELSE
    (* 无报警时的正常状态 *)
    gbAutoRunEnable := gbSystemReady AND gbAutoMode;
    gbAlarmBuzzer := FALSE;
END_IF;


(* --- 3.4 HMI状态更新 (可选增强) --- *)
mb_M201_ConvAlmLED := fbCommonAlarm.o_b输送机报警激活;    (* 输送机站点灯 *)
mb_M202_PickAlmLED := fbCommonAlarm.o_b取放料报警激活;    (* 取放料站点灯 *)
mb_M203_FeedAlmLED := fbCommonAlarm.o_b送料机构报警激活;  (* 送料站点灯 *)
```

### 4.2 调用时序注意事项

| 注意事项 | 说明 | 建议 |
|:---------|:-----|:-----|
| **调用顺序** | 公共报警FB必须在三个工站FB**之后**调用 | 先调FB_1001/1003/1004，再调FB_2001 |
| **扫描周期一致性** | 所有FB在同一扫描周期内调用 | 不要跨周期调用 |
| **复位信号同步** | 复位信号应同时传给各工站FB和公共报警FB | 主控统一分发i_b复位 |
| **MES读取时机** | 应在`o_b新报警标志`脉冲上升沿读取队列 | 使用R_TRIG检测 |

## 5. 报警码段分配说明

### 5.1 为什么分三段?

本系统的报警码采用**段式编码设计**，将20个报警码分为三个数值区间：

```
报警码数值空间:

  低数值 ───────────────────────────────────> 高数值
  高优先级                                    低优先级

  ┌─────────┬──────────┬──────────┬─────────────────────┐
  │ 1 ~ 7   │101 ~ 108 │201 ~ 205 │ 301+ (预留扩展)     │
  │输送机   │取放料    │送料机构  │系统/HMI/安全级等     │
  └─────────┴──────────┴──────────┴─────────────────────┘
```

### 5.2 分段设计的优势

| 优势 | 详细说明 |
|:-----|:---------|
| **天然优先级** | 数值越小越优先。当多站同时报警时，只需选最小值即可得到最高优先级报警，无需额外优先级配置表 |
| **工站归属一目了然** | 看报警码的百位数即可判断来源：1xx=输送机区域问题, 2xx=取放料区域问题, 3xx=送料区域问题 |
| **HMI显示友好** | 报警码本身携带工站信息，操作员看到"ALM-003"立即知道是输送机的"分料推出超时" |
| **MES分类方便** | MES系统按码段范围即可统计分析各工站的故障频率和MTBF |
| **独立扩展** | 每个段内有大量预留空间（如1~7段可扩展到1~99），新增报警不影响其他段的编号 |

### 5.3 优先级竞争实际场景

| 场景描述 | 输送机 | 取放料 | 送料机构 | 显示结果 | 原因 |
|:---------|:------:|:------:|:--------:|:-------:|:-----|
| 仅输送机阻挡下降超时 | 1 | 0 | 0 | **ALM-001** | 只有1个报警 |
| 输送机和取放料同时报警 | 5 | 102 | 0 | **ALM-005** | 5 < 102, 输送机优先 |
| 三站全部报警 | 7 | 105 | 203 | **ALM-007** | 7最小, 输送机始终最高优先 |
| 取放料和送料机构同时报警 | 0 | 108 | 201 | **ALM-108** | 108 < 201, 取放料优先 |

### 6.2 上传触发机制

#### 方式一：脉冲边沿检测（推荐）

```st
(* 推荐做法: 用R_TRIG检测新报警脉冲的上升沿 *)
VAR
    rtrig_MESUpload : R_TRIG;
END_VAR

rtrig_MESUpload(CLK := fbCommonAlarm.o_b新报警标志);

IF rtrig_MESUpload.Q THEN
    (* 在脉冲上升沿的当个周期内执行上传 *)
    (* 1. 读取队列数据 *)
    (* 2. 读取报警计数 *)
    (* 3. 读取当前报警码 *)
    (* 4. 组装MES报文并发送 *)
    Call_MES_UploadAlarm(
        i_iAlarmCode  := fbCommonAlarm.o_i当前报警代码,
        i_iTotalCount := fbCommonAlarm.o_iMES报警计数,
        ai_QueueData  := fbCommonAlarm.o_iMES报警队列
    );
END_IF;
```

#### 方式二：轮询检查（备选）

```st
(* 备选做法: 如果R_TRIG不方便使用, 可以在主控中维护上周状态 *)
VAR
    bLastNewAlarmFlag : BOOL := FALSE;
END_VAR

(* 手动实现上升沿检测 *)
IF fbCommonAlarm.o_b新报警标志 AND NOT bLastNewAlarmFlag THEN
    (* 检测到上升沿, 执行上传 *)
    Call_MES_UploadAlarm(...);
END_IF;

(* 更新上周状态 *)
bLastNewAlarmFlag := fbCommonAlarm.o_b新报警标志;
```

### 6.3 MES报文格式建议

```st
(* MES报文数据结构建议 *)

TYPE MES_AlarmReport :
STRUCT
    DeviceID      : STRING(16);   (* 设备ID: "DJ-2026-005" *)
    Timestamp     : DT;           (* 报警发生时间(PLC系统时钟) *)
    CurrentAlarm  : INT;          (* 当前最高优先级报警码 *)
    TotalCount    : INT;          (* 自复位以来的累计次数 *)
```

### 6.4 数据读取注意事项

| 注意事项 | 说明 | 建议 |
|:---------|:-----|:-----|
| **读取时机** | 必须在`o_b新报警标志=TRUE`的同一周期内读取队列 | 否则队列可能已被下一周期的数据覆盖 |
| **原子性** | 队列数据和计数应在同一次读取中完成 | 避免数据不一致 |
| **INT存储格式** | 三菱PLC的INT为16bit小端模式 | MES解析时注意字节序 |
| **空位处理** | 队列中值为0的位置表示未使用 | MES端应过滤掉0值记录 |
| **重复上传防护** | 同一脉冲不应触发多次上传 | R_TRIG保证单次触发 |

## 10. 使用约束与禁忌

### 10.1 禁止事项

| 序号 | 禁止操作 | 后果 | 正确做法 |
|:----:|:--------|:-----|:--------|
| 1 | **在工站FB之前调用FB_2001** | 收到的报警输入是上一周期的旧值，导致延迟一个周期 | 先调用FB_1001/1003/1004，再调用FB_2001 |
| 2 | **修改QUEUE_SIZE常量** | 内部算法硬编码了容量为10，改大不会自动扩展数组 | 如需更大队列需同步修改ARRAY声明和算法 |
| 3 | **直接写入o_*输出变量绕过FB** | 下一个扫描周期会被FB的计算结果覆盖 | 通过标准输入接口影响输出 |
| 4 | **将i_b复位做成脉冲信号** | 复位是电平有效的，脉冲可能导致复位不完全 | 使用保持型按钮或确保脉冲宽度足够长(>1个扫描周期) |
| 5 | **忽略o_b任何报警激活进行自动运行** | 有报警时强制运行可能导致设备损坏或安全事故 | 将此信号作为自动运行的前置否定条件 |

### 10.2 注意事项

| 序号 | 注意事项 | 说明 | 建议 |
|:----:|:---------|:-----|:-----|
| 1 | **调用顺序必须正确** | FB_2001依赖于三个工站FB在本周期内的输出值 | 在主控程序中将FB_2001放在最后调用 |
| 2 | **MES队列读取时机** | 必须在`o_b新报警标志=TRUE`的同一周期内读取 | 使用R_TRIG在其Q输出为TRUE时立即读取 |
| 3 | **复位是全局操作** | 复位会清除所有历史数据（包括MES队列和计数） | 操作员应了解复位的全部影响 |
| 4 | **报警计数的含义** | `o_iMES报警计数`包含重复报警（每周期有报警就+1） | 不等同于去重后的唯一报警数 |
| 5 | **纯逻辑块特性** | 本FB不含X/Y/M/D地址引用 | 所有IO映射由主控程序负责 |
| 6 | **D区地址空间** | D400~D425共26个D地址(13个WORD)被本FB占用 | 确保与其他功能不冲突 |

### 10.3 推荐做法

| 序号 | 推荐做法 | 好处 |
|:----:|:---------|:-----|
| 1 | 在HMI增加报警历史列表页面 | 利用`o_i工站报警状态[1..3]`显示所有活跃报警 |
| 2 | 为MES上传增加重试机制 | 通信失败时不丢失本次报警数据 |
| 3 | 定期备份D404(报警计数)到非易失区 | 断电重启后可恢复累计统计数据 |
| 4 | 在HMI增加"报警频率分析"画面 | 基于`o_iMES报警计数`和队列数据分析设备健康度 |
| 5 | 为复位按钮增加二次确认 | 防止误触导致重要诊断数据丢失 |
| 6 | 建立报警响应SOP | 明确每种报警的处理流程和责任人 |

## 11. 扩展应用场景

### 11.1 增加工站（预留扩展）

如果未来需要接入第四个工站（如组框机），可以扩展本FB：

```st
(* 扩展示例: 新增第四工站 *)
(* 1. 新增输入接口 *)

(* 2. 在Step 3中增加有效性判定 *)

(* 3. 在Step 4中增加OR运算 *)
IF s_b组框机有报警 THEN
    o_w全局报警字 := o_w全局报警字 OR INT_TO_WORD(i_i组框机报警);
END_IF;

(* 4. 在Step 5中增加比较(放在现有三段之后, 优先级最低) *)
IF s_b组框机有报警 THEN
    IF s_i最小报警码 = ALM_NONE THEN
        s_i最小报警码 := i_i组框机报警;
    ELSIF i_i组框机报警 < s_i最小报警码 THEN
        s_i最小报警码 := i_i组框机报警;
    END_IF;
END_IF;

(* 5. 相应更新Step 6/7/8/9中的逻辑 *)
```

### 11.2 队列容量调整

如果需要更大的MES队列（如从10条改为20条）：
