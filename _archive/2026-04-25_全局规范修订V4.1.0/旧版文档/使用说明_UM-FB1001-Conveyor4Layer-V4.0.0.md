## 2. 前置条件与依赖项

### 2.1 硬件环境要求

| 项目 | 要求 | 说明 |
|------|:----:|:-----|
| **PLC型号** | 支持ST语言的控制器（如西门子S7-1200/1500、三菱FX5U等） | 必须支持结构化文本编程和ARRAY数据类型 |
| **DI模块** | DI3 + DI4（各16点输入模块） | 共28个DI点用于4层传感器(7x4) |
| **DO模块** | DO1 + DO2（各16点输出模块） | 共20个DO点用于4层执行器(5x4) |
| **气缸系统** | 阻挡气缸x4 + 分料气缸x8（每层各1个） | 每个气缸配2个磁性开关(上位/下位) |
| **输送带/变频器** | 输送带x4 + 变频器x4(VF1~VF4) | 每台变频器支持正转/慢速/反转3路DI控制 |
| **传感器** | 光电传感器x14(分料前+到位x2)x4层 | 对射式或漫反射式，需调整灵敏度 |
| **HMI界面** | 已配置操作按钮、状态显示、参数设定画面 | 启动/停止/复位、手动操作按钮、步序显示 |

### 2.2 软件依赖

| 依赖项 | 版本要求 | 用途 |
|--------|:-------:|:-----|
| 主控程序 (PRG_Main) | V4.0.0+ | IO映射和调用本FB容器 |
| 公共报警FB (FB_2001) | V4.0.0+ | 接收101~107段报警代码 |
| 取放料机构FB (FB_1003) | V4.0.0+ | 提供`i_b取放料完成[1..4]`上游信号 |

### 2.3 必须连接的信号清单（最小化调用）

> 即使是测试，以下信号也**必须正确连接**，否则功能块无法正常工作：

#### 🔴 必须连接 -- 容器模式(FB_1001)

| 针脚名称 | 连接建议 | 原因 |
|---------|:---------|:-----|
| `i_b使能` | `gb系统就绪` | 总开关，FALSE时全部输出归零 |
| `i_b自动模式` | HMI M变量 | 模式选择必需 |
| `i_b手动模式` | HMI M变量 | 与自动模式互斥 |
| `i_b启动` | HMI启动按钮(边沿检测) | 触发自动循环 |
| `i_b停止` | HMI停止按钮 | 安全停止必需 |
| `i_b复位` | HMI复位按钮 | 故障清除必需 |
| `i_分料前感应器[1..4]` | IO映射(DI3/DI4) | 步骤1转换条件 |
| `i_到位感应器1[1..4]` | IO映射(DI3/DI4) | 步骤3转换条件(双冗余之一) |
| `i_到位感应器2[1..4]` | IO映射(DI3/DI4) | 步骤3转换条件(双冗余之二) |
| `i_阻挡气缸下位[1..4]` | IO映射(DI3/DI4) | 步骤2转出条件 |
| `i_阻挡气缸上位[1..4]` | IO映射(DI3/DI4) | 步骤5转出条件+初始化检查 |
| `i_分料气缸下位[1..4]` | IO映射(DI3/DI4) | 步骤6转出条件 |
| `i_分料气缸上位[1..4]` | IO映射(DI3/DI4) | 步骤7验证+初始化检查 |
| `i_b取放料完成[1..4]` | 取放料FB输出 | 启动前置条件+循环连续性 |

#### 🟡 强烈建议连接

| 针脚名称 | 连接建议 | 不连的影响 |
|---------|:---------|:----------|
| `i_r输送速度` | HMI参数设定 | 速度显示为0，使用默认值50.0Hz |
| `i_i分料时间` | HMI参数设定 | 使用默认值1000ms |
| `i_i阻挡等待时间` | HMI参数设定 | 使用默认值800ms |
| 手动操作信号x24 | HMI手动按钮x24 | 无法进行手动调试 |
| 执行器输出x20 | 映射到DO1/DO2 | 无法驱动实际设备 |
| `o_i本站报警代码` | 报警管理器 | HMI看不到报警信息 |

#### 🟢 可选连接

| 针脚名称 | 用途 |
|---------|:-----|
| `o_b运行中` → HMI | 显示四层整体运行状态 |
| `o_b故障` → HMI | 显示四层整体故障状态 |
| `o_i当前状态` → HMI | 显示主要状态码(0~99) |
| `o_Lx当前步序[1..4]` → HMI | 显示各层独立步序(便于调试) |
| `o_b放料完成[1..4]` → 取放料机构 | 各层放料完成通知 |

## 3. 数组索引约定

> **核心约定**: FB_1001 容器的所有 ARRAY [1..4] 信号均遵循以下索引规则。

```
数组索引约定:

┌─────────────────────────────────────────────────────┐
│                  物理层级布局                       │
│                                                     │
│    ┌─────────┐  L4 = 第4层 = 顶层  = 数组索引 [4]   │
│    │         │                                     │
│    ├─────────┤  L3 = 第3层        = 数组索引 [3]     │
│    │         │                                     │
│    ├─────────┤  L2 = 第2层        = 数组索引 [2]     │
│    │         │                                     │
│    └─────────┘  L1 = 第1层 = 底层  = 数组索引 [1]   │
│                                                     │
│  材料流向: 上方进料 → 下方出料(给取放料机构)        │
│                                                     │
└─────────────────────────────────────────────────────┘

IO模块对应关系:
  L4(顶层) ←→ DI3:X0~X7  + DO1:Y0~Y4   (第1组)
  L3       ←→ DI3:X11~X17 + DO1:Y10~Y14  (第2组)
  L2       ←→ DI4:X1~X7  + DO2:Y0~Y4    (第3组)
  L1(底层) ←→ DI4:X11~X17 + DO2:Y10~Y14  (第4组)
```

**关键规则**:
- `[1]` = L1 = 底层（最靠近取放料机构的层）
- `[4]` = L4 = 顶层（最远离取放料机构的层）
- 编号从底向上递增，与物理堆叠顺序一致
- **切勿混淆**: DI3的X0~X7对应L4(顶层)，不是L1！

## 4. ST调用示例

## 5. 参数调优建议

### 5.1 关键参数推荐值

| 参数名 | 默认值 | 推荐范围 | 调优依据 | 影响因素 |
|--------|:------:|:--------:|:---------|:---------|
| `i_i阻挡等待时间` | 800 ms | 500 ~ 1500 ms | 气缸行程长度、气压压力(0.4~0.6MPa) | 过短导致误报ALM_101/102，过长影响节拍 |
| `i_i分料时间` | 1000 ms | 800 ~ 2000 ms | 分料气缸行程、负载重量 | 根据边框尺寸调整 |
| `i_r输送速度` | 50.0 Hz | 30 ~ 80 Hz | 变频器额定频率、电机极数 | 超过额定值会损坏电机 |
| `T_DEF_输送超时时间` | 5000 ms | 3000 ~ 8000 ms | 输送带长度、材料重量 | 初次调试建议加大到8000ms |
| `T_DEF_慢速送出时间` | 2000 ms | 1500 ~ 3000 ms | 慢速距离、定位精度要求 | 影响取料位置精度 |
| `T_DEF_到位稳定时间` | 500 ms | 300 ~ 800 ms | 传感器响应速度、机械振动 | 影响步骤3->4的转换稳定性 |

### 5.2 超时时间调优方法

**阻挡动作超时 (i_i阻挡等待时间)**:
```
计算公式:
超时时间 = 正常动作时间 x 安全系数(1.5~2.0)

示例:
如果阻挡气缸正常升降需要400ms:
  推荐超时时间 = 400ms x 2.0 = 800ms (默认值)

如果经常出现ALM_101/102误报警:
  可适当增加到 1200~1500ms
  同时检查气压是否偏低(应≥0.4MPa)
```

**输送超时 (T_DEF_输送超时时间)**:
```
计算公式:
超时时间 = 输送距离 / 输送速度 x 安全系数(1.5)

示例:
输送距离=1000mm, 速度=50Hz(约300mm/s):
  正常输送时间 ≈ 1000/300 = 3.3秒
  推荐超时时间 = 3.3s x 1.5 ≈ 5秒 (默认值)

初次调试建议:
  先设为 8000ms, 记录实际耗时后再逐步收紧
```

### 5.3 典型工况参数配置参考

| 工况描述 | 阻挡等待(ms) | 分料时间(ms) | 输送速度(Hz) | 输送超时(ms) | 备注 |
|---------|:-----------:|:----------:|:----------:|:----------:|:-----|
| 标准边框(小尺寸) | 800 | 1000 | 50.0 | 5000 | 默认配置 |
| 大尺寸边框 | 1200 | 1500 | 40.0 | 8000 | 增加时间和降低速度 |
| 高速生产模式 | 600 | 800 | 70.0 | 4000 | 设备状态良好时使用 |
| 低温环境(<10°C) | 1000 | 1200 | 45.0 | 6000 | 气缸动作变慢，增加余量 |
| 调试/维护模式 | 1500 | 2000 | 30.0 | 10000 | 最大安全余量 |

## 6. 多层联动测试步骤

### 6.1 分阶段调试流程

```
调试阶段划分:

阶段1: 手动验证 [预计40分钟]
│  ├─ Step 1.1: 确认HMI选手动模式，逐层测试6个手动操作按钮(共24个)
│  ├─ Step 1.2: 验证每个气缸的动作方向正确(上升/下降/推出/复位)
│  ├─ Step 1.3: 验证输送带正转/反转/慢速方向正确(共12个方向)
│  ├─ Step 1.4: 确认所有传感器信号正常(无冲突、无抖动,共28个)
│  └─ Step 1.5: 验证同层内互锁关系(如阻挡上下互斥、输送正反互斥)
│
阶段2: 单层自动测试 [预计60分钟]
│  ├─ Step 2.1: 只启用L1(底层), 其他3层保持无材料状态
│  ├─ Step 2.2: 放入测试材料, 按启动观察9步状态机流转(Step0→...→Step8→Step0)
│  ├─ Step 2.3: 记录每步耗时, 确认均在超时保护范围内
│  ├─ Step 2.4: 人为制造超时条件, 验证报警触发(ALM_101~105)和复位功能
│  └─ Step 2.5: 依次对L2/L3/L4重复上述测试, 确保每层独立工作正常
│
阶段3: 多层联动测试 [预计60分钟]
│  ├─ Step 3.1: 4层同时放入材料, 观察是否独立运行互不干扰
│  ├─ Step 3.2: 验证某层故障时不影响其他层运行(隔离性)
│  ├─ Step 3.3: 验证停止按钮可同时停止所有层
│  ├─ Step 3.4: 验证汇总状态输出(o_b运行中/o_b故障/o_i当前状态)正确
│  ├─ Step 3.5: 验证o_Lx当前步序[1..4]各自独立反映各层状态
│  └─ Step 3.6: 验证多报警时优先级选择(o_i本站报警代码取MIN)
│
阶段4: 上游联调 [预计90分钟]
│  ├─ Step 4.1: 连接取放料机构, 验证o_b放料完成[1..4]信号时序(~500ms脉冲)
│  ├─ Step 4.2: 验证i_b取放料完成[1..4]信号的联锁逻辑
│  ├─ Step 4.3: 测试循环连续性(一层完成后能否自动开始下一轮)
│  ├─ Step 4.4: 全流程连续运行24小时稳定性测试
│  └─ Step 4.5: 断电重启后初始化流程验证
```

### 6.2 各阶段检查清单

#### 阶段1: 手动验证

- [ ] L1~L4各层的阻挡下降/上升按钮能正确控制对应气缸
- [ ] L1~L4各层的分料推出/复位按钮能正确控制对应气缸
- [ ] L1~L4各层的输送正转/反转按钮能正确控制对应变频器
- [ ] 同一层的阻挡上下按钮互斥有效（不能同时动作）
- [ ] 同一层的输送正反按钮互斥有效
- [ ] 所有28个传感器信号在HMI上可实时监控
- [ ] 无传感器冲突报警（ALM_106）
- [ ] 未选手动模式时手动按钮无效
- [ ] 存在故障时手动按钮无效

#### 阶段2: 单层自动测试

- [ ] 自动模式下按启动，状态机进入步骤1("检测材料...")
- [ ] 分料前感应器ON后转到步骤2("阻挡下降...")
- [ ] 阻挡下位ON后转到步骤3("输送中...")
- [ ] 双到位传感器同时ON后转到步骤4("到位检测...")
- [ ] 到位稳定定时器到期后转到步骤5("阻挡上升...")
- [ ] 阻挡上位ON后转到步骤6("分料中...")
- [ ] 分料下位ON后转到步骤7("慢速送出...")
- [ ] 慢速送出完成+分料复位OK后转到步骤8("放料完成")
- [ ] 500ms脉冲到期后回到步骤0（"待机"）
- [ ] 整个过程`o_b运行中`=TRUE，HMI运行灯亮
- [ ] 人为遮挡传感器可触发对应超时报警
- [ ] 按复位按钮可清除报警并重新初始化

#### 阶段3: 多层联动测试

- [ ] 4层同时放入材料，4个实例的状态机独立运转
- [ ] 各层步序可以不同步（如L1在步骤3，L4在步骤7）
- [ ] L2出现故障时，L1/L3/L4继续正常运行不受影响
- [ ] 按停止按钮，4层全部立即回到步骤0
- [ ] `o_b运行中`=TRUE当且仅当至少一层在自动运行
- [ ] `o_b故障`=TRUE当且仅当至少一层有报警
- [ ] `o_i当前状态`显示优先级最高的层状态（故障=99优先）
- [ ] `o_Lx当前步序[1..4]`分别显示各层独立步序
- [ ] 多层同时报警时`o_i本站报警代码`输出最小编号的报警码

#### 阶段4: 上游联调

- [ ] 取放料机构完成取料后`i_b取放料完成[n]`变为TRUE
- [ ] 本FB收到取放料完成信号后可进入下一循环
- [ ] `o_b放料完成[n]`发出500ms宽度的脉冲信号
- [ ] 取放料机构能正确接收并响应`o_b放料完成`脉冲
- [ ] 连续运行10个循环以上无异常
- [ ] 24小时老化测试无异常停机

## 8. 使用约束与禁忌

### 8.1 禁止事项

| 序号 | 禁止操作 | 后果 | 正确做法 |
|:----:|:--------|:-----|:--------|
| 1 | **同时设置`i_b自动模式`和`i_b手动模式`为TRUE** | 功能块进入不确定状态，可能全部输出归零 | 确保两者互斥（HMI应做联锁） |
| 2 | **在`i_b使能`=FALSE时尝试启动** | 启动信号被忽略，状态机不会离开步骤0 | 先确保系统就绪条件满足 |
| 3 | **忽略报警直接修改内部变量绕过状态机** | 破坏状态机逻辑，可能导致不可预测的行为 | 只通过标准输入接口操作 |
| 4 | **ARRAY索引超出[1..4]范围** | PLC运行时错误(RTE)，程序可能崩溃 | 严格使用1~4的整数索引 |
| 5 | **将传感器输入硬接线为常TRUE/常FALSE** | 失去保护和检测功能，可能导致安全事故 | 必须由真实传感器状态决定 |
| 6 | **在容器模式下直接操作内部实例(fbLayer1~4)** | 绕过容器的统一管理，破坏架构设计 | 通过容器接口统一操作 |

### 8.2 注意事项

| 序号 | 注意事项 | 说明 | 建议 |
|:----:|:--------|:-----|:-----|
| 1 | **启动信号必须做边沿检测** | 否则每次扫描都会重复触发启动 | 使用R_TRIG指令检测上升沿 |
| 2 | **双到位传感器必须同时有效** | `i_到位感应器1` AND `i_到位感应器2` 才确认到位 | 任一失效都会卡在步骤3 |
| 3 | **纯逻辑设计原则** | FB内部禁止直接读写X/Y/M/D物理地址 | 所有IO通过主控映射传入传出 |
| 4 | **报警代码需上报全局系统** | `o_i本站报警代码`要连接到公共报警FB | 否则HMI看不到此站的报警 |
| 5 | **复位后需等待初始化完成** | 初始化最多需要10秒（等待气缸安全位置） | HMI应有"初始化中..."提示 |
| 6 | **定期备份参数设置** | `i_r*`系列参数可能在HMI中被误修改 | 建议保存默认参数配置文件 |
| 7 | **DI3/DI4模块不要搞混** | DI3服务L3+L4，DI4服务L1+L2，地址范围不同 | 打印IO地址表贴在现场 |

### 8.3 推荐做法

| 序号 | 推荐做法 | 好处 |
|:----:|:--------|:-----|
| 1 | HMI增加各层独立步序显示(`o_Lx当前步序[1..4]`) | 快速定位哪一层出了问题 |
| 2 | 为每层分配独立的HMI手动操作面板 | 调试某层时不会误操作其他层 |
| 3 | 记录每个循环的周期时间(步骤0→0) | 用于评估节拍是否符合产能要求 |
| 4 | 在HMI显示4层报警码历史列表 | 快速回溯故障发生的时间和类型 |
| 5 | 定期测试急停和安全门功能 | 确保安全系统始终可靠 |
| 6 | 建立完整的报警日志系统(MES集成) | 记录每次报警的时间、原因、持续时间 |
| 7 | 为传感器增加状态指示灯(HMI) | 远程判断传感器健康状态 |
