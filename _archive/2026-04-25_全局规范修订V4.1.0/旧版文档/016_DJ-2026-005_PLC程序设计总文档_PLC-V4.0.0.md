## 2. 版本变更说明 (V3.0.0 -> V4.0.0)

### 2.1 架构重构核心变化

本次V4.0.0版本对PLC程序进行了根本性的架构重构，从V3.0.0的三层碎片化架构升级为扁平化的组件架构。

#### 详细对比表

| 对比维度 | V3.0.0 (旧架构) | V4.0.0 (新架构) | 变更收益 |
|----------|------------------|------------------|----------|
| **组件数量** | 21个功能块(6+9+6) | 5个组件(1+4) | 减少76%，大幅降低复杂度 |
| **架构层次** | 三层(Device/Logic/Main) | 扁平化(主控+工站) | 消除层间传递开销 |
| **IO映射方式** | 分散在Device层各FB中 | 集中在主控程序一处 | 单点修改，降低出错风险 |
| **物理地址依赖** | Device层FB直接引用X/Y/M/D | 所有工站FB为纯逻辑块 | 工站可独立测试和复用 |
| **数据耦合度** | 高(多层嵌套调用，接口复杂) | 低(主控统一协调，接口清晰) | 提升可读性和可维护性 |
| **报警系统** | 集中在Main_AlarmManager | 独立FB_公共报警 + D400~D425固定分配 | 支持MES对接，去重队列 |
| **状态机分布** | 分散在多个Logic FB中 | 每个工站FB内置完整状态机 | 工站内聚性更强 |
| **手动操作** | 通过多层传递 | HMI M区->主控->工站(直传) | 信号路径短，响应快 |
| **扩展性** | 新增功能需修改多层 | 新增工站只需新增一个FB | 符合开闭原则 |

## 3. 项目概述

### 3.1 系统功能

边框缓存机是一种自动化设备，用于存储和管理边框材料。主要功能包括：

| 功能类别 | 详细描述 |
|----------|----------|
| **多层存储** | 4层独立存储结构，每层可独立运行分料/输送循环 |
| **自动分料** | 通过阻挡气缸 + 分料气缸 + 输送带的配合实现精确分料 |
| **自动取放料** | Z轴升降 + X1轴横移 + 前后夹爪协同工作，一次取两根边框 |
| **4层循环取料** | 第1次取L1+L2层，第2次取L3+L4层，共2次循环完成全部取料 |
| **安全保护** | 急停按钮、双安全门锁定、安全继电器回路 |
| **与打胶机交互** | 通过硬接线信号(X75-X79,X102/Y50,Y51,Y44,Y47)实现自动对接，含故障/急停/通讯异常联锁 |
| **与组框机交互** | 预留接口(7DI+5DO)，含安全联锁/开门互锁/送料请求协调，地址待现场确定 |
| **外部设备报警** | 300段报警码(打胶机300~302, 组框机310~312)，纳入公共报警FB统一管理 |
| **HMI人机界面** | 模式选择(自动/手动)、启停控制、参数设定、状态监控、报警显示 |
| **MES数据接口** | D406~D425提供最近10条去重报警队列供MES系统读取上传 |

### 3.2 核心工艺流程

```
                    完整工艺流程 (V4.0.2架构视角 - 含外部设备交互)
                    ===============================================

  [组框机(上游)] ←→ [边框缓存机(本设备)] ←→ [打胶机(下游)]
       ↑ 预留接口          ↓                    ↑ 已定义接口
   (7DI+5DO)        [四层输送机] → [取放料机构] → [打胶机送料机构]
                    (分料/输送)    (Z升降+X1横移)    (X2横移+交互)

  外部设备交互协议:
  ┌───────────┐  控制信号  ┌──────────────┐  控制信号  ┌───────────┐
  │  组框机   │ ◄──────► │  边框缓存机   │ ◄──────► │  打胶机   │
  │ (上游)    │          │   (本设备)    │          │ (下游)    │
  └───────────┘          └──────────────┘          └───────────┘
     输入:              主控Step3处理:
     ·自动中             ·打胶机有效性检查+安全回退
     ·允许送料           ·组框机急停联锁(停止本机)
     ·有料请求           ·开门互锁(暂停+确认)
     ·开门请求           ·就绪信号生成
     ·急停/安全异常      ·送料请求协调(激活)
     ·通讯异常           ·外部设备报警汇总(300段)
     输出:
     ·请求送料
     ·暂停/急停回传
     ·申请开门
     ·就绪

  四层输送机 (每层独立9步):
    材料到达 → 阻挡下降 → 输送正转 → 到位检测 → 阻挡上升 → 分料推出 → 慢速送出 → 放料完成

  取放料机构 (全局11步, 一次取两层):
    等待放料完成(L1+L2或L3+L4) → Z轴下降 → 夹爪夹紧(前后同时) → 产品检测(4传感器)
    → Z轴上升 → X1轴后移到放料位 → Z轴下降 → 夹爪松开 → Z轴上升 → 放料完成通知

  打胶机送料机构 (6步):
    等待取放料放料完成 → X2轴前移到取料位 → 等待打胶机允许送料(X76)
    → 发送允许抓料(Y44) → 等待打胶机取料完成(X102) → X2轴返回待机位 → 循环
```

### 6.2 FB_取放料机构 -- 取放料控制

#### 6.2.1 功能概述

控制Z轴升降、X1轴横移和4组夹爪(前/后/前2/后2)的协同工作，实现从四层输送机上取料并放到打胶机送料位置的完整循环。**核心特点：一次取两根边框（长边x2 + 短边x2），4层循环2次完成全部取料。**

#### 6.2.2 取料策略

```
  4层取料循环策略
  ================

  总层数: 4层 (L1=底层, L4=顶层)
  每次取: 2层 (相邻两层)
  循环次数: 2次

  第1次取料 (s_i取料次数=1):
    当前层号 s_i当前取料层号 = 1
    等待条件: L1放料完成 AND L2放料完成
    取料对象: L1 + L2 (相邻底层两层)

  第2次取料 (s_i取料次数=2):
    当前层号 s_i当前取料层号 = 3
    等待条件: L3放料完成 AND L4放料完成
    取料对象: L3 + L4 (相邻顶层两层)

  完成: s_i已完成层数 >= 4 => 回到空闲，等待下一轮启动
```

#### 6.2.3 状态机设计 (11步全局状态机)

```
  取放料机构状态机 (全局唯一，非分层)
  ================================

  [STP_空闲(0)] ──启动──> [STP_等待放料完成(1)]
       ^                         |
       |                    对应层放料完成
       |                         v
       |            [STP_取料下降(2)] ──Z轴下位──> [STP_夹爪夹紧(3)]
       |                  |                          |
       |              超时(103)                   夹紧确认时间到
       |                  |                          v
       |            [FAULT]               [STP_产品检测(4)]
       |                                             |
       |                                       4传感器全检测到
       |                                             v
       |                              [STP_取料上升(5)] ──Z轴上位──> [STP_移动到放料位置(6)]
       |                                    |                          |
       |                                超时(104)               X1轴到达放料位
       |                                    |                          v
       |                              [FAULT]           [STP_放料下降(7)] ──Z轴下位──>
       |                                                          |          [STP_夹爪松开(8)]
       |                                                      超时(103)        |
       |                                                          |    松开确认到
       |                                                    [FAULT]   v
       |                                                  [STP_放料上升(9)] ──Z轴上位──>
```

### 6.3 FB_打胶机送料机构 -- 打胶机交互控制

#### 6.3.1 功能概述

接收取放料机构的放料完成信号后，控制X2轴移动到打胶机取料位置，通过硬接线信号与打胶机进行握手交互，完成送料循环。

#### 6.3.2 安全区管理逻辑

```
  安全区互锁机制
  =============

  目的: 防止取放料机构和打胶机同时在工作区域造成机械碰撞

  原则: 取放料机构优先(主动方，先放料后打胶机才能进入)

  判断依据:
    i_取放料_放料完成 = TRUE  =>  取放料机构已离开工作区
    i_取放料_放料完成 = FALSE =>  取放料机构可能还在工作中

  安全区信号输出(o_安全区信号):
    TRUE  = 安全区开放(打胶机可以进入工作区域)
    FALSE = 安全区关闭(禁止打胶机进入，防止碰撞)

  信号流向:
    o_安全区信号 --> 主控 --> Y47 --> 打胶机控制器
```

#### 6.3.3 与打胶机的硬接线握手协议

```
  PLC <-> 打胶机 硬接线信号时序
  ============================

  信号列表:
    X76 : 打胶机允许送料  (打胶机->PLC)  "我准备好了，可以给我送边框"
    X102: 打胶机取料完成  (打胶机->PLC)  "我已经抓走了边框"
    Y44 : 允许抓料        (PLC->打胶机) "边框已经到位，你可以抓取"
    Y47 : 安全区信号      (PLC->打胶机) "工作区域安全/不安全"

  正常交互时序:

    PLC侧                    打胶机侧
    =====                    ==========

    1. X2轴移动到取料位
    2. 等待 X76=TRUE  <───────────────── "我准备好了"
    3. Y44=TRUE  ────────────────────────> "可以抓料"
    4. 等待 X102=TRUE <───────────────── "我抓完了"
    5. Y44=FALSE ───────────────────────> (关闭允许)
    6. X2轴返回待机位
    7. (循环回到步骤1)

  异常情况:
    - X76长时间不来 -> 201号报警(通讯超时, 10s)
    - X102长时间不来 -> 201号报警(通讯超时, 10s)
    - 取放料还在工作时X2试图进入 -> 202号报警(安全区冲突)
```

#### 6.3.4 状态机设计 (6步)

| 步骤 | 名称 | 执行动作 | 转移条件 | 超时报警 |
|------|------|----------|----------|----------|
| 0 | 空闲/待机 | 全部OFF | 启动触发 | - |
| 1 | 等待放料完成 | 等待取放料放料完成(即安全区开放) | i_取放料_放料完成=TRUE | - |
| 2 | 移动到取料位置 | X2轴向前移动请求 | X2到达取料位 | 204(5000ms) |
| 3 | 等待允许送料 | 等待打胶机X76信号 | i_打胶机允许送料=TRUE | 201(10000ms) |
| 4 | 发送允许抓料 | Y44=TRUE，等待打胶机取料完成 | i_打胶机取料完成=TRUE | 201(10000ms) |
| 5 | 返回待机位置 | X2轴向后移动请求 | X2到达待机位 | 204(5000ms) |

> 步骤5完成后自动回到步骤1，形成连续循环。

#### 6.3.5 报警类型 (201~205段)

| 码值 | 名称 | 触发条件 | 严重程度 |
|------|------|----------|----------|
| 201 | 通讯超时 | 打胶机响应超时(允许送料或取料完成信号超时) | 中 |
| 202 | 安全区冲突 | 取放料机构仍在工作区时X2轴试图进入 | 高 |
| 203 | X2轴伺服故障 | X2轴驱动器ALM输出 | 高 |
| 204 | 移动超时 | X2轴移动未在规定时间内到达目标位置 | 中 |
| 205 | 到达超时 | 到达目标位置后等待下游响应超时 | 低 |

### 6.4 FB_公共报警 -- 报警汇总管理

#### 6.4.1 功能概述

作为独立的报警管理中心，汇聚三个工站的报警代码，提供统一的报警字计算、优先级选取、MES数据接口等功能。

#### 6.4.2 报警码段分配

```
  报警码段分配 (便于区分来源)
  ==========================

  |------- 1~99 ---------|---- 101~199 -------|---- 201~299 -------|
  |   四层输送机          |   取放料机构         |   打胶机送料机构     |
  |   (高优先级)          |   (中优先级)         |   (低优先级)         |
  |   7种报警码           |   8种报警码          |   5种报警码          |
  |-----------------------|----------------------|---------------------|

  优先级规则: 数值越小 -> 优先级越高
  当多站同时报警时，显示数值最小的那个
```

#### 6.4.3 核心算法

**全局报警字计算**:
```
o_w全局报警字 = 0
IF 输送机报警 != 0 THEN  o_w全局报警字 |= INT_TO_WORD(输送机报警)
IF 取放料报警 != 0 THEN  o_w全局报警字 |= INT_TO_WORD(取放料报警)
IF 送料机构报警 != 0 THEN o_w全局报警字 |= INT_TO_WORD(送料机构报警)
```

**最高优先级选取** (数值最小优先):
```
最小值初始化为0
遍历三站: 如果该站报警非零 且 (最小值为0 或 该站报警<最小值) 则更新最小值
最终输出最小值作为当前报警码
```

**MES队列入队规则**:
1. 仅在有报警激活时才考虑入队
2. 先累加报警计数(含重复)
3. 遍历现有队列检查去重(相同码不入队)
4. 去重通过则写入队尾索引位置
5. 队列指针+1，模10回绕
6. 复位时整个队列清零

**新报警脉冲生成**:
- 检测三站任一报警输入发生变化(与上周快照比较)
- 变化且当前有报警 -> o_b新报警标志=TRUE(仅当周期)
- 下一周期自动恢复FALSE
