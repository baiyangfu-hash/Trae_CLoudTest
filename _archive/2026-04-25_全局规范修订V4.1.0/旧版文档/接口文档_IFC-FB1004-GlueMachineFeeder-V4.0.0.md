## 3. 输入接口定义 (VAR_INPUT)

### 3.1 系统控制信号组 (3个BOOL)

#### 3.1.1 i_b使能

| 属性 | 值 |
|------|-----|
| **针脚名称** | `i_b使能` |
| **数据类型** | BOOL |
| **功能描述** | 总使能信号，控制本功能块的运行/停止 |
| **有效值域** | TRUE=允许运行, FALSE=停止并清零全部输出 |
| **默认行为** | FALSE时强制归零所有输出，状态机和计数器复位 |
| **来源** | 主控系统就绪标志（系统急停解除、安全门关闭等条件满足后置TRUE） |
| **更新周期** | 每扫描周期 |
| **优先级** | 最高（覆盖所有其他输入） |

**使用示例**:
```st
i_b使能 := gb系统就绪;  (* 系统全局就绪标志 *)
```

#### 3.1.2 i_b自动模式

| 属性 | 值 |
|------|-----|
| **针脚名称** | `i_b自动模式` |
| **数据类型** | BOOL |
| **功能描述** | 自动运行模式选择信号 |
| **有效值域** | TRUE=选中自动模式, FALSE=未选中 |
| **互斥关系** | 与`i_b手动模式`互斥（不能同时为TRUE） |
| **来源** | HMI M区变量（操作员通过HMI界面切换） |
| **更新方式** | 操作员手动切换 |

**重要约束**: 若`i_b自动模式`与`i_b手动模式`同时为FALSE，则功能块进入安全停机状态（全部输出归零）。

#### 3.1.3 i_b手动模式

| 属性 | 值 |
|------|-----|
| **针脚名称** | `i_b手动模式` |
| **数据类型** | BOOL |
| **功能描述** | 手动调试模式选择信号 |
| **有效值域** | TRUE=选手动模式, FALSE=未选中 |
| **互斥关系** | 与`i_b自动模式`互斥（不能同时为TRUE） |
| **来源** | HMI M区变量（操作员通过HMI界面切换） |
| **更新方式** | 操作员手动切换 |

### 3.2 操作按钮组 (3个BOOL)

#### 3.2.1 i_b启动

| 属性 | 值 |
|------|-----|
| **针脚名称** | `i_b启动` |
| **数据类型** | BOOL |
| **功能描述** | 自动循环启动按钮信号 |
| **触发方式** | **上升沿触发**（建议主控做边沿检测） |
| **生效条件** | 仅在`i_b自动模式=TRUE`且`i_b使能=TRUE`时有效 |
| **来源** | HMI按钮（操作员按下启动按钮） |
| **典型用法** | 按下后状态机从步骤0(空闲)进入步骤1(等待放料完成) |

**实现建议**: 在主控程序中使用R_TRIG指令检测上升沿。

#### 3.2.2 i_b停止

| 属性 | 值 |
|------|-----|
| **针脚名称** | `i_b停止` |
| **数据类型** | BOOL |
| **功能描述** | 自动循环停止按钮信号 |
| **触发方式** | **电平有效**（按下即停止） |
| **生效条件** | 无需模式判断，随时可停止 |
| **行为描述** | 按下后状态机立即回到步骤0(空闲)，清除所有运动请求 |
| **来源** | HMI按钮（操作员按下停止按钮） |

#### 3.2.3 i_b复位

| 属性 | 值 |
|------|-----|
| **针脚名称** | `i_b复位` |
| **数据类型** | BOOL |
| **功能描述** | 故障复位/初始化按钮信号 |
| **触发方式** | **电平有效**（按下执行复位） |
| **行为描述** | 清除所有报警锁存标志，重新执行初始化流程 |
| **前置条件** | 必须先排除故障原因（如伺服故障、通讯超时等） |
| **来源** | HMI按钮（操作员在故障排除后按复位） |

**复位后的动作序列**:
1. 清除所有报警锁存标志
2. `o_i本站报警代码 := 0`
3. `o_b故障 := FALSE`
4. 强制状态机回到步骤0(空闲)
5. 重新执行初始化流程（检查伺服状态）
6. 初始化完成后可重新启动

### 3.6 打胶机交互信号组 (2个BOOL)

> 这些是来自**外部打胶机**的硬接线信号，已由主控完成IO映射。

#### 3.6.1 i_打胶机允许送料

| 属性 | 值 |
|------|-----|
| **针脚名称** | `i_打胶机允许送料` |
| **数据类型** | BOOL |
| **功能描述** | 打胶机允许送料信号（表示打胶机准备好接收边框） |
| **对应物理地址** | X76 （来自打胶机的硬接线输入） |
| **信号方向** | 输入（从打胶机到本PLC） |
| **有效电平** | TRUE = 打胶机已准备好，可以送边框过去 |
| **时序要求** | 在步骤3（等待打胶机允许送料）中检测此信号 |
| **超时保护** | 如果10秒内未收到TRUE → 触发ALM_201（通讯超时） |
| **典型时序** | 当打胶机的上道工序完成、夹具准备就绪后置为TRUE |

#### 3.6.2 i_打胶机取料完成

| 属性 | 值 |
|------|-----|
| **针脚名称** | `i_打胶机取料完成` |
| **数据类型** | BOOL |
| **功能描述** | 打胶机取料完成信号（表示打胶机已完成边框抓取动作） |
| **对应物理地址** | X102 （来自打胶机的硬接线输入） |
| **信号方向** | 输入（从打胶机到本PLC） |
| **有效电平** | TRUE = 打胶机已完成抓取，可以收回 |
| **时序要求** | 在步骤4（发送允许抓料+等待取料完成）中检测此信号 |
| **超时保护** | 如果10秒内未收到TRUE → 触发ALM_201（通讯超时） |
| **典型时序** | 打胶机收到允许抓料信号(Y44)后，执行抓取动作，完成后置位此信号 |

**交互时序图**:
```
本FB(X2轴)              打胶机(外部)
   │                        │
   │── 移动到取料位 ────────>│
   │                        │
   │── (等待...)             │
   │                        │
   │<── 允许送料(X76)=ON ──│  打胶机准备好
   │                        │
   │── 允许抓料(Y44)=ON ──>│  开始抓取
   │                        │
   │── (等待抓取完成...)     │  执行抓取动作...
   │                        │
   │<── 取料完成(X102)=ON ─│  抓取完成!
   │                        │
   │── 允许抓料(Y44)=OFF ─>│  关闭信号
   │                        │
   │── 返回待机位 ─────────│
```

### 3.7 上游信号组 (1个BOOL)

#### 3.7.1 i_取放料_放料完成

| 属性 | 值 |
|------|-----|
| **针脚名称** | `i_取放料_放料完成` |
| **数据类型** | BOOL |
| **功能描述** | 取放料机构放料完成信号（上游设备通知本FB可以开始工作） |
| **信号来源** | FB_1003_PickPlace_BufferFraming.o_放料完成_给送料机构 |
| **有效电平** | TRUE = 取放料机构已完成放料并离开工作区 |
| **多重作用** | ① 步骤1的转换条件（放料完成后才能开始X2轴移动）<br>② 决定`o_安全区信号`的状态（同步输出给打胶机） |
| **时序配合** | 此信号必须保持稳定（建议至少500ms），直到本FB开始步骤2 |

**安全区联动机制**:
```
i_取放料_放料完成 的双重作用:

作用1: 状态机转换条件
├─ 步骤1中持续检测此信号
├─ =TRUE 时转出到步骤2（开始移动X2轴）
└─ =FALSE 时保持在步骤1等待

作用2: 安全区信号输出
├─ o_安全区信号 := i_取放料_放料完成
├─ =TRUE → 打胶机可以进入工作区
└─ =FALSE → 打胶机禁止进入工作区（防止碰撞）
```

## 4. 输出接口定义 (VAR_OUTPUT)

### 4.1 执行器请求输出组 (2个BOOL)

> 这些输出需要由**主控程序映射到物理Y地址**。

#### 4.1.1 o_允许抓料

| 属性 | 值 |
|------|-----|
| **针脚名称** | `o_允许抓料` |
| **数据类型** | BOOL |
| **功能描述** | 允许抓料信号（通知打胶机可以抓取边框） |
| **映射目标** | CPU: Y44 （具体以IO分配表为准） |
| **有效电平** | TRUE = 允许打胶机执行抓取动作 |
| **激活时机** | 仅在**步骤4（发送允许抓料+等待取料完成）**期间为TRUE |
| **保持时间** | 从打胶机允许送料开始，到打胶机取料完成为止（取决于打胶机实际速度） |
| **安全考虑** | 超时后自动置为FALSE（防止一直发送允许信号） |

**时序特性**:
```
o_允许抓料 信号时序:

步骤4开始 ──────────────────────────── 步骤4结束
    │                                    │
    ├─ o_允许抓料 = TRUE ─────────────┤
    │                                    │
    │<──── 等待 i_打胶机取料完成 ────>│
    │                                    │
    └─ o_允许抓料 = FALSE ────────────┘
         (进入步骤5)
```

#### 4.1.2 o_安全区信号

| 属性 | 值 |
|------|-----|
| **针脚名称** | `o_安全区信号` |
| **数据类型** | BOOL |
| **功能描述** | 安全区状态信号（通知打胶机是否可以进入工作区域） |
| **映射目标** | CPU: Y47 （具体以IO分配表为准） |
| **有效电平** | TRUE = 安全区开放（打胶机可进入），FALSE = 安全区锁定（禁止进入） |
| **逻辑关系** | 与`i_取放料_放料完成`信号**完全同步** |
| **接收方** | 打胶机控制系统（用于联锁控制） |
| **安全等级** | 高（直接影响设备安全） |

**安全区逻辑真值表**:

| i_取放料_放料完成 | o_安全区信号 | 含义 | 打胶机动作 |
|:-:|:-:|:---|:---|
| FALSE | FALSE | 取放料机构在工作区 | **禁止进入**（防止碰撞） |
| TRUE | TRUE | 取放料机构已离开工作区 | **允许进入**（安全） |

**重要性**: 这是防止取放料机构和打胶机发生机械碰撞的关键安全信号！

### 4.3 报警输出组 (1个INT)

#### 4.3.1 o_i本站报警代码

| 属性 | 值 |
|------|-----|
| **针脚名称** | `o_i本站报警代码` |
| **数据类型** | INT |
| **功能描述** | 当前最高优先级的活跃报警代码 |
| **接收方** | 公共报警管理器 (FB_Main_AlarmManager 或 FB_2001_CommonAlarm_AllStation) |
| **取值范围** | 0（无报警）或 201~205（有报警） |
| **优先级规则** | 数值越小优先级越高；多个报警同时存在时输出最小值 |

**完整报警码定义表**:

| 报警码 | 名称 | 触发条件 | 触发步骤 | 清除条件 | 优先级 |
|:------:|:-----|:---------|:--------|:---------|:--------:|
| 0 | 无报警 | - | - | - | - |
| 201 | 通讯超时(打胶机无响应) | 打胶机未在10秒内响应 | 步骤3 / 步骤4 | 复位按钮 | 高 |
| 202 | 安全区冲突(取放料未离开) | *(预留扩展)* | - | - | 中 |
| 203 | X2轴伺服故障 | X2轴伺服驱动器ALM有效 | 所有步骤(持续) | 故障消除+复位 | **最高** |
| 204 | X2轴移动超时 | X2轴未在5秒内到达目标 | 步骤2 / 步骤5 | 复位按钮 | 高 |
| 205 | 取料位置到达超时 | *(预留扩展)* | 步骤2 | 复位按钮 | 中 |

**报警优先级处理示例**:
```
示例场景: 同时出现 ALM_203 和 ALM_201
├─ ALM_203 (伺服故障) 优先级最高
└─ 输出: o_i本站报警代码 := 203

示例场景: 同时出现 ALM_201 和 ALM_204
├─ ALM_201 (数值更小) 优先级高于 ALM_204
└─ 输出: o_i本站报警代码 := 201
```

## 5. 时序要求汇总

### 5.1 关键时序参数

| 参数 | 默认值 | 用途 | 可调整性 |
|------|:------:|:-----|:--------:|
| 通讯超时时间 | 10000ms (10s) | 打胶机交互超时保护 | ✅ 可通过常量修改 |
| 移动超时时间 | 5000ms (5s) | X2轴移动超时保护 | ✅ 可通过常量修改 |
| 放料完成稳定时间 | ≥500ms | 上游信号稳定要求 | ⚠️ 由上游保证 |

### 5.2 信号时序配合要求

| 信号对 | 发送方 | 接收方 | 最小保持时间 | 说明 |
|:------:|:------:|:------:|:----------:|:-----|
| 放料完成→允许抓料 | 取放料机构 | 本FB | ≥500ms | 保证步骤1转换稳定 |
| 允许送料→取料完成 | 打胶机 | 本FB | ≤10s | 受超时保护限制 |
| 允许抓料→取料完成 | 本FB | 打胶机 | ≤10s | 受超时保护限制 |
