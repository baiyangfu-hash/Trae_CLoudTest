### 4.2 变更原因

#### 4.2.1 背景
原有FBD使用说明文档(`FBD使用说明_FBD-DJ-2026-005-V4.0.0.md`)存在以下问题：

1. **内容臃肿**: 单个文档约1065行，混合了架构设计、接口定义、调用示例、状态机详解、调试指南、FAQ、版本历史等多种不同职责的内容
2. **查找效率低**: 需要在1065行文档中搜索特定信息（如某个针脚定义或IO地址映射）
3. **维护困难**: 修改任何一部分都需要编辑整个大文档，容易引入格式错误
4. **不符合规范**: 未遵循项目的文档分层标准（DSN/IFC/UM/CHG分离原则）
5. **ASCII art冗余**: 包含大量过于详细的图形化表示（约350行），占用大量篇幅但信息密度低
6. **与同项目其他FB文档不一致**: 打胶机送料机构等已完成DSN/IFC/UM/CHG四类拆分

#### 4.2.2 必要性
用户明确要求：
- "为四层输送机(FB_1001_Conveyor4Layer + FB_1002_SingleLayerConveyor)创建以下3个文档"
- "基于现有的FBD使用说明文档内容"进行拆分
- "文档内容详实、格式规范、无重复内容(通过交叉引用链接)、符合项目Markdown规范"

#### 4.2.3 参考
- 用户需求: 2026-04-24 直接指令
- 项目规范: 文档分层标准（DSN设计、IFC接口、UM使用、CHG变更）
- 参考模板: 打胶机送料机构文档体系(DSN/IFC/UM/CHG已完成)

### 4.4 变更影响分析

#### 4.4.1 PMBOK五大约束评估

| 约束维度 | 影响等级 | 说明 |
|---------|:-------:|:-----|
| **范围 (Scope)** | 🟢 低 | 仅文档结构变化，不影响功能代码。新增3个文档，保留原FBD文档作为设计参考 |
| **进度 (Schedule)** | 🟢 低 | 文档重构耗时可接受（预计3小时），不影响开发进度 |
| **成本 (Cost)** | 🟢 无 | 纯文档工作，无额外硬件/软件成本 |
| **质量 (Quality)** | 🟡 中→高 | 提升文档质量和可维护性；降低后续维护成本；提高现场调试人员工作效率 |
| **风险 (Risk)** | 🟢 低→负 | 降低后续维护风险；提高团队协作效率；减少因文档不清导致的误操作 |

#### 4.4.2 双FB联动特殊考虑

本次变更涉及**两个功能块**的联合文档体系，需特别关注：

| 特殊考量点 | 说明 | 应对措施 |
|-----------|:-----|:---------|
| **版本同步要求** | FB_1001和FB_1002为强耦合关系（容器+单元），文档版本必须保持一致 | 所有4个文档统一使用V4.0.0版本号 |
| **接口引用关系** | IFC文档同时描述两个FB的接口，UM文档同时提供两种调用方式 | 在文档中明确标注适用对象(FB_1001或FB_1002) |
| **报警码段分配** | 四层输送机使用101~107段报警码，需确保不与其他FB冲突 | 在IFC文档中明确定义101~107段的归属 |
| **变更传播链** | 任一FB的源码变更可能需要同时更新多个文档 | 在CHG文档的版本联动规则中特别说明双FB情况 |

#### 4.4.3 跨领域影响

| 受影响领域 | 具体影响 | 应对措施 |

### 8.2 文档版本联动规则 (双FB特供版)

```
版本号联动机制 (FB_1001 + FB_1002 强耦合版):

当任一FB源代码(.st文件)发生修改时:
  │
  ├──→ CHG文档: 新增变更记录条目 (必须!)
  │        注意: 必须标注影响的是 FB_1001 还是 FB_1002 还是两者
  │
  ├──→ IFC文档: 
  │   ├─ 如果是 FB_1002 的接口变更(输入/输出增删改) → 升级版本号
  │   ├─ 如果是 FB_1001 的容器接口变更(ARRAY增删/公共信号变) → 升级版本号
  │   └─ 如果仅是参数默认值调整 → 可仅修订号(如 V4.0.0 → V4.0.0.1)
  │
  ├──→ UM文档: 
  │   ├─ 如果影响调用方式或使用流程 → 升级版本号
  │   └─ 如果仅是参数建议值微调 → 可仅修订号
  │
  └──→ 所有3个文档(IFC/UM/CHG)版本号必须保持同步!
  
示例:
  FB_1002源码: V4.0.0 → V4.0.1 (Bug修复: 超时常量调整)
      │
      ├──→ CHG-FB1001: V4.0.0 → V4.0.1 (追加变更记录)
      ├──→ IFC-FB1001: V4.0.0 → V4.0.1 (默认值表更新)
      └──→ UM-FB1001:  V4.0.0 → V4.0.1 (调优建议更新)
      
  如果只是文档修正(不影响功能):
      │
      └──→ 可以只升级文档版本号: V4.0.0 → V4.0.0.1 (修订号)
          (FB源代码版本不变)
```
