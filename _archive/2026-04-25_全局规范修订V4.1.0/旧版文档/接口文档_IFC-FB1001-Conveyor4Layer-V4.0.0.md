## 3. FB_1002_SingleLayerConveyor 输入接口定义 (VAR_INPUT)

> 以下为单层基础单元的完整24个输入接口定义。在容器模式下，这些输入通过 `FB_1001_Conveyor4Layer` 的公共信号和分层ARRAY信号映射到各层实例。

### 3.1 系统控制信号组 (3个BOOL)

#### 3.1.1 i_b使能

| 属性 | 值 |
|------|-----|
| **针脚名称** | `i_b使能` |
| **数据类型** | BOOL |
| **功能描述** | 总使能信号，控制本功能块的运行/停止 |
| **有效值域** | TRUE=允许运行, FALSE=停止并清零全部输出 |
| **默认行为** | FALSE时强制归零所有输出，状态机和计数器复位，定时器停止 |
| **来源** | 主控系统就绪标志（系统急停解除、安全门关闭等条件满足后置TRUE） |
| **更新周期** | 每扫描周期 |
| **优先级** | 最高（覆盖所有其他输入） |

#### 3.1.2 i_b自动模式

| 属性 | 值 |
|------|-----|
| **针脚名称** | `i_b自动模式` |
| **数据类型** | BOOL |
| **功能描述** | 自动运行模式选择信号 |
| **有效值域** | TRUE=选中自动模式, FALSE=未选中 |
| **互斥关系** | 与`i_b手动模式`互斥（不能同时为TRUE） |
| **来源** | HMI M区变量（操作员通过HMI界面切换） |

**重要约束**: 若`i_b自动模式`与`i_b手动模式`同时为FALSE，则功能块进入安全停机状态（全部输出归零）。

#### 3.1.3 i_b手动模式

| 属性 | 值 |
|------|-----|
| **针脚名称** | `i_b手动模式` |
| **数据类型** | BOOL |
| **功能描述** | 手动调试模式选择信号 |
| **有效值域** | TRUE=选手动模式, FALSE=未选中 |
| **互斥关系** | 与`i_b自动模式`互斥（不能同时为TRUE） |
| **来源** | HMI M区变量（操作员通过HMI界面切换） |

---

### 3.2 操作按钮组 (3个BOOL)

#### 3.2.1 i_b启动

| 属性 | 值 |
|------|-----|
| **针脚名称** | `i_b启动` |
| **数据类型** | BOOL |
| **功能描述** | 自动循环启动按钮信号 |
| **触发方式** | **上升沿触发**（建议主控做边沿检测） |
| **生效条件** | 仅在`i_b自动模式=TRUE`且`i_b使能=TRUE`时有效 |
| **来源** | HMI按钮 |

**实现建议**: 在主控程序中使用R_TRIG指令检测上升沿。

#### 3.2.2 i_b停止

| 属性 | 值 |
|------|-----|
| **针脚名称** | `i_b停止` |
| **数据类型** | BOOL |
| **功能描述** | 自动循环停止按钮信号 |
| **触发方式** | **电平有效**（按下即停止） |
| **生效条件** | 无需模式判断，随时可停止 |
| **行为描述** | 按下后状态机立即回到步骤0(空闲)，清除所有运动请求 |
| **来源** | HMI按钮 |

#### 3.2.3 i_b复位

| 属性 | 值 |
|------|-----|
| **针脚名称** | `i_b复位` |
| **数据类型** | BOOL |
| **功能描述** | 故障复位/初始化按钮信号 |
| **触发方式** | **电平有效**（按下执行复位） |
| **行为描述** | 清除所有报警锁存标志，重新执行初始化流程 |
| **前置条件** | 必须先排除故障原因（如传感器故障、超时条件等） |
| **来源** | HMI按钮 |

**复位后的动作序列**:
1. 清除所有报警锁存标志
2. `o_i报警代码 := 0`
3. `o_b故障 := FALSE`
4. 强制状态机回到步骤0(空闲)
5. 重新执行初始化流程（检查气缸安全位置）
6. 初始化完成后可重新启动

### 4.4 报警输出组 (1个INT)

#### 4.4.1 o_i报警代码

| 属性 | 值 |
|------|-----|
| **针脚名称** | `o_i报警代码` |
| **数据类型** | INT |

**完整报警码定义表 (101~105段)**:

| 报警码 | 名称 | 触发条件 | 触发步骤 | 清除条件 | 优先级 |
|:------:|:-----|:---------|:--------:|:---------|:------:|
| 0 | 无报警 | - | - | - | - |
| 101 | 阻挡下降超时 | 阻挡气缸未在规定时间内到达下位 | 步骤2 | 复位按钮 | 高 |
| 102 | 阻挡上升超时 | 阻挡气缸未在规定时间内到达上位 | 步骤5 | 复位按钮 | 高 |
| 103 | 分料推出超时 | 分料气缸未在规定时间内到达下位 | 步骤6 | 复位按钮 | 中 |
| 104 | 分料复位超时 | 分料气缸在慢速送出结束时未正常复位 | 步骤7 | 复位按钮 | 中 |
| 105 | 输送超时 | 材料未在规定时间内到达/离开到位位置 | 步骤1/步骤3 | 复位按钮 | 高 |
| 106 | 传感器冲突 | 阻挡或分料气缸的上下位传感器同时为TRUE | 所有步骤(持续检测) | 冲突消失后自动清除 | 低 |
| 107 | 变频器故障 | 变频器故障信号(预留扩展接口) | - | 故障消除+复位 | **最高** |

**报警优先级处理示例**:
```
示例场景: 同时出现 ALM_107 和 ALM_101
├─ ALM_107 (变频器故障) 优先级最高
└─ 输出: o_i报警代码 := 107

示例场景: 同时出现 ALM_101 和 ALM_105
├─ ALM_101 (数值更小) 优先级高于 ALM_105
└─ 输出: o_i报警代码 := 101
```

> **注意**: 在四层容器(FB_1001)中，报警代码统一使用101~105段（而非单层的1~7），以便在全站报警体系中唯一标识四层输送机的报警。

### A.3 报警码速查 (101~107段)

```
报警代码速查 (四层输送机专用):

  0   = 无报警 (正常)
  101 = 阻挡下降超时 (步骤2, >800ms未到下位)
  102 = 阻挡上升超时 (步骤5, >800ms未到上位)
  103 = 分料推出超时 (步骤6, >1000ms未到下位)
  104 = 分料复位超时 (步骤7, 未正常复位)
  105 = 输送超时      (步骤1/3, >5000ms未到位)
  106 = 传感器冲突    (上下位同时TRUE, 自动恢复)
  107 = 变频器故障    (最高优先级! 预留扩展)

处理方法:
  101~105 → 排除原因 → 按复位按钮
  106     → 检查磁性开关间距(推荐>10mm)
  107     → 排除变频器故障 → 复位
```
